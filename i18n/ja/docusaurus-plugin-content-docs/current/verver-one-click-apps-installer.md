## Introduction

ワンクリックアプリインストーラーを使うと、豊富なアプリカタログから直接VPSのウェブインターフェースでアプリを展開できます。このガイドでは、サーバーの準備、適切なインストールタイプの選択、アプリのインストールと管理、必要に応じた高度な設定やトラブルシューティングツールの使い方を学びます。

<InlineVoucher />

## Preparation

ワンクリックアプリインストーラーを使用する前に、VPSが最低要件を満たし、対応するOSを使用している必要があります。

:::info 最低要件
アプリ機能を利用するには、VPSがオンラインであり、プラットフォームサービスとインストールしたアプリを安定して動作させるために十分なリソースが設定されている必要があります。
:::

| 要件 | 最低 | 推奨 |
| --- | --- | --- |
| サーバーステータス | オンライン | オンライン |
| ストレージ | 30 GB | 50 GB |
| RAM | 2 GB | 4 GB |
| CPU | 2コア | 4コア |
| OS | Debian または Ubuntu LTS | Debian 13 - EN - 64bit |

:::caution 新規OSインストール推奨
既存のサービスや使用中のポート、Docker設定、カスタム構成との競合を避けるため、アプリ機能は新規OSインストール上でセットアップすることを強く推奨します。
:::

続行前にVPSを再インストールする必要がある場合は、VPS管理画面から対応するDebianまたはUbuntu LTSの新規イメージを使用してください。

## Accessing the One Click Apps Installer

ZAP-HostingのウェブインターフェースでVPSを開き、ワンクリックアプリセクションを探します。ここから初期セットアップを開始し、後でサーバーにインストールされたすべてのアプリを管理できます。

初めてこの機能を開くと、プラットフォームのインストール方法を選択する必要があります。

## Choosing an Installation Type

フルインストールとノードのみインストールのどちらかを選択できます。どちらを選ぶかは、このVPSが管理パネルとアプリの両方をホストするか、別のパネルのリンクノードとしてのみ機能するかによります。

### Full Installation

管理パネルとノードとしてのサーバーをセットアップしたい場合は、**フルインストール**を選択してください。初めてのユーザーや標準的な展開に推奨されるオプションです。

この方法でできること：

- アプリカタログからアプリを展開
- プロジェクトと環境を管理
- すべてのアプリを一元管理
- 統合されたウェブインターフェース管理ツールを使用
- 必要に応じて高度なパネルにアクセス可能

![img](https://screensaver01.zap-hosting.com/index.php/s/nqGL3PFPicTArY9/preview)

:::tip ほとんどのユーザーに推奨
アプリ機能を初めて使う場合はフルインストールを選択してください。最も簡単なセットアップで、ZAP-Hostingのウェブインターフェースから完全な管理が可能です。
:::

### Node-only Installation

このVPSをノードとしてのみ使いたい場合は、**ノードのみインストール**を選択します。別のVPSサーバーや外部パネルインスタンスにリンクできます。

このオプションは以下の用途に便利です：

- 複数のVPSにまたがるスケールアウト
- 管理を一つのパネルに集中
- 既存の展開にこのサーバーをリンク
- マルチノード構成の拡張

:::note 上級者向け展開オプション
ノードのみインストールは、すでに別のパネルインスタンスを持っているか、外部環境に接続したいユーザー向けです。
:::

## Installing the Apps Feature

インストールタイプを選択したら、ウェブインターフェースからセットアップを開始します。初期インストールは通常数分で完了します。

このプロセス中に、ワンクリックアプリ機能に必要なプラットフォームサービスがVPSに展開されます。完了すると、ウェブインターフェースにアプリカタログと管理セクションが表示されます。

:::info インストールの挙動
インストールの進行状況はVPSの性能、OSの状態、ネットワーク状況によって異なる場合があります。セットアップが正常に完了しない場合は、このガイドのトラブルシューティングセクションを参照してください。
:::

## Browsing the App Catalogue

インストール完了後、アプリカタログを開いて利用可能なアプリを閲覧できます。カタログには300以上の即展開可能なアプリがあり、フィルター機能で目的のアプリを素早く見つけられます。

![](https://screensaver01.zap-hosting.com/index.php/s/fBwcCiNJTZooqnB/preview)

### Searching and Filtering Apps

カタログ内ではアプリ名で検索したり、カテゴリで絞り込みができます。これにより、アプリストア風のインターフェースで展開対象を効率的に絞り込めます。

よく使う操作：

- アプリ名で検索
- カテゴリでフィルター
- 利用可能なアプリの確認
- 展開したいアプリの選択

多くのアプリ展開先を管理したり、複数サービスを比較検討する際に特に便利です。

### Starting an App Installation

カタログからアプリを選択したら、ウェブインターフェースでインストールを開始します。アプリによっては以下のような値を指定する必要があります：

- `[your_project_name]`
- `[your_environment_name]`
- `[your_domain]`
- `[your_app_name]`

これらは展開に特化した値で、例えば`[your_domain]`はアプリで使いたいドメインに置き換えてください。

:::tip プロジェクトと環境の活用
複数のアプリをホストしたり、バージョンのテスト、運用とステージングの分離にプロジェクトと環境を使うと展開が整理しやすくなります。
:::

## Managing Installed Apps

展開後、インストール済みアプリはウェブインターフェースの管理セクションに一覧表示されます。ここから、基盤のコンテナを直接操作せずにサービスの監視や制御が可能です。

![](https://screensaver01.zap-hosting.com/index.php/s/WTMSqkJ5zPKWjmn/preview)

### Available Management Actions

インストール済みアプリセクションでは、一般的なライフサイクル操作をウェブインターフェースから直接行えます。

| 操作 | 目的 |
| --- | --- |
| 起動 | 停止中のアプリを起動 |
| 停止 | 実行中のアプリを停止 |
| ログ確認 | トラブルシューティング用にアプリログを開く |
| 再インストール | アプリを再展開 |

これらのツールにより、コマンドライン操作なしで一元的にアプリ管理が簡単になります。

### Organizing Projects and Environments

プロジェクトと環境は展開を明確に構造化するのに役立ちます。以下のように分けて使えます：

- 個人用と本番用アプリ
- テスト環境とライブ環境
- 複数の顧客やチームの展開
- 同一VPS上の異なるスタック

アプリストアのアプリ数が増え、複数ノードで管理する際に特に重要になります。

## Domains, Cloud Access, and SSL

ワンクリックアプリ機能はカスタムドメインとZAP Cloud連携をサポートしています。これにより、アプリを安全に公開し、SSL証明書を自動で発行できます。

:::info ドメインとSSLのサポート
カスタムドメインをアプリに割り当てられ、プラットフォーム設定で対応していればSSL証明書の発行も利用可能です。
:::

ドメインを割り当てる際は、DNSレコードが必ず`[your_server_ip]`を指すように設定してください。正しく設定されていないと、アプリがインターネットからアクセスできなくなります。

| 値 | 説明 |
| --- | --- |
| `[your_domain]` | アプリに割り当てるドメインまたはサブドメイン |
| `[your_server_ip]` | VPSのパブリックIPアドレス |

:::caution DNSは必ずVPSを指すこと
`[your_domain]`のDNSが`[your_server_ip]`を指していない場合、ドメイン認証やアプリアクセスに失敗します。
:::

## Using Advanced Settings

上級ユーザーは高度な設定セクションから外部パネルにアクセスして、より詳細なカスタマイズや展開制御が可能です。

![](https://screensaver01.zap-hosting.com/index.php/s/BHnZWBGAzgo5myp/preview)

標準のウェブインターフェースを超えた機能や直接操作が必要な場合に便利です。

### Important Support Limitation

:::caution 高度なパネル利用時のサポート範囲
外部Coolifyパネルの利用は歓迎しますが、サポートはZAP-Hostingのウェブインターフェースに限定されます。ウェブインターフェース外での変更にはサポートが提供されない場合があります。
:::

### When to Use the Advanced Panel

以下のような場合に高度なパネルを使うと良いでしょう：

- 詳細な展開カスタマイズが必要
- ノードリンクの直接操作
- 高度なアプリ設定
- 簡易インターフェースを超えたパネル管理

アクセスを失ったり連携が停止した場合は、手動変更前に下記のトラブルシューティングを試してください。

## Linking Nodes

ノードのみインストールを選択した場合、このVPSを別のVPSや外部パネルにリンクできます。これによりインフラをスケールし、複数ノードを一元管理可能です。

ノードリンクは以下の用途に便利です：

- 複数VPSにアプリを分散
- 管理を集中化
- 孤立展開を作らずに拡張
- 外部パネル環境に接続

:::note 外部パネルリンク
ノードは別VPSや外部パネルにリンク可能です。リンクの詳細は既存の環境に依存します。
:::

## Troubleshooting Degraded Status

ウェブインターフェースで状態が劣化している場合、プラットフォームがOne Click Apps機能を動かすインスタンスに接続できていません。以下の手順を順に実施してください。

### Status Indicators

トラブルシューティングのポップアップには以下のような状態が表示されます：

| ステータス | 意味 |
| --- | --- |
| Coolify Status: Active | パネルサービスが稼働中の可能性あり |
| API Status: Accessible | APIに現在アクセス可能 |
| Service Status: Online | VPS自体がオンライン |

いずれかが正常でない場合は次の手順へ進みます。

### Step 1: Confirm the VPS Is Online

まず、ZAP-HostingのウェブインターフェースでVPSがオンラインであることを確認してください。サーバーがオフラインの場合、アプリプラットフォームは機能しません。

### Step 2: Refresh the Status

**ステータス更新**操作で最新データによるヘルスチェックを再実行します。これで一時的な通信障害が解消されることがあります。

:::tip 一時的な接続問題の可能性
劣化状態は短時間の通信問題が原因の場合があります。まずはステータス更新を試すのが最速です。
:::

### Step 3: Restart Docker

状態が改善しない場合は**Docker再起動**を実行してください。プラットフォームインスタンスが再起動されます。成功後は`1-2分`待ってから再度ステータスを更新します。

:::caution サービス再起動の注意
Docker再起動中は実行中のアプリが一時的に停止する可能性があります。
:::

### Step 4: Refresh Credentials

最後に**認証情報の更新**を行います。状態がオンラインでもアプリインストールなどの機能が動作しない場合に特に重要です。

この操作は：

- データを削除しません
- rootユーザーの認証情報を更新
- ウェブインターフェースが使うAPIトークンを更新
- 成功時にパネル認証情報を自動更新

:::info 認証情報更新は安全
認証情報の更新はインストール済みアプリや展開データを削除しません。アクセスと通信に必要な認証情報のみを更新します。
:::

### When to Contact Support

すべてのトラブルシューティングを試しても問題が解決しない場合は、ZAP-Hostingのウェブサイトからサポートチケットを作成してください。

## Best Practices

ワンクリックアプリインストーラーを安定して管理しやすく保つために、以下の推奨事項を守ってください：

| ベストプラクティス | 理由 |
| --- | --- |
| 新規OSインストールを使う | 既存サービスやポートとの競合を減らすため |
| 推奨リソースを満たす | パフォーマンスと信頼性向上のため |
| アプリをプロジェクトと環境で整理 | 展開管理を簡単にするため |
| ドメイン割り当て前にDNS設定を済ませる | アクセスとSSL問題を防ぐため |
| 高度な設定は慎重に使う | サポート対象外のカスタム変更を避けるため |

:::tip 展開を整理しよう
多くのアプリをホストする場合は、最初に`[your_project_name]`、`[your_environment_name]`、`[your_app_name]`の命名ルールを決めておくと便利です。
:::
