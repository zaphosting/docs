## はじめに

ホワイトリストは、参加する前にユーザーアカウントを明示的に作成する必要があるアクセス制御モードです。Project Zomboidでは、サーバー設定でオープン登録を無効にし、コンソールコマンドでユーザーを追加することで一般的に実現されます。

## ホワイトリストの有効化

FTPでサーバーに接続し、サーバー設定ファイルを開きます。通常は `Zomboid/Server/` フォルダ内にあり、サーバー名にちなんだファイル名（例：`servertest.ini` または `<ServerName>.ini`）です。`Open` 設定を探して、`false` に変更してください。

```ini
Open=false
```

ファイルを保存してサーバーを再起動します。オープン登録が無効になると、あなたが作成したアカウントのみが参加可能になります。

## ホワイトリストプレイヤーの管理

ZAP-Hostingのゲームサーバー管理画面のライブコンソールを開き、許可したいプレイヤーのアカウントを作成します。`adduser` コマンドを使い、ユーザー名とパスワードを指定してください。

```text
/adduser "username" "password"
```

ユーザーのアクセスを解除したい場合は、専用コマンドでホワイトリストから削除します。

```text
/removeuserfromwhitelist "username"
```

以前にオープン登録を許可していて、既に接続中のアカウントをホワイトリストに変換したい場合は、現在接続中の全ユーザーをホワイトリストに追加するコマンドを使います。

```text
/addalltowhitelist
```

## ホワイトリスト機能の確認

`Open=false` が有効になった後、`/adduser` で追加されていない新規ユーザーは参加できないはずです。ホワイトリストに登録されたアカウントでアクセスをテストしてみてください。

もし未登録ユーザーがまだ参加できる場合は、編集したのが実際に稼働中の `<ServerName>.ini` ファイルかどうか、そしてサーバーを再起動したかを確認してください。

## まとめ

上記の手順を正しく実行すれば、ホワイトリストが有効になり、誰がサーバーに参加できるかを正確にコントロールできます。もしアクセスに問題がある場合は、一度サーバーを再起動し、ファイルやコマンドの出力を再確認して変更が反映されているかチェックしましょう。

ご不明点やサポートが必要な場合は、いつでもお気軽にサポートチームまでご連絡ください！毎日対応していますよ！🙂
