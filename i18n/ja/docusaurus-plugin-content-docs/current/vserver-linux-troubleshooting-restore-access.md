## はじめに

よくあることです。Linuxサーバーにログインしようとしたら、パスワードが受け付けられない、または忘れてしまった。結果として、システムや重要なデータ、サービスへのアクセスができなくなります。特にサーバーが稼働中の業務に必要な場合、この状況はかなりストレスですよね。ほとんどのLinuxディストリビューションでは、OSを再インストールせずにリカバリーモードを使ってアクセスを復旧できます。以下の手順に従えば、サーバーへのアクセスを取り戻し、作業を最小限の中断で続けられます。

## アクセス復旧

リカバリーモードでパスワードをリセットするには、GRUBブートメニューにアクセスしてシステムを起動する必要があります。システム起動時にGRUBメニューを開き、**Advanced options for Ubuntu** を選択してください。

![img](https://screensaver01.zap-hosting.com/index.php/s/j6DKWdM95fFWWZi/preview)

次の画面で、**recovery mode** でシステムを起動するエントリを選びます。これにより通常のOSではなくリカバリー環境が読み込まれます。

リカバリーメニューが表示されたら、**root – Drop to root shell prompt** を選択して確定します。もう一度Enterを押すとrootシェルに入れます。

![img](https://screensaver01.zap-hosting.com/index.php/s/WFjTj3FxKQwHemD/preview)

リカバリーモードでは、rootファイルシステムはデフォルトで読み取り専用でマウントされています。パスワード変更を可能にするため、書き込み可能に再マウントする必要があります。以下のコマンドを実行してください：

```bash
mount -o remount,rw /
```

ユーザー名がわからない場合は、ホームディレクトリを確認して利用可能なユーザーアカウントを一覧表示できます：

```
ls /home
```

パスワードをリセットするには、`passwd` コマンドに続けてユーザー名を指定します：

```
passwd username
```

`username` は実際のアカウント名に置き換えてください。新しいパスワードを入力し、確認のため再入力します。パスワード入力は画面に表示されません。パスワードの更新が完了したら、`reboot` コマンドでシステムを再起動してください。

## まとめ

この手順を終えれば、目的のユーザーの新しいパスワードが設定されているはずです。これでリモートデスクトップ接続から再度ログインできます。もし質問やサポートが必要な場合は、いつでもお気軽にサポートチームにお問い合わせくださいね！🙂
