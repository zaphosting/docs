## はじめに

自動サーバーメッセージは、重要な情報をユーザーに直接伝えるシンプルだけど効果的な方法です。サーバールールの表示、イベント告知、便利なヒントの共有、定期的なリマインダーなど、管理者が手動で入力しなくても自動で配信できます。

**スケジュールタスク**を使えば、あらかじめ決めたコマンドを定期的に実行して、**Discord Bot**のチャンネルに自動でメッセージを送信できます。

## 自動メッセージの仕組み

自動メッセージは、指定した時間や間隔でサーバーコマンドを実行するタスクをスケジューリングして作成します。そのコマンドはチャットメッセージや放送、サーバー通知の送信に使えます。

設定が完了すると、タスクはバックグラウンドで自動的に動作し、接続中の全ユーザーにメッセージを表示します。これにより、スタッフがオンラインでなくても安定した情報発信が可能です。



## 自動メッセージの作成方法

自動サーバーメッセージを作成するには、ゲームサーバーのダッシュボードを開き、**スケジュールタスク**のセクションに移動します。新しいスケジュールタスクを作成し、**コマンド実行**のオプションを選択してください。コマンド欄に、ゲームサーバーが対応しているメッセージ送信コマンドを入力します。

```
<message command> <your message text>
```

`<message command>`はメッセージ送信に使いたいゲームコマンドや拡張機能のコマンドに置き換え、`<your message text>`はユーザーに表示したいメッセージテキストに置き換えてください。

その後、メッセージを送信する頻度（数分ごと、毎時、特定の時間など）を設定できます。スケジュールタスクの詳しい作成方法は、[スケジュールタスク](gameserver-scheduled-tasks.md)ガイドを参照してください。

## メッセージ送信間隔のカスタマイズ

スケジュールタスクは、メッセージの表示タイミングや頻度を自由にコントロールできます。よくある使い方は：

- 数分ごとにサーバールールを表示
- 固定時間にイベント告知を配信
- 新規・復帰ユーザー向けにゲームプレイのヒントを送信
- Discordやウェブサイトのリンクをリマインド

間隔はサーバーを再起動せずにいつでも変更可能なので、サーバーの成長に合わせて柔軟に調整できます。



## 注意点

自動サーバーメッセージの機能は、ゲーム本体や利用可能なコマンドの性能に依存します。基本的なスケジュールメッセージは標準のサーバーコマンドで実装可能ですが、フォーマットや条件分岐、高度なカスタマイズには制限がある場合があります。

もしゲームがプラグインやMOD、拡張機能をサポートしていて、より柔軟でカスタマイズ性の高いメッセージ機能を提供しているなら、それらを利用するのもおすすめです。そういった専用のメッセージシステムは、標準コマンドよりも豊富な機能を持ち、複雑または高度にカスタマイズされたサーバー運用に適しています。



## まとめ

自動サーバーメッセージは、手間をかけずにユーザーに情報を届けてエンゲージメントを高める信頼できる方法です。スケジュールタスクでメッセージコマンドを実行することで、サーバー全体で安定したコミュニケーションを実現できます。

一度設定すれば、自動メッセージは独立して動作し、ゲーム固有のメッセージコマンドやテキストを変えるだけで簡単に再利用可能です。
