## はじめに
ホワイトリストはセキュリティとアクセス制御の機能で、Hytaleゲームサーバーに参加できるプレイヤーを限定できます。有効にすると、ホワイトリストに明示的に登録されたプレイヤーのみが接続可能になります。これはプライベートサーバーやコミュニティサーバー、開発環境、テスト段階など、信頼できるプレイヤーだけにアクセスを制限したい場合に特に便利です。

## ホワイトリストを有効化する

ホワイトリストはサーバーコンソールまたはゲーム内から有効化できます。実行するにはオペレーター権限など必要な権限を持っていることを確認してください。

ホワイトリストを有効にするには、以下のコマンドを実行します：

```
/whitelist enable
```



有効化すると、ホワイトリストに登録されていないプレイヤーはサーバーへの参加が拒否されます。現在ホワイトリストが有効かどうか確認するには：

```
/whitelist status
```

このコマンドはホワイトリストの現在の状態を返し、設定が正しく適用されているか確認できます。



## ホワイトリストを無効化する

すべてのプレイヤーの参加を再び許可したい場合は、いつでもホワイトリストを無効化できます。以下のコマンドで無効化してください：

```
/whitelist disable
```

無効化後はホワイトリストによるアクセス制限が解除され、すべてのプレイヤーが接続可能になります。



## ホワイトリストの管理

ホワイトリストはサーバー稼働中でも動的に管理可能です。コマンドを使えばサーバー再起動なしでプレイヤーの追加・削除ができます。



### プレイヤーをホワイトリストに追加する

特定のプレイヤーにサーバーアクセスを許可するには、以下のコマンドでホワイトリストに追加します：

```
/whitelist add <playername>
```

ホワイトリストが有効な場合、指定したプレイヤーは即座にサーバーに参加可能になります。



### プレイヤーをホワイトリストから削除する

プレイヤーのアクセス権を取り消すには、以下のコマンドでホワイトリストから削除します：

```
/whitelist remove <playername>
```

削除後、そのプレイヤーはホワイトリストが有効な間はサーバーに参加できなくなります。



### ホワイトリストのプレイヤー一覧を表示する

現在サーバー参加が許可されているプレイヤー全員を確認するには、以下のコマンドを実行します：

```
/whitelist list
```

このコマンドはホワイトリストに登録されている全プレイヤー名を表示します。



### ホワイトリストを全削除する

ホワイトリストの全プレイヤーを一括で削除したい場合は、以下のコマンドでリストをクリアできます：

```
/whitelist clear
```

これによりホワイトリストの全エントリーが削除されますが、ホワイトリスト自体は無効化されません。



## ファイルでホワイトリストを管理する

コマンド以外に、設定ファイルを直接編集してホワイトリストを管理することも可能です。大量の変更やサーバー起動前の準備に便利です。設定ファイルはサーバーディレクトリ内の `whitelist.json` という名前です。

````
{
  "enabled": true,
  "players": [
    "PlayerName1",
    "PlayerName2"
  ]
}
````
