## はじめに

Windowsサーバーが正常に稼働しているのに、**コントロールパネル上でオフライン表示される**ことがあります。この現象は多くの場合、**ICMPv4 Echoリクエスト**をブロックするファイアウォール設定が原因です。これらのリクエストは有名な `ping` コマンドで使われており、監視システムがサーバーやIPアドレスの到達性を確認するために一般的に利用されています。

WindowsファイアウォールがICMP Echoリクエストをブロックすると、監視システムはサーバーからの応答を受け取れません。その結果、サーバーや割り当てられたIPアドレスがオフラインと表示されてしまいますが、実際にはサーバーは問題なく稼働しています。

![img](https://screensaver01.zap-hosting.com/index.php/s/ptD4W4MkAo85PAo/download)



## 原因

この問題の最も一般的な原因は、**Windows Defenderファイアウォール**にICMPv4 Echoリクエストを許可するルールが存在しないことです。このルールがないと、ファイアウォールはデフォルトでpingリクエストをブロックします。

監視システムはこれらの応答を使ってサーバーの稼働状況を判断しているため、サービスは正常に動いていてもサーバーがオフラインと見なされてしまいます。



## 解決方法

この問題を解決するには、ICMPv4 Echoリクエストを許可するファイアウォールルールを作成する必要があります。まずはVNCで接続しましょう。VPS管理画面の**VNCコンソール**をクリックし、普段使っているログイン情報でログインしてください。

![img](https://screensaver01.zap-hosting.com/index.php/s/mDQt7LLbLNSPC6a/download)

ログインしたら、**Windows Defender ファイアウォール 高度なセキュリティ**の画面を開きます。`Windows + R`キーを押して、以下のコマンドを入力しEnterを押してください：

```
wf.msc
```
高度なファイアウォール設定ウィンドウが開きます。左のメニューから**受信の規則**を選択します。サーバーは受信するICMPリクエストを許可する必要があるためです。右側のメニューから**新しい規則**を選んで新規ルールを作成しましょう。

ルール作成時に、ルールの種類は**カスタム**を選択します。次のステップでプロトコルの種類は**ICMPv4**を選択。プロトコルを選んだら、**カスタマイズ**設定を開き、**エコー要求**を有効にしてください。これでサーバーがpingリクエストに応答できるようになります。

ウィザードを進めて、アクションは**接続を許可する**を選択。プロファイルの選択では、通常はすべてのプロファイル（パブリック、プライベート、ドメイン）にチェックを入れておくと、どのネットワーク環境でもルールが適用されます。

最後にわかりやすい名前を付けましょう。例えば `Allow ICMPv4 Echo Requests` など。ウィザードを完了すると、新しいルールが即座にICMP Echoリクエストの受信を許可します。

![img](https://screensaver01.zap-hosting.com/index.php/s/iciYwqAkYLTMkgC/download)



## 動作確認

ルール作成後、サーバーがpingに応答するか確認しましょう。別の端末から以下のコマンドを実行します：
```
ping <YOUR_SERVER_IP>
```
サーバーが正常に応答すれば、監視システムも再びサーバーを検出できるようになります。しばらくするとコントロールパネルのサーバーステータスがオフラインからオンラインに変わるはずです。



## まとめ

おめでとう！WindowsファイアウォールにICMPv4 Echoルールを作成して、Windows VPSがpingリクエストに応答できるようになりました。

これで監視システムがサーバーの稼働状況を正しく検出でき、コントロールパネルでオフライン表示される問題を防げます。
