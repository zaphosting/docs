## はじめに

自動サーバーメッセージは、ゲーム内でプレイヤーに重要な情報を直接伝えるシンプルだけど超便利な方法です。サーバールールの表示、イベント告知、役立つヒントの共有、定期的なリマインダーなど、管理者が手動で入力しなくても自動で届けられます。

**スケジュールタスク**を使えば、あらかじめ設定したコマンドを定期的に実行して、**[Game]** のゲーム内チャットに自動でメッセージを送信できます。

## 自動メッセージの仕組み

自動メッセージは、指定した時間や間隔でサーバーコマンドを実行するタスクをスケジューリングして作成します。コマンドはチャットメッセージ、ブロードキャスト、サーバー通知などに使えます。

設定が完了すると、タスクはバックグラウンドで自動的に動作し、接続中の全プレイヤーにメッセージを表示します。スタッフがオンラインでなくても、安定した情報発信が可能です。

## 自動メッセージの作成方法

自動サーバーメッセージを作るには、ゲームサーバーのダッシュボードを開き、**スケジュールタスク**のセクションに移動します。新しいスケジュールタスクを作成し、**コマンド実行**のオプションを選択。コマンド欄にゲームサーバーが対応しているメッセージコマンドを入力します。

```
<message command> <your message text>
```

`<message command>` はメッセージ送信に使うゲームコマンドや拡張機能のコマンドに置き換え、`<your message text>` はプレイヤーに見せたいメッセージテキストに変えてください。

メッセージの送信頻度も設定可能で、数分ごと、毎時、特定の時間など自由に指定できます。スケジュールタスクの詳しい作り方は、[スケジュールタスク](gameserver-scheduled-tasks.md) ガイドをチェックしてね。

## メッセージ間隔のカスタマイズ

スケジュールタスクなら、メッセージの表示タイミングや頻度を自由にコントロール可能。よくある使い方は：

- 数分ごとにサーバールールを表示
- 固定時間にイベント告知
- 新規・復帰プレイヤー向けにゲームプレイのヒントを送信
- Discordやウェブサイトのリンクをリマインド

間隔はサーバーを再起動せずにいつでも変更できるので、サーバーの成長に合わせて柔軟に調整できます。

## 注意点

自動サーバーメッセージの機能は、ゲームや利用可能なコマンドの仕様に依存します。基本的なスケジュールメッセージは標準のサーバーコマンドで実装可能ですが、フォーマットや条件分岐、高度なカスタマイズには限界があります。

もしゲームがプラグインやMOD、拡張機能をサポートしていて、より柔軟でカスタマイズ性の高いメッセージ機能を提供しているなら、そちらを使うのもアリです。専用のメッセージングシステムは、標準コマンドよりリッチな機能を備えていて、複雑なサーバー運営に向いています。

## まとめ

自動サーバーメッセージは、手間なくプレイヤーに情報を届けて盛り上げるのに最適な方法です。スケジュールタスクでメッセージコマンドを実行するだけで、安定したコミュニケーションを実現できます。

一度設定すれば、ゲーム固有のメッセージコマンドやテキストを変えるだけで簡単に使い回せます。
