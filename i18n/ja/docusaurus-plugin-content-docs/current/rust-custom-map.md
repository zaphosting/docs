## はじめに

カスタムマップを使うことで、Rustのサーバーオーナーはデフォルトのプロシージャルマップを超えたユニークなゲーム体験を作り出せます。多くのコミュニティでは、新しいモニュメントの追加、地形レイアウトの変更、特別イベント、または完全に再設計された環境を導入するためにカスタムマップを利用しています。

これらのマップは通常、**RustEdit**などのツールで作成され、**Lone Design**や**Codefling**、コミュニティのマップリポジトリなどのプラットフォームを通じて配布されます。マップファイルやダウンロードリンクを入手したら、サーバー設定で直接設定可能です。

## 準備

カスタムマップを設定する前に、有効な**マップダウンロードURL**が必要です。カスタムマップは通常、マップ配布プラットフォームにホストされているか、マップ作成者から提供されます。

URLは通常、`.map`ファイルやダウンロード可能なマップパッケージを指します。Rustサーバーが起動時にダウンロードできるよう、リンクは公開されている必要があります。例：

```
https://example-map-host.com/maps/custommap.map
```



## 設定方法

Rustサーバーでカスタムマップを設定するには、**ゲームサーバー管理パネル**を開き、**設定**に進みます。

**Level URL**という設定項目を探してください。この設定はサーバーが起動時にカスタムRustマップをダウンロードして読み込むためのものです。カスタムマップのURLを**Level URL**欄に貼り付けます。例：

```
https://example-map-host.com/maps/custommap.map
```

URLを入力したら、パネルで設定を保存してください。サーバー起動時にRustは指定されたURLからマップを自動でダウンロードし、プロシージャルマップの代わりに読み込みます。

もしカスタムマップに特定のシード値やワールドサイズなど追加の設定パラメータが必要な場合は、マップ作成者から提供されることが多いので、それらもサーバー設定に追加してください。

設定を保存したら、**サーバーを再起動**してカスタムマップのダウンロードと読み込みを完了させましょう。



## 注意点

カスタムマップは、使用しているRustサーバーのバージョンと互換性がある必要があります。マップファイルが古いか破損していると、サーバーが起動に失敗したり、プロシージャルマップに戻ったりすることがあります。

大きなカスタムマップは、サーバーの起動時間が長くなる場合があります。これはサーバーがマップをダウンロードして準備するのに時間がかかるためです。

マップ切り替え後にプレイヤーが接続できない場合は、マップURLが正しくインターネットからアクセス可能かどうかを必ず確認してください。
