## はじめに

Subnautica 2のゲームサーバーで起こる問題は、起動失敗やクラッシュからラグ、接続トラブル、ゲーム内パフォーマンスの低下まで多岐にわたります。このガイドでは、Subnautica 2ゲームサーバーのよくある問題を特定し、実用的な解決策を適用してゲームをオンラインでスムーズに稼働させる方法を学びます。

:::info アーリーアクセスの変更点
Subnautica 2のサーバーソフトウェアがまだ頻繁に更新されている場合、動作や設定ファイル、利用可能な機能がバージョン間で異なることがあります。サーバーに違いを感じたら、必ずZAP-Hostingのウェブインターフェースで現在インストールされているバージョンと比較してください。
:::

## 問題と解決策

サーバーの問題は単一の原因だけで起こるとは限りません。多くの場合、古いファイル、不適切な設定、リソース不足、一時的なネットワーク状況が絡んでいます。以下のセクションでは、よくある問題を段階的にトラブルシュートする方法を紹介します。

### 一般的な問題

一般的な問題は、基本的なサーバーの稼働状況、表示、起動動作に影響します。これらは、より高度な原因を調査する前に最初に確認すべき項目です。

<details>
  <summary>ゲーム内でサーバーが見えない</summary>

Subnautica 2サーバーがゲーム内で見えない場合、サーバーが正しく起動していないか、初期化処理が完了していない可能性があります。無効な設定値、不完全なインストールファイル、起動エラーが原因となることがあります。

まず以下を確認してください：

- ZAP-Hostingのウェブインターフェースでサーバーが稼働中と表示されているか
- サーバーコンソールに起動エラーがないか
- ログファイルにファイル欠損や初期化失敗の記録がないか
- ゲーム内サーバーブラウザで正しい検索フィルターを使っているか
- 起動直後はサーバーが公開されるまで数分かかることがあるため少し待つ

それでもサーバーが表示されない場合は、一度再起動して起動プロセスの最初からコンソールを監視してください。

</details>

<details>
  <summary>サーバーが起動しない</summary>

サーバーが全く起動しない場合、破損したファイル、不完全なアップデート、無効な設定が原因であることが多いです。

以下の点を確認してください：

| チェック項目 | 理由 |
| --- | --- |
| サーバーコンソールの出力 | 起動エラーの詳細が表示される場合がある |
| 最近の設定変更 | 1つの無効な値で起動が阻害されることがある |
| 最近のファイルアップロードや変更 | 手動変更したファイルが不完全または非互換の可能性 |
| 現在のゲームバージョン | サーバーファイルの不一致は即時起動失敗を招く |

設定ファイルを最近変更した場合は、最後の変更を元に戻して再試行してください。アップデートや手動ファイル変更後に問題が発生した場合は、バックアップから復元すると時間を節約できます。

:::tip 大きな変更前にバックアップを活用
重要なファイル編集やサーバー更新前にバックアップを作成し、新たな問題が発生した場合にすぐに元に戻せるようにしましょう。
:::

</details>

<details>
  <summary>サーバーパスワードやアクセス設定が機能しない</summary>

サーバーがオンライン表示されているのにプレイヤーが参加できない場合、アクセス関連の設定をよく確認してください。パスワードの誤入力、古いクライアント情報の保存、公開・非公開設定の不一致がアクセスを妨げることがあります。

以下を確認しましょう：

- プレイヤーが入力するパスワードが現在のサーバーパスワードと完全に一致しているか
- アクセス設定変更後にサーバーを再起動したか
- 古いクライアント側の接続情報が再利用されていないか
- 現在のサーバーバージョンでプライベートやホワイトリスト設定が意図通りに構成されているか

設定が原因か不明な場合は、一時的に設定を簡素化して最小限の制限でテストしてください。

</details>

### クラッシュ問題

予期しないクラッシュはゲームプレイを中断し、繰り返すとデータ損失のリスクもあります。主な原因は古いサーバーファイル、非互換のMOD、リソース過負荷です。

<details>
  <summary>サーバーを最新状態に保つ</summary>

最新のSubnautica 2サーバーバージョンを使用することは、安定性、互換性、セキュリティのために重要です。古いバージョンは既に修正されたバグを含むか、現在のゲームクライアントと正しく動作しない可能性があります。

古いサーバーは以下の問題を引き起こします：

- 起動失敗
- ランダムクラッシュ
- バージョン不一致エラー
- 不安定なゲームプレイ挙動

![](https://screensaver01.zap-hosting.com/index.php/s/SCREENSHOT_PLACEHOLDER/preview)

ゲームアップデート後にクラッシュが増えた場合は、まずウェブインターフェースで新しいサーバーバージョンがあるか確認してください。更新後はサーバーを再起動して再度テストしましょう。

</details>

<details>
  <summary>不具合や非互換のMOD・追加コンテンツ</summary>

Subnautica 2がMODや追加のサーバー側コンテンツをサポートしている場合、これらがクラッシュの原因になることがあります。非互換または古いコンテンツは部分的に読み込まれても、ゲームプレイ中にエラーを引き起こすことがあります。

典型的な症状は：

- 起動後すぐにサーバーがクラッシュする
- プレイヤー参加時にクラッシュする
- ワールド読み込み中にフリーズする
- 新しいコンテンツ追加後にエラーが出る

原因を特定するには以下の手順を試してください：

1. サーバーを停止する
2. 最近追加したコンテンツを削除または無効化する
3. 最小構成でサーバーを起動する
4. 安定性をテストする
5. コンテンツを1つずつ再有効化する

この方法で問題のファイルやパッケージを特定できます。

:::caution バージョン互換性の重要性
現在のSubnautica 2サーバーバージョンと互換性が確認されたMODや拡張のみを使用してください。1つでも古いコンポーネントがあると繰り返しクラッシュする可能性があります。
:::

</details>

<details>
  <summary>破損したワールドやセーブデータ</summary>

ワールド読み込み中やプレイヤー接続直後にクラッシュする場合、セーブデータが破損している可能性があります。強制終了、中断された書き込み、繰り返すクラッシュが原因です。

考えられる兆候は：

- 毎回同じ読み込み段階でクラッシュする
- 新しいワールドは正常に始まるが既存ワールドは問題がある
- コンソールにセーブやワールド読み込みエラーが表示される

セーブ破損が疑われる場合は、バックアップや新規ワールドでテストしてください。新規ワールドで問題がなければ既存のセーブデータに問題がある可能性が高いです。

:::danger データ損失のリスク
既存のワールドファイルをすぐに削除しないでください。まずバックアップを作成し、必要に応じて後で復旧を試みられるようにしましょう。
:::

</details>

### パフォーマンス問題

パフォーマンスの問題は、ラグ、遅延動作、ワールドの読み込み遅延、不安定なゲームプレイとして現れます。設定ミス、ハードウェアリソース不足、重い追加コンテンツが主な原因です。

<details>
  <summary>不適切なサーバー設定</summary>

誤った設定値はCPU、メモリ、ストレージの負荷を増やし、全体のパフォーマンスを低下させます。Subnautica 2サーバーが遅い、または不安定に感じる場合は設定を見直しましょう。

特に以下を確認してください：

- プレイヤー関連の制限が契約プランに見合っているか
- ワールド関連の設定が必要以上に負荷をかけていないか
- 実験的またはデフォルト以外の設定を最近変更していないか
- 変更後にサーバーを再起動したか

設定はZAP-Hostingのウェブインターフェースの**設定**や**Configs**内のファイルで管理できます。

</details>

<details>
  <summary>サーバーパッケージがプロジェクト要件に合っていない</summary>

プロジェクトに対してサーバーが小さすぎると、カクつき、長い読み込み時間、不安定さが発生します。プレイヤー数増加、追加コンテンツ使用、大規模で活発なワールドで特に顕著です。

ZAP-Hostingでは注文時に推奨構成を案内し、適切なリソース選択をサポートしています。

![](https://screensaver01.zap-hosting.com/index.php/s/SCREENSHOT_PLACEHOLDER/preview)

以下の表は一般的なトラブルシュートの参考です：

| 症状 | 考えられる原因 | 推奨アクション |
| --- | --- | --- |
| 頻繁なラグスパイク | CPUリソース不足 | 負荷軽減またはプランアップグレード |
| 読み込み遅延 | ストレージやメモリのボトルネック | 使用状況見直しとリソース増強検討 |
| プレイヤー負荷でクラッシュ | 全体的な容量不足 | 負荷軽減またはサーバースケールアップ |
| 空き時は良好、プレイヤー多いと悪化 | アクティブ利用に対してパッケージが小さい | 予想プレイヤー数に応じてアップグレード |

:::note 最低推奨値について
推奨パッケージサイズはあくまで出発点です。大きなワールド、多数のプレイヤー、追加コンテンツを使う場合は初めからより多くのリソースが必要になることがあります。
:::

</details>

<details>
  <summary>MODや追加コンテンツによる高リソース使用</summary>

追加コンテンツがクラッシュを起こさなくても、パフォーマンス問題を引き起こすことがあります。最適化されていないコンテンツはCPU使用率増加、過剰なメモリ消費、繰り返すバックグラウンド処理を招きます。

トラブルシュートには、追加コンテンツあり・なしでのパフォーマンス比較が有効です。最小構成で正常なら、問題が再現するまで徐々にコンテンツを戻していきます。

特に以下の場合に役立ちます：

- 新しいコンテンツ導入後にラグが始まった
- 時間経過でサーバーが不安定になる
- 特定のエリアや状況でのみパフォーマンス低下
- 設定変更なしに使用率が増加した

</details>

<details>
  <summary>大規模ワールドや長期間稼働による遅延</summary>

サーバーワールドが大きくなり、プレイヤー活動が蓄積するとパフォーマンスが低下します。長期間稼働しているサーバーで大量のセーブデータがある場合に一般的です。

開始時は良好だったが後で悪化した場合、以下を検討してください：

- 新しいバックアップでのパフォーマンステスト
- 同時にアクティブなプレイヤー数の確認
- 不要な追加コンテンツの削除
- 問題が特定のワールドだけに起きているか確認

新規テストワールドで大幅に改善するなら、ワールドサイズやセーブの複雑さが原因の可能性があります。

</details>

### ネットワーク問題

ネットワーク関連の問題は、プレイヤーの接続やサーバーとのやり取りに影響します。高Ping、ラバーバンディング、切断、完全な接続不能として現れることがあります。

<details>
  <summary>Pingの急上昇、ラグ、ネットワーク遅延</summary>

Pingの急上昇や遅延は、サーバー負荷とルーティング状況の組み合わせで起こります。ゲーム自体はオンラインでも、過負荷のハードウェアや地理的距離が悪影響を与えます。

主な原因は：

- サーバー所在地がプレイヤーベースから遠い
- サーバーが高いCPUやメモリ負荷にある
- 多数のプレイヤーが同時接続している
- バックグラウンドタスクや他のプロセスがリソースを消費している
- プレイヤーとサーバー間で一時的なルーティングやパケットロスが発生している

ネットワーク品質改善には以下を確認してください：

| チェック項目 | 効果 |
| --- | --- |
| プレイヤーに近いサーバー所在地を選ぶ | 平均レイテンシ低減 |
| 不要なサーバー負荷を減らす | ゲーム中のラグスパイク減少 |
| サーバーを最新状態に保つ | 安定性と互換性向上 |
| 時間帯を変えてテストする | 一時的なルーティング問題の特定に役立つ |

問題が特定のプレイヤーだけなら、そのプレイヤーの接続問題の可能性が高いです。全員に影響がある場合はサーバー側や経路の問題が疑われます。

</details>

<details>
  <summary>プレイヤーがランダムに切断される</summary>

ランダムな切断は、不安定なルーティング、過負荷のサーバーリソース、一時的なソフトウェア不安定性が原因です。プレイヤーがゲーム内で明確な理由なく切断される場合、切断タイミングとコンソールメッセージやパフォーマンスの急上昇を比較してください。

確認すべき点：

- ピーク負荷時に切断が起きているか
- サーバーが短時間再起動やフリーズしていないか
- 全プレイヤーが同時に切断されるか、個別か
- 最近のアップデートや設定変更後に問題が始まったか

全員同時切断ならサーバーやネットワーク経路の問題が多いです。特定の1人だけなら、そのプレイヤー側の問題の可能性が高いです。

</details>

<details>
  <summary>DDoS攻撃やその他のネットワーク脅威</summary>

ゲームサーバーは時折、分散型サービス拒否（DDoS）攻撃など悪意あるトラフィックの標的になります。これによりラグ、接続失敗、一時的なダウンタイムが発生します。

ZAP-HostingはホスティングサービスにDDoS保護を提供し、一般的かつ高度な攻撃パターンの影響を軽減します。ただし、Subnautica 2サーバーが攻撃を受けている疑いがある場合は、以下の情報を添えてサポートに連絡してください：

- 問題が始まった日時
- すべてのプレイヤーに影響があるか
- サーバーが完全に到達不能になるか
- 問題が特定の時間帯に繰り返されるか

:::info 組み込み保護
DDoS対策はインフラ側で処理されます。悪意あるトラフィックが疑われる場合、通常はまずゲームサーバー設定を変更する必要はありません。代わりに症状を集め、サポートに連絡してください。
:::

</details>

### ゲームプレイ関連のサーバー問題

一部の問題は直接的なクラッシュやネットワーク障害ではありませんが、プレイヤーにとってゲームが正しく動作するかに影響します。

<details>
  <summary>クライアントとサーバーのバージョン不一致</summary>

ゲーム更新後にプレイヤーが参加できない場合、サーバーとゲームクライアントが異なるバージョンで動作している可能性があります。これはパッチ直後に特に多く見られます。

解決方法：

1. サーバーが更新されていることを確認する
2. 更新後にサーバーを再起動する
3. プレイヤーがゲームクライアントを更新していることを確認する
4. 接続を再度テストする

一部のプレイヤーだけが参加できる場合は、まずゲームバージョンとサーバーバージョンを比較してください。

</details>

<details>
  <summary>設定変更後に変更が反映されない</summary>

設定値を変更したのにゲーム内で違いが見えない場合、サーバーがキャッシュされたまたは変更されていないランタイムデータを使用しているか、編集したファイルがアクティブなものではない可能性があります。

以下を確認してください：

- 正しいファイルを編集したか
- 値が正常に保存されたか
- その後サーバーが完全に再起動されたか
- 変更した設定がインストール済みバージョンでサポートされているか

現在のSubnautica 2バージョンで正確な設定動作が不明な場合は、推測を避け、変更を1つずつテストして何が適用されるかを明確に特定してください。

</details>

## 予防措置

問題を事前に防ぐ方が、プレイヤーに影響した後に修正するよりも通常は簡単です。いくつかの定期的なメンテナンス手順で、安定性を大幅に向上できます。

### 推奨メンテナンスチェックリスト

将来の問題の可能性を減らすために、次のチェックリストを使用してください：

| タスク | 理由 |
| --- | --- |
| 定期的にバックアップを作成する | ワールドと設定をデータ損失から保護する |
| サーバーを慎重に更新する | バグとバージョン不一致の問題を減らす |
| 変更を1つずつテストする | 問題が発生した場合のトラブルシューティングを容易にする |
| 変更後のパフォーマンスを監視する | CPUやメモリ使用量の増加を早期に発見する |
| 必要なコンテンツのみをインストールする | 互換性とパフォーマンスのリスクを減らす |

### バックアップを定期的に使用する

定期的なバックアップは多くの時間を節約し、永続的なデータ損失を防ぎます。設定変更、更新、またはワールドの問題が障害を引き起こした場合、すべてを手動で再構築する代わりに、既知の動作状態を復元できます。

[バックアップソリューション](gameserver-backups.md)を使用して、ゲームサーバーの手動またはスケジュールされたバックアップを作成できます。

:::tip リスクのある変更の前にバックアップ
サーバーの更新、重要なファイルの編集、新しいModやコンテンツの追加の前にバックアップを作成してください。新しい問題が発生した場合の安全なロールバックポイントになります。
:::
