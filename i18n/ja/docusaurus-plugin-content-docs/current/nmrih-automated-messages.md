## はじめに

自動サーバーメッセージは、ゲーム内でプレイヤーに重要な情報を直接伝えるシンプルかつ効果的な方法です。サーバールールの表示、イベント告知、役立つヒントの共有、定期的なリマインダーなど、管理者が手動で入力しなくても自動で配信できます。

**スケジュールタスク**を使うことで、**No More Room In Hell**のゲーム内チャットにあらかじめ設定したコマンドを定期的に実行し、自動的にメッセージを送信できます。

## 自動メッセージの仕組み

自動メッセージは、指定した時間や間隔でサーバーコマンドを実行するタスクをスケジューリングして作成します。このコマンドはチャットメッセージの送信やブロードキャスト、サーバー通知に使えます。

設定が完了すると、タスクはバックグラウンドで自動的に動作し、接続中の全プレイヤーにメッセージを表示します。これにより、スタッフが常にオンラインでなくても安定した情報発信が可能です。



## 自動メッセージの作成方法

自動サーバーメッセージを作成するには、ゲームサーバーのダッシュボードを開き、**スケジュールタスク**のセクションに移動します。新しいスケジュールタスクを作成し、**コマンド実行**のオプションを選択してください。コマンド欄にゲームサーバーが対応しているメッセージ送信コマンドを入力します。

```
<message command> <your message text>
```

`<message command>`はメッセージ送信用のゲームコマンドや拡張機能のコマンドに置き換え、`<your message text>`にはプレイヤーに見せたいメッセージを入力してください。

その後、メッセージを送信する頻度（数分ごと、毎時、特定の時間など）を設定できます。スケジュールタスクの詳しい作成方法は、[スケジュールタスク](gameserver-scheduled-tasks.md)ガイドを参照してください。

## メッセージ送信間隔のカスタマイズ

スケジュールタスクは、メッセージの表示タイミングや頻度を自由にコントロールできます。よくある使い方は：

- 数分ごとにサーバールールを表示
- 固定時間にイベント告知を配信
- 新規・復帰プレイヤー向けにゲームプレイのヒントを送信
- Discordやウェブサイトのリンクをリマインド

間隔はサーバーを再起動せずにいつでも変更可能なので、サーバーの成長に合わせて柔軟に調整できます。



## 注意点

自動サーバーメッセージの機能は、ゲーム本体や利用可能なコマンドの仕様に依存します。基本的なスケジュールメッセージは標準のサーバーコマンドで実装可能ですが、フォーマットや条件分岐、高度なカスタマイズには制限がある場合があります。

もしゲームがプラグインやMOD、拡張機能をサポートしているなら、より柔軟で多機能なメッセージシステムを利用するのもおすすめです。そうした専用のメッセージングシステムは、標準コマンドよりも豊富な機能を持ち、複雑または高度にカスタマイズされたサーバー運営に適しています。



## まとめ

自動サーバーメッセージは、手間をかけずにプレイヤーに情報を届けてエンゲージメントを高める信頼できる方法です。スケジュールタスクでメッセージコマンドを実行することで、サーバー全体で一貫したコミュニケーションを実現できます。

一度設定すれば、自動メッセージは独立して動作し、ゲーム固有のメッセージコマンドやテキストを調整するだけで簡単に再利用可能です。
