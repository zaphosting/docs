## はじめに

**Rust**では、サーバーのワイプはゲーム世界の一部をリセットして、すべてのプレイヤーに新鮮なスタートを提供します。時間が経つにつれて、Rustのマップは大規模な拠点や蓄積された資源、確立されたプレイヤーグループでいっぱいになります。これがサーバーのバランスやパフォーマンスに影響を与えることがあります。

サーバーワイプは既存の進行状況をリセットし、プレイヤーがゼロから再スタートできるクリーンな環境を復元します。多くのRustコミュニティは、公平でエキサイティングなゲームプレイを維持するために、特に新しいアップデートやマップがリリースされた際に定期的にワイプを実施しています。

ワイプの種類によっては、**マップ**、**プレイヤーブループリント**、または**両方**がリセットされます。

## サーバーワイプで何が起きるの？

Rustのサーバーワイプは、保存されたワールドやプレイヤーの進行データを削除します。これには以下が含まれます：

- プレイヤーが建てた建造物や拠点  
- プレイヤーのインベントリや保管アイテム  
- マップの探索データ  
- 習得済みのブループリント（ワイプの種類による）

ワイプ後は、サーバーは新しい世界のように振る舞い、プレイヤーは資源を集め、拠点を再建し、進行を再度学ぶ必要があります。



## Rustのワイプの種類

Rustでは、リセットしたい内容に応じていくつかのワイプタイプが用意されています。以下でそれぞれのワイプタイプを説明します。

### マップワイプ

**マップワイプ**は、ワールドのセーブファイル全体を削除します。すべての拠点、建造物、マップの進行状況が消えます。サーバー再起動時に新しいマップが生成またはダウンロードされます。ブループリントの進行は、ブループリントワイプを行わない限り変更されません。

### ブループリントワイプ

**ブループリントワイプ**は、習得済みのクラフトブループリントをすべてリセットします。プレイヤーはアイテムを作成する前に再度リサーチが必要になります。このワイプは進行状況のみをリセットし、マップ自体はリセットしません。

### フルワイプ

**フルワイプ**は、マップとすべてのプレイヤーブループリントの両方をリセットします。これにより完全に新しいサーバー環境が作られ、シーズンリセットや大規模アップデート時によく使われます。



## なぜサーバーワイプが重要なのか

サーバーワイプはRustサーバーの健全な環境維持に役立ちます。時間が経つと、サーバーには大量の建造物、ストレージコンテナ、ワールドエンティティが蓄積されます。これがサーバーのパフォーマンスやゲームバランスに悪影響を及ぼすことがあります。定期的なワイプには以下のようなメリットがあります：

- サーバーのパフォーマンスを回復
- すべてのプレイヤーに新鮮なスタートを提供
- 強力なクランがマップを独占し続けるのを防止
- 新しいマップやマップシードの導入を可能にする

多くのRustコミュニティは、サーバースタイルに応じて週次、隔週、または月次でスケジュールされたワイプを実施しています。



## サーバーワイプの実施方法

Rustサーバーでワイプを行うには、まず[FTPアクセス](gameserver-ftpaccess.md)でサーバーファイルにアクセスします。次にRustサーバーディレクトリ内のセーブファイルを探します。通常、以下のファイルが含まれます：

```
proceduralmap.*
player.blueprints.*
player.deaths.*
player.identities.*
player.states.*
```
削除するファイルによってワイプの種類が決まります。`proceduralmap.*`ファイルを削除すると**マップワイプ**が実行され、既存のワールドが削除されます。`player.blueprints.*`ファイルを削除すると**ブループリントワイプ**が実行されます。

![img](https://screensaver01.zap-hosting.com/index.php/s/QdeoFWbaaQz7Yaf/preview)

両方を削除すると**フルワイプ**になります。必要なファイルを削除したらサーバーを再起動してください。サーバーは自動的に新しいワールドを生成し、プレイヤーは新たにスタートできます。
