## はじめに

リモートデスクトッププロトコル（RDP）はデフォルトでポート3389を使用します。このポートは広く知られており、自動スキャンの標的になりやすいです。RDPポートを変更することで、不正な接続試行を減らし、基本的なセキュリティリスクを軽減できます。

## レジストリでRDPポートを変更する

サーバーにローカルまたは既存のリモートデスクトップ接続でログインします。**Win + R**で「ファイル名を指定して実行」を開き、`regedit`と入力してレジストリエディターを起動します。以下のレジストリパスに移動してください：

```
HKEY_LOCAL_MACHINE
 └─ SYSTEM
    └─ CurrentControlSet
       └─ Control
          └─ Terminal Server
             └─ WinStations
                └─ RDP-Tcp
```

**PortNumber**という名前のレジストリエントリを探します。値の形式を**10進数**に変更し、希望の新しいポート番号を入力してください。1024以上で、他のサービスが使用していないポートを推奨します。変更を適用したら、レジストリエディターを閉じます。

## Windowsファイアウォールのルールを調整する

新しいRDPポートをWindowsファイアウォールで許可する必要があります。**Windows Defender ファイアウォール 高度なセキュリティ**を開き、受信の規則セクションに移動します。

既存のリモートデスクトップのルールを編集するか、新しい受信ルールを作成して、新しく設定したポートのTCPトラフィックを許可してください。接続を許可し、必要なネットワークプロファイルに適用されていることを確認しましょう。

## 変更を反映するためにサービスを再起動する

ポート変更はリモートデスクトップサービスの再起動またはシステムの再起動後に有効になります。設定を確実に反映させるため、サーバーの完全再起動を推奨します。

## 新しいポートで接続する

新しいリモートデスクトップ接続を確立する際は、IPアドレスまたはホスト名の後にポート番号を明示的に指定する必要があります。例：

```
203.0.113.10:3390
```

## まとめ

WindowsのRDPポートを変更することで、自動スキャンや不正な接続試行のリスクを減らせます。手順はWindowsレジストリの更新、新ポートのファイアウォール許可、そしてシステムまたはサービスの再起動です。

完了後はすぐにアクセス確認を行い、設定したポート番号を記録しておくことをおすすめします。ご不明点やサポートが必要な場合は、いつでもお気軽に当社のサポートチームへお問い合わせください！🙂
