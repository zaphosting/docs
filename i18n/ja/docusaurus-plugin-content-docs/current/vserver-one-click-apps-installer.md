## はじめに

ワンクリックアプリインストーラーを使えば、豊富なアプリカタログをブラウズして、VPSのウェブインターフェースから直接アプリを展開できます。このガイドでは、サーバーの準備、適切なインストールタイプの選択、アプリのインストールと管理、必要に応じた高度な設定やトラブルシューティングツールの使い方を学べます。

## 準備

ワンクリックアプリインストーラーを使う前に、VPSが最低要件を満たし、対応OSを使用している必要があります。

:::info 最低要件
アプリ機能を利用するには、VPSがオンラインで、プラットフォームサービスとインストールしたアプリを安定して動かせる十分なリソースが設定されている必要があります。
:::

| 要件 | 最低 | 推奨 |
| --- | --- | --- |
| サーバーステータス | オンライン | オンライン |
| ストレージ | 30 GB | 50 GB |
| RAM | 2 GB | 4 GB |
| CPU | 2コア | 4コア |
| OS | Debian または Ubuntu LTS | Debian 13 - EN - 64bit |

:::caution 新規OSインストール推奨
既存のサービスや使用中のポート、Docker設定、カスタム構成との競合を避けるため、アプリ機能は新規OSインストール上でセットアップすることを強く推奨します。
:::

続行前にVPSを再インストールする必要がある場合は、VPS管理画面から対応するDebianまたはUbuntu LTSの新規イメージを使ってください。

## ワンクリックアプリインストーラーへのアクセス

ZAP-HostingのウェブインターフェースでVPSを開き、ワンクリックアプリセクションを探します。ここから初期セットアップを開始し、後でサーバーにインストールしたすべてのアプリを管理できます。

初回起動時は、プラットフォームのインストール方法を選択する必要があります。

## インストールタイプの選択

フルインストールかノードのみインストールかを選べます。どちらを選ぶかは、このVPSが管理パネルとアプリの両方をホストするか、他のパネルのリンクノードとしてのみ使うかによります。

### フルインストール

管理パネルとノードとしてサーバーをセットアップしたい場合は、**フルインストール**を選択してください。初めての方や標準的な展開にはこちらが推奨です。

この方法でできること：

- アプリカタログからアプリを展開
- プロジェクトや環境を管理
- すべてのアプリを一元管理
- 統合されたウェブインターフェース管理ツールを利用
- 必要に応じて高度なパネルにアクセス可能

![img](https://screensaver01.zap-hosting.com/index.php/s/nqGL3PFPicTArY9/preview)

:::tip ほとんどのユーザーに推奨
アプリ機能を初めて使うなら、フルインストールを選びましょう。最も簡単にセットアップでき、ZAP-Hostingのウェブインターフェースから完全に管理できます。
:::

### ノードのみインストール

このVPSをノードとしてのみ使いたい場合は、**ノードのみインストール**を選択します。別のVPSや外部パネルにリンク可能です。

このオプションは以下の用途に便利です：

- 複数VPSにまたがるスケールアウト
- 管理を一つのパネルに集中
- 既存の展開にこのサーバーをリンク
- マルチノード構成の拡張

:::note 上級者向け展開オプション
ノードのみインストールは、すでに別のパネルがあるか、外部環境に接続したいユーザー向けです。
:::

## アプリ機能のインストール

インストールタイプを選んだら、ウェブインターフェースからセットアップを開始します。初期インストールは通常数分で完了します。

この間、ワンクリックアプリ機能に必要なプラットフォームサービスがVPSに展開されます。完了すると、ウェブインターフェースにアプリカタログと管理セクションが表示されます。

:::info インストールの挙動
インストールの進行状況はVPSの性能、OSの状態、ネットワーク環境によって異なる場合があります。セットアップがうまくいかない場合は、後述のトラブルシューティングを参照してください。
:::

## アプリカタログの閲覧

インストール完了後、アプリカタログを開いて利用可能なアプリをブラウズできます。300以上の即展開可能なアプリがあり、フィルター機能で目的のアプリを素早く見つけられます。

![](https://screensaver01.zap-hosting.com/index.php/s/fBwcCiNJTZooqnB/preview)

### アプリの検索とフィルター

カタログ内ではアプリ名で検索したり、カテゴリで絞り込みが可能です。これにより、アプリストア風のインターフェースで目的の展開先を簡単に見つけられます。

よく使う操作：

- アプリ名で検索
- カテゴリでフィルター
- 利用可能なアプリの確認
- 展開したいアプリの選択

多くのアプリを管理したり、複数サービスを比較検討する際に便利です。

### アプリのインストール開始

カタログからアプリを選んだら、ウェブインターフェースでインストールを開始します。アプリによっては以下のような値を設定する必要があります：

- `[your_project_name]`
- `[your_environment_name]`
- `[your_domain]`
- `[your_app_name]`

これらは展開に合わせた固有の値に置き換えてください。例えば、`[your_domain]`はアプリで使いたいドメインに差し替えます。

:::tip プロジェクトと環境の活用
複数のアプリをホストしたり、バージョンのテスト、運用とステージングの分離にプロジェクトと環境を使うと管理が楽になります。
:::

## インストール済みアプリの管理

展開後、インストール済みアプリはウェブインターフェースの管理セクションに一覧表示されます。ここから、基盤のコンテナを直接操作せずにサービスの監視や制御が可能です。

![](https://screensaver01.zap-hosting.com/index.php/s/WTMSqkJ5zPKWjmn/preview)

### 利用可能な管理操作

インストール済みアプリセクションでは、一般的なライフサイクル操作をウェブインターフェースから直接実行できます。

| 操作 | 内容 |
| --- | --- |
| 起動 | 停止中のアプリを起動 |
| 停止 | 実行中のアプリを停止 |
| ログ確認 | トラブルシューティング用にアプリログを開く |
| 再インストール | アプリを再展開 |

これらのツールで、コマンドラインを使わずに簡単にアプリを管理できます。

### プロジェクトと環境の整理

プロジェクトと環境を使うと展開を明確に構造化できます。例えば：

- 個人用と本番用アプリの分離
- テスト環境とライブ環境の区別
- 複数顧客やチームの展開管理
- 同じVPS上の異なるスタック管理

アプリ数が増えたり複数ノードを管理する際に特に重要です。

## ドメイン、クラウドアクセス、SSL

ワンクリックアプリ機能はカスタムドメインとZAP Cloud連携をサポート。アプリを安全に公開し、SSL証明書の発行も可能です。

:::info ドメインとSSL対応
カスタムドメインを使い、プラットフォーム設定で対応していればSSL証明書の発行も利用できます。
:::

ドメインを割り当てる際は、DNSレコードが`[your_server_ip]`を指していることを必ず確認してください。正しく設定されていないと、アプリがインターネットからアクセスできません。

| 値 | 説明 |
| --- | --- |
| `[your_domain]` | アプリに割り当てるドメインまたはサブドメイン |
| `[your_server_ip]` | VPSのパブリックIPアドレス |

:::caution DNSは必ずVPSを指すこと
`[your_domain]`のDNSが`[your_server_ip]`を指していないと、ドメイン認証やアプリアクセスに失敗します。
:::

## 高度な設定の利用

上級者は高度な設定セクションから外部パネルにアクセスして、さらに細かいカスタマイズや展開制御が可能です。

![](https://screensaver01.zap-hosting.com/index.php/s/BHnZWBGAzgo5myp/preview)

標準のウェブインターフェース以上の機能に直接アクセスしたい場合に便利です。

### 重要なサポート制限

:::caution 高度なパネル利用時のサポート範囲
外部Coolifyパネルの利用は自由ですが、サポートはZAP-Hostingのウェブインターフェースに限定されます。ウェブインターフェース外での変更にはサポートが提供されない場合があります。
:::

### 高度なパネルを使うタイミング

以下のような場合に高度なパネルを使うと良いでしょう：

- 詳細な展開カスタマイズが必要
- ノードリンクの直接操作
- 高度なアプリ設定
- 簡易インターフェースを超えたパネル管理

アクセスできなくなったり連携が止まった場合は、手動変更前に下記のトラブルシューティングを試してください。

## ノードのリンク

ノードのみインストールを選んだ場合、このVPSを別のVPSや外部パネルにリンクできます。これによりインフラをスケールし、複数ノードを一元管理可能です。

ノードリンクは以下の用途に便利です：

- 複数VPSにアプリを分散
- 管理を集中化
- 孤立しない拡張
- 外部パネル環境との接続

:::note 外部パネルリンク
ノードは別VPSや外部パネルにリンク可能。リンク方法は対象環境によって異なります。
:::

## 劣化ステータスのトラブルシューティング

ウェブインターフェースで劣化ステータスが表示されたら、プラットフォームがOne Click Apps機能を動かすインスタンスに接続できていない状態です。以下の手順を順に試してください。

### ステータス指標

トラブルシューティングポップアップには以下のような指標が表示されます：

| ステータス | 意味 |
| --- | --- |
| Coolifyステータス: アクティブ | パネルサービスが稼働中の可能性あり |
| APIステータス: アクセス可能 | APIに現在アクセス可能 |
| サービスステータス: オンライン | VPS自体はオンライン |

いずれかが正常でない場合は次の手順へ進みます。

### ステップ1: VPSがオンラインか確認

まずZAP-HostingのウェブインターフェースでVPSがオンラインか確認してください。オフラインならアプリプラットフォームは動作しません。

### ステップ2: ステータスの更新

**ステータス更新**を使って最新データでヘルスチェックを再実行します。短時間の通信障害ならこれで解決することがあります。

:::tip 一時的な接続問題
劣化状態は一時的な通信問題が原因のこともあります。ステータス更新が最速の初期対応です。
:::

### ステップ3: Dockerの再起動

劣化状態が続く場合は**Docker再起動**を実行してください。プラットフォームインスタンスが再起動されます。成功後は`1-2分`待ってから再度ステータスを更新しましょう。

:::caution サービス再起動の影響
Docker再起動中はアプリが一時的に停止する可能性があります。
:::

### ステップ4: 認証情報の更新

最後に**認証情報の更新**を行います。ステータスがオンラインでもウェブインターフェースの機能が動かない場合に特に有効です。

この操作は：

- データを削除しません
- rootユーザーの認証情報を更新
- ウェブインターフェースが使うAPIトークンを更新
- 成功時にパネル認証情報を自動更新

:::info 認証情報更新は安全
認証情報の更新はインストール済みアプリや展開データを削除しません。アクセスと通信に必要な認証情報だけをリフレッシュします。
:::

### サポートに連絡するタイミング

すべての手順を試しても問題が解決しない場合は、ZAP-Hostingのウェブサイトからサポートチケットを作成してください。

## ベストプラクティス

ワンクリックアプリインストーラーを安定して管理しやすくするため、以下の推奨事項を守りましょう：

| ベストプラクティス | 理由 |
| --- | --- |
| 新規OSインストールを使う | 既存サービスやポートとの競合を減らす |
| 推奨リソースを満たす | パフォーマンスと信頼性向上 |
| プロジェクトと環境で整理 | 展開管理を簡単に |
| ドメイン割り当て前にDNS設定 | アクセスやSSLの問題防止 |
| 高度な設定は慎重に使う | サポート対象外のカスタム変更を避ける |

:::tip 展開は整理して管理
多くのアプリをホストするなら、最初に`[your_project_name]`、`[your_environment_name]`、`[your_app_name]`の命名ルールを決めておくと管理が楽です。
:::
