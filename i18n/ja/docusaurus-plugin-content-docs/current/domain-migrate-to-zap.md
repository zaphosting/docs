## はじめに

ZAP-Hostingのサービスに興味がある初心者さん？それならここがピッタリ！まずはZAP-Hostingへようこそ！当社のサービスに興味を持っていただき、他社から既存のドメインをどうやって移管するか知りたい方に、このガイドではドメインを当社に無事に移管するための基本的なステップをお伝えします。

<InlineVoucher />

## メリット

どのプロバイダーが自分に合っているか、迷っているかもしれませんね。そんな決断は時間がかかることもあります。だからこそ、ZAP-Hostingのメリットをシェアして、あなたの決断をサポートしたいと思います。ZAP-Hostingは10年以上にわたりサービスを提供しており、サーバーのレンタル業界で確かな実績と経験を持つプロバイダーです。

当社の管理画面は、ドメイン管理を簡単かつ直感的に行えます。さらに、他のZAP製品用のサブドメインもワンクリックで作成可能です。

もし当社に決めたなら、既存のドメインをZAP-Hostingに移管する準備を始めましょう。このプロセスは3つのステップに分かれています：準備（Authコードの取得）、移管（ドメインの登録と移管）、そして移管後の設定調整で、すべてが正しく動作するようにします。

## 準備

すでに登録済みのドメインを移管するには、**Authコード**（認証コード）が必要です。これはドメインのパスワードのようなもの。ドメインを別のプロバイダーに移管する際に必須で、このコードがあることで、ドメイン移管が正当な所有者によってリクエストされたことを証明します。

一部のプロバイダーは管理画面で直接Authコードを提供しています。もしそうでなければ、現在のプロバイダーに移管希望を伝え、Authコードをリクエストしてください。

:::warning 有効期限について

Authコードには有効期限があります。有効期限が切れたら、新しいAuthコードを再度取得する必要があります。有効期限はドメインの種類によって異なります。
:::

## 移管

Authコードを用意できたら、ZAP-Hostingでドメインの注文と登録を進めましょう。当社の[**ドメイン注文ページ**](https://zap-hosting.com/en/shop/product/domain/)にアクセスし、まずは個人情報を入力、その後移管したいドメインを検索します。

![](https://screensaver01.zap-hosting.com/index.php/s/3dmY76dZscz9DPM/preview)

:::info
ドメインを移管する際は、注文時に1年間の延長が必要です。これはすべてのプロバイダーで共通のルールです。
:::

希望のドメインと利用可能なドメイン拡張子の一覧が表示されます。すでに他社で登録されているドメインは、矢印付きのグレーのボタンで識別できます。

![](https://screensaver01.zap-hosting.com/index.php/s/omnaMqXJgarxsqW/preview)

移管したいドメインを見つけたら矢印をクリック。すると、ZAP-Hostingアカウントへの移管を承認するための入力フォームが開きます。

![](https://screensaver01.zap-hosting.com/index.php/s/fXjwGCX7kFtPnTB/preview)

ここに、準備段階で前のプロバイダーから受け取った**Authコード**を入力してください。入力が完了したら、緑の**追加**ボタンで送信します。

:::danger 入力内容の確認を忘れずに

**Authコード**は正確かつ有効なものを入力してください。間違っていると注文処理やドメイン移管が遅れたり、失敗する可能性があります。
:::

これでドメイン移管が開始されました。移管にかかる時間はドメイン拡張子や前のプロバイダーによって異なりますが、通常は72時間以内に完了します。ただし、状況によってはもっと時間がかかることもあります。
