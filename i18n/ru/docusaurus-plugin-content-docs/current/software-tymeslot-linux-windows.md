## Introduction

Tymeslot — это open-source платформа для организации встреч и записи на прием, построенная на Elixir и Phoenix LiveView. В этом руководстве вы узнаете, что требуется для Tymeslot, как развернуть его с помощью Docker на Linux или Windows и какие настройки стоит проверить перед запуском в продакшн.

## Preparation

Перед началом убедитесь, что у вас есть система, способная надежно запускать Docker и предоставлять доступ к Tymeslot по сети.

### Требования

| Требование | Рекомендация |
| --- | --- |
| CPU | 1 vCPU или больше |
| RAM | Минимум 2 ГБ |
| Место на диске | Минимум 10 ГБ |
| Операционная система | Linux или Windows |
| Платформа контейнеров | [Docker](https://www.docker.com/) |
| Сетевой порт | `4000/tcp` |

### Что нужно подготовить

| Элемент | Зачем нужен |
| --- | --- |
| Сервер на Linux или система Windows | Для хостинга приложения |
| Установленный Docker | В репозитории Tymeslot есть файлы для Docker |
| Открытый сетевой доступ к порту `4000` или обратный прокси | Для доступа к веб-интерфейсу |
| Доменное имя, например `[your_domain]` | Рекомендуется для продакшн-использования |
| Сильный сгенерированный секрет | Требуется для безопасности сессий и приложения |

:::info Требование к установке Docker
Это руководство ориентировано на Docker, так как в репозитории Tymeslot есть файлы, связанные с Docker, такие как `Dockerfile.docker` и `docker-compose.yml`. Нативная установка с Elixir и Phoenix возможна, но точные шаги для продакшна из исходников не были полностью проверены.
:::

### Примечание для Windows

На Windows обычно требуется рабочая среда Docker, например Docker Desktop. Если вы используете Docker для Windows, убедитесь, что включена виртуализация и Docker Desktop запущен перед продолжением.

## Deployment Method

Для большинства пользователей Docker — самый простой способ развертывания, так как он изолирует зависимости и облегчает обновления.

### Почему рекомендуют Docker

| Преимущество | Объяснение |
| --- | --- |
| Проще настроить | Не нужно вручную устанавливать Elixir, Erlang и Phoenix |
| Чище обновления | Можно заменить docker-образы без перестройки хоста |
| Лучшая переносимость | Один и тот же контейнер работает на Linux и Windows |
| Проще откатиться | Можно вернуться к предыдущей версии образа при необходимости |

### Важное ограничение

:::caution Необходимо проверить имя Docker-образа
В примере использовался образ `youruser/tymeslot:latest` как заглушка, но это неофициальное имя. Перед развертыванием проверьте репозиторий Tymeslot, файл `README-Docker.md` или инструкции по контейнерам, чтобы подтвердить правильный источник образа.
:::

## Install Tymeslot with Docker

В этом разделе показан процесс развертывания с Docker с использованием заглушки для образа. Замените `[your_tymeslot_image]` на проверенное имя образа из официального проекта Tymeslot.

### Скачайте образ

```bash
docker pull [your_tymeslot_image]
```

### Сгенерируйте секретный ключ

В Linux можно создать сильный секрет так:

```bash
openssl rand -base64 64 | tr -d '\n'
```

Сохраните результат надежно и используйте как `[your_secret_key_base]`.

В Windows можно сгенерировать безопасное случайное значение другим проверенным способом, если `openssl` недоступен в вашей оболочке.

### Запустите контейнер

```bash
docker run -d \
  --name tymeslot \
  -p 4000:4000 \
  -e SECRET_KEY_BASE='[your_secret_key_base]' \
  -e PHX_HOST='[your_domain_or_ip]' \
  -v tymeslot_data:/app/data \
  -v tymeslot_pg:/var/lib/postgresql/data \
  [your_tymeslot_image]
```

### Описание параметров

| Параметр | Значение |
| --- | --- |
| `-d` | Запускает контейнер в фоновом режиме |
| `--name tymeslot` | Задает читаемое имя контейнера |
| `-p 4000:4000` | Пробрасывает порт `4000` хоста на порт `4000` контейнера |
| `-e SECRET_KEY_BASE=...` | Устанавливает секрет приложения |
| `-e PHX_HOST=...` | Устанавливает имя хоста для Phoenix |
| `-v tymeslot_data:/app/data` | Сохраняет данные приложения |
| `-v tymeslot_pg:/var/lib/postgresql/data` | Сохраняет данные PostgreSQL, если используется внутри контейнера |

:::caution Проверьте пути томов
Пути томов в примере взяты из черновика и не были полностью подтверждены в официальной документации. Перед использованием в продакшне проверьте официальные инструкции по Docker для Tymeslot.
:::

### Проверка статуса контейнера

```bash
docker ps
```

Для просмотра логов:

```bash
docker logs tymeslot
```

Если возникают проблемы при запуске, логи — первое место для проверки.

## Optional Docker Compose Deployment

Если предпочитаете управлять контейнерами через Compose, можно описать сервис в файле `docker-compose.yml`. Это часто удобнее, чем длинные команды `docker run`.

### Пример файла Compose

```yaml
services:
  tymeslot:
    image: [your_tymeslot_image]
    container_name: tymeslot
    ports:
      - "4000:4000"
    environment:
      SECRET_KEY_BASE: "[your_secret_key_base]"
      PHX_HOST: "[your_domain_or_ip]"
    volumes:
      - tymeslot_data:/app/data
      - tymeslot_pg:/var/lib/postgresql/data
    restart: unless-stopped

volumes:
  tymeslot_data:
  tymeslot_pg:
```

Запустите стек командой:

```bash
docker compose up -d
```

:::tip Управление через Compose
Compose упрощает обновления, перезапуски и резервное копирование, так как вся конфигурация контейнера хранится в одном файле. Особенно полезно, если вы управляете несколькими docker-образами на одном хосте.
:::

## Configure Tymeslot

После запуска контейнера проверьте основные параметры конфигурации.

### Основные переменные окружения

| Переменная | Назначение |
| --- | --- |
| `SECRET_KEY_BASE` | Защищает сессии и секреты приложения |
| `PHX_HOST` | Определяет публичное имя хоста или IP |

Используйте заполнители так:

- `[your_secret_key_base]` — ваш сгенерированный секрет
- `[your_domain_or_ip]` — публичное доменное имя или IP сервера, например `schedule.[your_domain]` или `[your_server_ip]`

### Настройка SMTP

В черновике указано, что SMTP можно настроить для email-уведомлений в файле `config/prod.exs`. Пример:

```elixir
config :tymeslot, Tymeslot.Mailer,
  adapter: Bamboo.SMTPAdapter,
  server: "smtp.example.com",
  port: 587,
  username: "your_email@example.com",
  password: "your_password",
  tls: :if_available,
  auth: :always
```

:::note Проверка настройки SMTP
Пример взят из черновика и отражает типичную конфигурацию Elixir mailer. Тем не менее, сравните с актуальными файлами конфигурации Tymeslot, так как настройки почты могут меняться между версиями.
:::

## Network and Access

После запуска Tymeslot убедитесь, что к нему можно подключиться из браузера.

### Доступ к веб-интерфейсу

Откройте в браузере URL:

```text
http://[your_server_ip]:4000
```

Если настроен обратный прокси и DNS, можно использовать:

```text
https://[your_domain]
```

### Справка по портам

| Порт | Протокол | Назначение |
| --- | --- | --- |
| `4000` | TCP | Стандартный порт веб-доступа из черновика |

### Особенности брандмауэра

Необходимо разрешить входящий трафик на `4000/tcp`, если доступ к Tymeslot идет напрямую.

:::caution Открытый доступ в интернет
Если вы открываете Tymeslot напрямую в интернет, настоятельно рекомендуется разместить его за обратным прокси с HTTPS. Это повысит безопасность и обеспечит удобный доступ по домену.
:::

## Reverse Proxy Recommendation

Обратный прокси рекомендуется, если вы хотите использовать доменное имя и TLS-сертификат.

### Зачем нужен обратный прокси

| Преимущество | Объяснение |
| --- | --- |
| Поддержка HTTPS | Шифрует трафик между пользователями и сервисом |
| Доступ по домену | Позволяет использовать `[your_domain]` вместо порта |
| Удобство управления сертификатами | Хорошо работает с Let's Encrypt и подобными |
| Чистый публичный доступ | Скрывает внутренние порты приложения |

### Что проверить

Точная конфигурация обратного прокси зависит от используемого веб-сервера — Nginx, Apache или другого. Поскольку в исходном материале не было полного примера, перед развертыванием проверьте требования к заголовкам и обработке websocket в официальной документации Tymeslot.

## Verify the Installation

После развертывания убедитесь, что приложение работает корректно.

### Базовые проверки

| Проверка | Ожидаемый результат |
| --- | --- |
| `docker ps` | Контейнер `tymeslot` запущен |
| `docker logs tymeslot` | Нет повторяющихся ошибок запуска или базы данных |
| Доступ из браузера | Интерфейс Tymeslot загружается |
| Тест порта | Порт `4000` отвечает на сервере |

### Частые проблемы

| Проблема | Возможная причина |
| --- | --- |
| Страница не загружается | Порт `4000` заблокирован или контейнер не запущен |
| Контейнер сразу выходит | Неверные переменные окружения или отсутствуют зависимости |
| Ошибки базы в логах | Ошибка инициализации базы или неправильный путь хранения |
| Неправильное поведение хоста | `PHX_HOST` задан неверно |

## Updating Tymeslot

При выходе новой версии обновляйте развертывание аккуратно.

### Процесс обновления Docker

1. Скачайте обновленный образ:
   ```bash
   docker pull [your_tymeslot_image]
   ```
2. Остановите текущий контейнер:
   ```bash
   docker stop tymeslot
   ```
3. Удалите старый контейнер:
   ```bash
   docker rm tymeslot
   ```
4. Запустите новый контейнер с той же конфигурацией.

Если используете Compose, обычно достаточно:

```bash
docker compose pull
docker compose up -d
```

:::tip Резервное копирование перед обновлением
Перед обновлением сделайте бэкап постоянных томов и базы данных. Это даст возможность откатиться, если новая версия вызовет проблемы с миграциями или совместимостью.
:::
