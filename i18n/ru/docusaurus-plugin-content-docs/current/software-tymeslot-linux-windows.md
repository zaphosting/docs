## Введение

Tymeslot — это платформа с открытым исходным кодом для планирования встреч и назначений, построенная на Elixir и Phoenix LiveView. В этом руководстве вы узнаете, что такое Tymeslot, что нужно для его запуска и как развернуть его на Linux или Windows с помощью Docker.

![](https://screensaver01.zap-hosting.com/index.php/s/SCREENSHOT_PLACEHOLDER/preview)

## Подготовка

Перед развертыванием Tymeslot убедитесь, что ваша система соответствует базовым требованиям и что Docker установлен.

### Требования

Следующие требования основаны на доступной информации о проекте и предоставленном черновике установки:

| Требование | Рекомендация |
|---|---|
| CPU | Минимум `1` виртуальное ядро |
| ОЗУ | Минимум `2 ГБ` |
| Место на диске | Минимум `10 ГБ` |
| Операционная система | Linux или Windows |
| Среда контейнеров | Docker |
| Сеть | Доступ к порту `4000` |

### Что проверено, а что нет

:::info Доступность исходников
Репозиторий Tymeslot подтверждает, что проект — это платформа для планирования встреч с открытым исходным кодом, построенная на Elixir и Phoenix LiveView, а также что в репозитории есть файлы, связанные с Docker, такие как `Dockerfile.docker` и `docker-compose.yml`.

Однако точные шаги для продакшн-развертывания, имя образа и все необходимые переменные окружения не полностью проверены на основе предоставленных исходных материалов. Поэтому в этом руководстве используются доступные черновые данные с явным указанием значений, которые вы должны проверить в официальном репозитории перед использованием в продакшне.
:::

### Установка Docker

Для запуска Tymeslot в контейнере вам нужен Docker.

- На Linux установите Docker Engine согласно официальной [документации Docker](https://docs.docker.com/engine/install/)
- На Windows установите [Docker Desktop](https://www.docker.com/products/docker-desktop/)

:::tip Docker для Windows
Если вы используете Docker для Windows, убедитесь, что виртуализация включена, а Docker Desktop запущен перед продолжением.
:::

## Установка Tymeslot через Docker

Это самый практичный способ для систем Linux и Windows.

### Загрузка Docker-образа

В предоставленном черновике используется следующий образ:

```bash
docker pull youruser/tymeslot:latest
```

:::caution Проверьте Docker-образ перед использованием
Точное имя публичного образа не подтверждено содержимым репозитория. Перед использованием этой команды в продакшне проверьте правильное имя образа в официальном репозитории Tymeslot, в релизах или на странице Docker Hub, если она опубликована.
:::

### Запуск контейнера

Вы можете запустить Tymeslot с помощью следующей команды из черновика установки:

```bash
docker run --name tymeslot \
  -p 4000:4000 \
  -e SECRET_KEY_BASE="$(openssl rand -base64 64 | tr -d '\n')" \
  -e PHX_HOST=localhost \
  -v tymeslot_data:/app/data \
  -v tymeslot_pg:/var/lib/postgresql/data \
  youruser/tymeslot:latest
```

Если вы используете Windows PowerShell, команда `openssl` может быть недоступна по умолчанию. В этом случае сгенерируйте безопасное случайное значение отдельно и вручную замените `SECRET_KEY_BASE`.

### Справка по команде

| Опция | Назначение |
|---|---|
| `--name tymeslot` | Задает имя контейнера |
| `-p 4000:4000` | Открывает порт `4000` для доступа к Tymeslot |
| `-e SECRET_KEY_BASE=...` | Определяет секрет приложения для шифрования и сессий |
| `-e PHX_HOST=localhost` | Устанавливает имя хоста для Phoenix |
| `-v tymeslot_data:/app/data` | Сохраняет данные приложения |
| `-v tymeslot_pg:/var/lib/postgresql/data` | Сохраняет данные PostgreSQL |
| `youruser/tymeslot:latest` | Имя Docker-образа из черновика |

### Генерация секретного ключа на Linux

Если установлен `openssl`, секрет можно сгенерировать так:

```bash
openssl rand -base64 64 | tr -d '\n'
```

### Генерация секретного ключа в Windows PowerShell

В PowerShell можно сгенерировать пример секрета так:

```powershell
[Convert]::ToBase64String((1..64 | ForEach-Object { Get-Random -Maximum 256 }))
```

:::danger Используйте собственный секрет
Не используйте примерные секреты в продакшне. Всегда генерируйте свой `SECRET_KEY_BASE` и храните его в безопасности.
:::

## Настройка Tymeslot

Перед первым использованием рекомендуется проверить основные параметры запуска.

### Переменные окружения

Черновик подтверждает следующие переменные окружения:

| Переменная | Пример | Назначение |
|---|---|---|
| `SECRET_KEY_BASE` | `[your_secret_key]` | Защищает сессии и функции шифрования |
| `PHX_HOST` | `[your_domain]` или `[your_server_ip]` | Определяет публичное имя хоста приложения |

При использовании заполнителей:

- `[your_secret_key]` — ваш сгенерированный безопасный секрет
- `[your_domain]` — ваш публичный домен, если он есть
- `[your_server_ip]` — IP-адрес сервера для прямого доступа

### Настройка SMTP

В черновике также упоминается конфигурация SMTP в `config/prod.exs` для email-уведомлений:

```elixir
config :tymeslot, Tymeslot.Mailer,
  adapter: Bamboo.SMTPAdapter,
  server: "smtp.example.com",
  port: 587,
  username: "your_email@example.com",
  password: "your_password",
  tls: :if_available,
  auth: :always
```

:::note Проверка SMTP
Точный формат конфигурации почтового клиента следует проверить в актуальном репозитории Tymeslot перед применением в продакшне, так как настройки приложения могут меняться между версиями.
:::

## Доступ к Tymeslot

После запуска контейнера можно проверить веб-интерфейс.

### Локальный доступ

Если вы тестируете локально, откройте:

```text
http://localhost:4000
```

### Удаленный доступ

Если Tymeslot запущен на удаленном сервере, откройте:

```text
http://[your_server_ip]:4000
```

Если настроен домен и обратный прокси, доступ может быть через:

```text
https://[your_domain]
```

## Брандмауэр и сеть

Tymeslot требует сетевого доступа на порт, который вы открываете в Docker.

### Необходимый порт

| Порт | Протокол | Назначение |
|---|---|---|
| `4000` | TCP | Веб-доступ к приложению Tymeslot |

### Особенности брандмауэра

Убедитесь, что:

- порт `4000` открыт в брандмауэре сервера для прямого доступа
- брандмауэр облачного провайдера или хостинга также разрешает этот порт
- обратный прокси корректно перенаправляет трафик, если вы используете домен

:::caution Публичный доступ
Если вы открываете порт `4000` напрямую в интернет, убедитесь, что понимаете риски безопасности. Для продакшн-развертываний обычно лучше использовать обратный прокси с HTTPS.
:::

## Опциональная нативная установка

Нативная установка возможна, если вы не хотите использовать Docker. Этот способ более сложный и требует среды, совместимой с Elixir и Phoenix.

### Основные шаги нативной установки

Согласно черновику, процесс такой:

```bash
git clone https://github.com/tymeslot/tymeslot.git
cd tymeslot
mix deps.get
mix ecto.create
mix ecto.migrate
mix phx.server
```

### Особенности нативного развертывания

Для нативного запуска вам также понадобятся:

- Elixir
- Erlang/OTP
- зависимости Phoenix
- настроенная поддерживаемая база данных
- проверка конфигурации для продакшна

:::info Сложность нативной установки
Предоставленные исходные материалы не содержат достаточно проверенной информации для безопасного документирования полного нативного продакшн-развертывания на Linux или Windows. Если вы хотите использовать этот способ, изучите официальную документацию репозитория перед развертыванием.
:::

## Обслуживание и устранение неполадок

После развертывания проверьте, что сервис доступен и данные сохраняются.

### Проверка статуса контейнера

```bash
docker ps
```

### Просмотр логов

```bash
docker logs tymeslot
```

### Перезапуск контейнера

```bash
docker restart tymeslot
```

### Распространенные проблемы

| Проблема | Возможная причина | Рекомендуемые действия |
|---|---|---|
| Веб-страница не загружается | Порт `4000` заблокирован | Проверьте брандмауэр и проброс портов Docker |
| Контейнер сразу завершается | Неверные переменные окружения | Проверьте `SECRET_KEY_BASE` и настройки образа |
| Ошибки, связанные с базой данных | Проблемы с постоянным хранилищем или инициализацией БД | Проверьте логи и используемые тома |
| Неправильный хост в ссылках | Неверно задан `PHX_HOST` | Установите `PHX_HOST` в `[your_domain]` или `[your_server_ip]` |

## Лучшие практики

### Используйте постоянные тома

Всегда сохраняйте данные приложения и базы данных в постоянных Docker-томах, чтобы при пересоздании контейнера данные не потерялись.

### Используйте обратный прокси

Для публичных развертываний лучше разместить Tymeslot за обратным прокси, например Nginx или Apache, и включить HTTPS.

### Проверяйте изменения в исходниках

Поскольку Tymeslot активно развивается, перед обновлением изучайте официальный репозиторий. Это особенно важно для:

- изменений Docker-образов
- обновленных переменных окружения
- миграций базы данных
- изменений в конфигурации почты
