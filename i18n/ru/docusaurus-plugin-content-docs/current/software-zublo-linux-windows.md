## Введение

Zublo — это open-source трекер подписок с возможностью самостоятельного хостинга, который помогает управлять регулярными платежами и связанными данными в одном месте. В этом руководстве вы узнаете, как развернуть Zublo на сервере с Linux или Windows с помощью Docker и проверить корректность работы веб-интерфейса.

![](https://screensaver01.zap-hosting.com/index.php/s/SCREENSHOT_PLACEHOLDER/preview)

## Подготовка

Перед началом убедитесь, что ваш сервер соответствует базовым требованиям и на нем установлен Docker.

### Требования

| Компонент | Рекомендуемый минимум |
| --- | --- |
| CPU | 1 vCore |
| RAM | 512 МБ |
| Хранилище | 1 ГБ свободного места на диске |
| Операционная система | Linux или Windows с поддержкой Docker |
| Сеть | Доступ к порту `9597` |

### Необходимое ПО

Вам потребуется установить следующее ПО:

| Программа | Назначение |
| --- | --- |
| [Docker](https://www.docker.com/) | Запуск контейнера Zublo |
| Docker Compose | Запуск и управление стеком контейнеров |

:::info Требование Docker
Это руководство основано на методе развертывания через Docker, указанном в репозитории проекта. Если Docker еще не установлен, сначала выполните его установку.
:::

:::caution Доступ к порту
По умолчанию Zublo использует порт `9597`. Убедитесь, что этот порт не занят другим сервисом и разрешен в вашем файрволе, если хотите получить удаленный доступ к веб-интерфейсу.
:::

## О Zublo

Zublo предназначен для отслеживания подписок, регулярных платежей и связанных расходов через веб-приложение с самостоятельным хостингом. Согласно информации из репозитория, проект ориентирован на Docker и рассчитан на пользователей, желающих контролировать свои данные самостоятельно.

### Что предлагает приложение

Исходя из доступной информации, Zublo фокусируется на следующих функциях:

| Область функционала | Описание |
| --- | --- |
| Отслеживание подписок | Управление регулярными сервисами и платежами |
| Самостоятельный хостинг | Хранение данных подписок на собственном сервере |
| Веб-интерфейс | Доступ к приложению через браузер |
| Доступ к API | Использование встроенного API |
| Админ-панель | Доступ к админ-интерфейсу PocketBase |

:::note Доступность функций
Некоторые продвинутые функции, упомянутые в сторонних обзорах, например AI-поддержка анализа, могут зависеть от версии проекта. Если функция отсутствует в вашей установке, проверьте официальную документацию: [Zublo GitHub repository](https://github.com/danielalves96/zublo)
:::

## Создание директории проекта Zublo

Сначала создайте отдельную папку для развертывания Zublo, чтобы конфигурация и данные оставались организованными.

### Linux

Выполните в терминале:

```bash
mkdir -p zublo-data
cd zublo-data
```

### Windows

Если используете PowerShell, выполните:

```powershell
mkdir zublo-data
cd zublo-data
```

В этой папке будут храниться файл `docker-compose.yml` и директория с постоянными данными приложения.

## Создание конфигурации Docker Compose

Далее создайте файл `docker-compose.yml` в каталоге проекта.

### Пример конфигурации

Используйте следующий пример:

```yaml
services:
  zublo:
    image: ghcr.io/danielalves96/zublo:latest
    container_name: zublo
    restart: unless-stopped
    ports:
      - "9597:9597"
    environment:
      PB_ENCRYPTION_KEY: "[your_secure_encryption_key]"
    volumes:
      - ./zublo-data:/pb/pb_data
```

### Обзор конфигурации

| Ключ | Пример значения | Назначение |
| --- | --- | --- |
| `image` | `ghcr.io/danielalves96/zublo:latest` | Загружает последний образ контейнера Zublo |
| `container_name` | `zublo` | Задает читаемое имя контейнера |
| `restart` | `unless-stopped` | Автоматически перезапускает контейнер после сбоев или перезагрузок |
| `ports` | `9597:9597` | Открывает веб-приложение на порту `9597` |
| `PB_ENCRYPTION_KEY` | `[your_secure_encryption_key]` | Защищает чувствительные данные в хранилище |
| `volumes` | `./zublo-data:/pb/pb_data` | Сохраняет данные приложения на сервере |

:::danger Используйте надежный ключ шифрования
Не оставляйте ключ шифрования слабым или по умолчанию. Замените `[your_secure_encryption_key]` на длинный случайный секрет перед запуском контейнера в продакшене.
:::

### Сохранение файла

Сохраните файл под именем `docker-compose.yml` в текущей папке. После этого можно запускать сервис.

## Запуск Zublo

После создания файла Compose запустите стек контейнеров.

```bash
docker compose up -d
```

Если у вас используется старая версия Docker Compose, возможно, потребуется команда:

```bash
docker-compose up -d
```

### Проверка работы контейнера

Проверьте статус контейнера командой:

```bash
docker ps
```

В списке должен быть контейнер с именем `zublo`.

### Просмотр логов при необходимости

Если сервис не запускается корректно, посмотрите логи:

```bash
docker logs zublo
```

Это поможет выявить проблемы с конфигурацией, конфликтами портов или правами доступа к данным.

## Доступ к веб-интерфейсу

Когда контейнер запущен, откройте Zublo в браузере.

### URL приложения

Замените `[your_server_ip]` на публичный IP вашего сервера или локальный IP в сети.

| Интерфейс | URL |
| --- | --- |
| Основное приложение | `http://[your_server_ip]:9597` |
| Админ-панель PocketBase | `http://[your_server_ip]:9597/_/` |
| API | `http://[your_server_ip]:9597/api/` |

:::info Локальный и удаленный доступ
Если вы работаете непосредственно на сервере с Zublo, можно использовать `http://localhost:9597`. Для удаленного доступа используйте реальный IP сервера или настроенный домен.
:::

## Настройка безопасного доступа

После проверки доступности Zublo стоит продумать, как приложение открыто в интернет.

### Настройки файрвола

Если не удается подключиться с другого устройства, убедитесь, что порт `9597` открыт в настройках файрвола.

| Порт | Протокол | Назначение |
| --- | --- | --- |
| `9597` | TCP | Веб-интерфейс Zublo, админ-панель и API |

### Использование обратного прокси

Если хотите использовать домен, например `[your_domain]`, можно разместить Zublo за обратным прокси, например Nginx или другим поддерживаемым решением. Прокси будет перенаправлять запросы на `http://127.0.0.1:9597` или внутренний адрес Docker.

:::tip Используйте HTTPS для публичного доступа
Если планируете открывать Zublo в интернет, рекомендуется настроить обратный прокси с SSL, чтобы данные подписок передавались в зашифрованном виде.
:::

## Управление постоянными данными

Zublo хранит свои данные в смонтированной директории, указанной в Compose-файле.

### Расположение данных

В этом руководстве используется следующая настройка тома:

```yaml
volumes:
  - ./zublo-data:/pb/pb_data
```

Это значит, что данные приложения сохраняются в локальной папке `zublo-data` внутри каталога проекта.

### Зачем нужна постоянность

Постоянное хранилище гарантирует сохранность данных при:

- перезапуске контейнера
- перезагрузке сервера
- обновлении образа
- техническом обслуживании

:::caution Рекомендация по резервному копированию
Поскольку Zublo хранит важные личные данные подписок, регулярно делайте резервные копии папки `zublo-data` перед крупными изменениями или обновлениями.
:::

## Базовое обслуживание

После запуска Zublo время от времени может понадобиться обновить или перезапустить сервис.

### Перезапуск контейнера

```bash
docker restart zublo
```

### Остановка развертывания

```bash
docker compose down
```

### Обновление до последнего образа

Скачайте последний образ и пересоздайте контейнер:

```bash
docker compose pull
docker compose up -d
```

### Полезные команды

| Команда | Назначение |
| --- | --- |
| `docker compose up -d` | Запуск Zublo в фоне |
| `docker ps` | Проверка запущенных контейнеров |
| `docker logs zublo` | Просмотр логов контейнера |
| `docker restart zublo` | Перезапуск контейнера |
| `docker compose down` | Остановка и удаление контейнера |
| `docker compose pull` | Загрузка последнего образа |

## Устранение неполадок

Если Zublo работает не так, как ожидалось, проверьте распространенные причины.

### Порт `9597` уже занят

Если Docker выдает ошибку привязки порта, значит другой сервис использует `9597`. Остановите конфликтующий сервис или измените проброс порта на хосте.

Пример с другим внешним портом:

```yaml
ports:
  - "9600:9597"
```

В этом случае доступ к приложению будет по адресу `http://[your_server_ip]:9600`.

### Контейнер запущен, но страница не загружается

Проверьте следующее:

- контейнер действительно запущен (`docker ps`)
- изучите логи (`docker logs zublo`)
- файрвол разрешает выбранный порт
- используете правильный IP или имя хоста

### Проблемы с ключом шифрования

Если вы измените `PB_ENCRYPTION_KEY` после первоначальной настройки, существующие зашифрованные данные могут стать недоступны. При необходимости смены ключа сначала изучите официальную документацию проекта, так как точное поведение миграции не описано в исходных материалах.

:::note Отсутствие деталей поведения
Исходные материалы не полностью описывают все шаги первого запуска, создание аккаунтов или продвинутые настройки. Для подробностей обратитесь к официальной документации: [Zublo GitHub repository](https://github.com/danielalves96/zublo)
:::
