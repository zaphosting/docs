## مقدمة

تكوين عناوين IP على سيرفر Linux المخصص الخاص بك يتيح لك التحكم في كيفية تواصل السيرفر داخل الشبكة. يشمل ذلك تعيين عناوين IP، تحديد البوابة الافتراضية، وتكوين خوادم DNS المطلوبة للاتصال بالشبكة.

:::warning هل السيرفر غير قابل للوصول بسبب فقدان اتصال الإنترنت؟
في حال أصبح السيرفر غير قابل للوصول بسبب تكوين شبكة خاطئ أو فقدان اتصال الإنترنت، قد لا يعمل الوصول عن بُعد عبر SSH. استخدم [وحدة تحكم iLO HTML](dedicated-ilo.md#the-html5-console) للاتصال بسيرفرك وإصلاح تكوين الشبكة.
:::

## التحضير

قبل تكوين عنوان IP ثابت، تأكد من توفر المعلومات التالية:

- عنوان IP الذي تريد تعيينه  
- قناع الشبكة الفرعية  
- البوابة الافتراضية  
- خوادم DNS  

يمكنك العثور على هذه المعلومات في واجهة الويب. للقيام بذلك، افتح **إدارة سيرفر مخصص** وانتقل إلى **عناوين IP**.

## التكوين

<Tabs>
<TabItem value="ubuntu" label="أوبونتو" default>

تستخدم توزيعات أوبونتو **Netplan** لتكوين الشبكة. اتصل بسيرفرك المخصص عبر SSH وابحث عن ملف تكوين Netplan في مجلد `/etc/netplan/`. افتح ملف التكوين:

```bash
nano /etc/netplan/50-cloud-init.yaml
```

قم بتعديل التكوين لتعريف عنوان IP ثابت:
```
network:
  version: 2
  ethernets:
    eth0:
      match:
        macaddress: "bc:24:11:5f:5c:34"
      addresses:
      - "109.230.238.45/24"
      nameservers:
        addresses:
        - 8.8.8.8
        - 1.1.1.1
      set-name: "eth0"
      routes:
      - to: "default"
        via: "109.230.238.1"
```

قم بتعديل القيم التالية لتتناسب مع تكوين شبكتك:

- 109.230.238.45 → عنوان IP الثابت الخاص بك
- 109.230.238.1 → البوابة الخاصة بك
- عناوين DNS → خوادم DNS المفضلة لديك

احفظ الملف وطبق التكوين:

```
sudo netplan apply
```

</TabItem> <TabItem value="debian" label="ديبيان">

عادةً ما يستخدم ديبيان ملف تكوين واجهات الشبكة بدلاً من Netplan. اتصل بسيرفرك المخصص عبر SSH وافتح ملف تكوين واجهات الشبكة:

```
nano /etc/network/interfaces
```

أضف أو عدّل التكوين لواجهة الشبكة الخاصة بك. مثال على التكوين:

```bash
auto lo
iface lo inet loopback

# الواجهة الشبكية الأساسية
allow-hotplug ens18
iface ens18 inet static
        address 134.255.219.160/24
        gateway 134.255.219.1
        # خيارات dns-* يتم تنفيذها بواسطة حزمة resolvconf، إذا كانت مثبتة
        dns-nameservers 1.1.1.1
```

قم بتعديل القيم حسب تكوين شبكتك:
- address → عنوان IP الثابت الخاص بك
- netmask → قناع الشبكة الفرعية
- gateway → البوابة الافتراضية
- dns-nameservers → خوادم DNS المفضلة لديك

بعد حفظ ملف التكوين، أعد تشغيل خدمة الشبكة:

```
systemctl restart networking
```

</TabItem> </Tabs>



## التحقق

بعد تطبيق التكوين، يُنصح بالتحقق من أن عنوان IP الجديد تم تطبيقه بشكل صحيح. للقيام بذلك، نفذ الأمر التالي في الطرفية:

```
ip a
```


يعرض هذا الأمر جميع **واجهات الشبكة** المتوفرة على النظام مع تفاصيل التكوين الحالية. في الناتج، ابحث عن الواجهة التي قمت بتكوينها سابقًا، عادةً ما تكون مسماة `eth0`، `ens18`، أو معرف مشابه حسب النظام.

داخل قسم الواجهة، ابحث عن الإدخال **`inet`**، الذي يمثل عنوان IPv4 المعين للواجهة. يجب أن يحتوي الناتج على عنوان IP الثابت الذي قمت بتكوينه سابقًا. على سبيل المثال:

```
inet 109.230.238.45/24
```

![img](https://screensaver01.zap-hosting.com/index.php/s/TmaCRcfLgiabTBp/preview)

إذا ظهر العنوان الصحيح في الناتج، فقد تم تطبيق تكوين IP الثابت بنجاح. إذا لم يظهر عنوان IP الجديد، تحقق من ملف التكوين مرة أخرى وتأكد من حفظ التغييرات بشكل صحيح قبل إعادة تطبيق التكوين.



## الخلاصة

مبروك! لقد قمت بتكوين عنوان IP الخاص بك على سيرفر Linux بنجاح. لأي أسئلة أو مساعدة إضافية، لا تتردد في التواصل مع فريق الدعم لدينا، المتوفر يوميًا لمساعدتك! 🙂
