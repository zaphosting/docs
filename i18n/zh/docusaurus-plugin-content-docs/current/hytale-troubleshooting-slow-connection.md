## 介绍

连接 Hytale 服务器时，遇到连接极慢或世界始终加载不完全的问题非常令人头疼。
这些问题乍看之下通常会被认为是服务器端的问题，但实际上，它们可能既源自服务器端，也可能是客户端网络行为导致的。

很多情况下，服务器本身运行正常，而客户端却难以正确建立或维持接收世界数据所需的连接。理解这一点对于后续排查非常重要。

## 症状

受影响的玩家通常反馈他们的网络连接总体稳定且速度快。视频流、网页浏览或文件下载等活动均无明显问题。
Hytale 的单人世界也能正常加载和运行。

问题只在加入多人服务器时显现。
连接时间异常长，世界可能永远加载不完。
有时服务器上的其他玩家能看到受影响玩家移动，但该玩家自己无法与世界互动。这些症状主要出现在 Windows 系统上。

## 背景与原因

虽然表现类似服务器性能问题，但调查显示，常见原因在客户端。
某些网络适配器，尤其是 Intel 和 Realtek 芯片组，存在与 Hytale 使用的网络协议驱动兼容性罕见问题。

Hytale 依赖 QUIC 协议，这是一种现代网络技术。
并非所有网络驱动都完全支持该协议，导致连接虽然建立，但世界数据无法正确传输。这种情况可能发生在 Wi-Fi 和有线以太网连接上。

## 验证原因

要判断问题是否出在本地系统，可以使用外接 USB 以太网适配器进行测试。
通过该适配器重新连接服务器，便能隔离问题。

如果使用 USB 适配器时连接正常，则问题可归因于原网络适配器或其驱动配置。

## 解决方案

确认问题后，调整客户端系统中网络适配器的高级设置通常能解决问题。
这些调整主要是关闭某些优化、省电和卸载功能，这些功能会干扰现代网络协议。

常见调整包括禁用优先级和 VLAN 处理、IPv4 和 IPv6 的接收分段合并，以及各种节能功能，如节能以太网和绿色以太网。

如果这些调整无效，可以尝试更全面的配置，修改大量高级适配器设置。
由于此方法侵入性强且难以恢复默认值，建议仅作为最后手段使用。


### 禁用优先级和 VLAN

某些情况下，网络优先级功能会干扰 Hytale 建立和维护连接的方式。部分网络驱动对优先级和 VLAN 标记的处理与现代传输协议不兼容。

禁用受影响网络适配器上的优先级和 VLAN 功能，已被证明能解决世界加载失败或连接卡顿的问题。

应用此更改后，网络连接会短暂重置，恢复后请测试服务器连接，确认问题是否解决。

### 禁用接收分段合并

接收分段合并是一种网络优化功能，通过将多个数据包合并成更大段来提升性能。
虽然通常有益，但对依赖实时数据传输的应用可能造成问题。

禁用 IPv4 和 IPv6 的接收分段合并，可以提升与 Hytale 使用的 QUIC 协议的兼容性。许多受影响系统通过此调整解决了连接缓慢和世界加载不全的问题。

禁用后，请再次测试网络连接，确认世界数据是否能正常加载。

### 禁用节能和省电功能

许多网络适配器内置激进的节能和能效功能，可能会影响连接稳定性。这些功能在需要持续数据流时，可能会中断或延迟网络流量。禁用高级 EEE、节能以太网、绿色以太网、ARP 卸载和流控等选项，能显著提升连接可靠性。

此方案对问题偶发或连接一段时间后出现的情况尤其有效。调整后，网络适配器将以更稳定但能耗较高的模式运行。



### 高级适配器配置（最后手段）

如果以上方案均无效，可尝试更全面的网络适配器配置。此方法禁用大量卸载、优化和电源管理功能，同时调整缓冲区大小和队列处理。

由于这些更改会大幅改变适配器行为，且可能无法保留默认值，恢复设置可能需要重新安装网络驱动。因此，只有在所有其他方法失败时才建议使用。

强烈建议在应用此配置前，记录所有原始适配器设置。



## 总结

Hytale 服务器连接缓慢和世界加载失败可能由服务器端或客户端因素引起。
排除服务器问题后，客户端网络驱动兼容性是已知原因。

通过调整特定网络适配器设置，受影响玩家可以恢复正常连接，顺利加载多人世界，无需更改服务器本身。
