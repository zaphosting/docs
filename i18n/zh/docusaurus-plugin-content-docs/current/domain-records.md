## 介绍

域名可以通过多种DNS记录类型轻松配置，这些记录执行不同的操作。在本指南中，我们将介绍所有不同的记录类型并解释它们的工作原理。

<InlineVoucher />

## 配置

你可以在域名管理的DNS管理中配置域名记录。点击绿色的 **新建条目** 按钮即可打开弹窗窗口。在这里，你可以根据需要配置并创建新的记录。

![img](https://screensaver01.zap-hosting.com/index.php/s/5qGjz8jDi4sNGcQ/download)

:::info
我们使用 **zap-hosting.com** 域名来演示各类DNS记录。请在实际操作中使用你自己的域名。
:::

:::warning
配置DNS记录时，请确保在记录末尾添加一个点（.），以表明域名完整，避免DNS系统分配时产生误解。
:::

## DNS记录类型

DNS记录有多种类型，含义和用途各不相同。下面将详细说明它们的含义和区别。

### A记录

A记录用于将“**子域**”或“**主域名**”转发到“**IPv4地址**”。

| 名称（示例）       | 类型 | 值（示例）      | TTL  | 优先级 |
| ------------------ | ---- | --------------- | ---- | ------ |
| zap-hosting.com.   | A    | IPv4地址        | 1440 | 0      |


### AAAA记录

AAAA记录用于将“**子域**”或“**主域名**”转发到“**IPv6地址**”。

| 名称（示例）       | 类型 | 值（示例）      | TTL  | 优先级 |
| ------------------ | ---- | --------------- | ---- | ------ |
| zap-hosting.com.   | AAAA | IPv6地址        | 1440 | 0      |


### CNAME记录

CNAME记录用于将“**子域名**”转发到“**子域**”或“**主域名**”。

| 名称（示例） | 类型  | 值（示例）         | TTL  | 优先级 |
| ------------ | ----- | ------------------ | ---- | ------ |
| zap-test.    | CNAME | zap-hosting.com    | 1440 | 0      |


### MX记录

MX记录用于将“**子域**”或“**主域名**”重定向到“**域名**”或“**A记录**”。MX记录用于指定一个或多个邮件服务器。优先级指定当存在多个MX记录时，优先使用哪个邮件服务器。优先级必须是0到65535之间的整数，数值越小优先级越高。

| 名称（示例）       | 类型  | 值（示例）          | TTL  | 优先级 |
| ------------------ | ----- | ------------------- | ---- | ------ |
| zap-hosting.com.   | CNAME | mailserverxy.com    | 1440 | 0      |


### PTR记录

PTR记录用于将“**IPv4地址**”或“**IPv6地址**”转发到“**子域**”或“**主域名**”。PTR记录是A记录或AAAA记录的反向映射。

| 名称（示例）                 | 类型  | 值（示例）          | TTL  | 优先级 |
| ---------------------------- | ----- | ------------------- | ---- | ------ |
| 1.0.0.127.in-addr.arpa.      | PTR   | zap-hosting.com.    | 1440 | 0      |


### RP记录

RP记录用于提供域名所有者的信息。通常填写一个邮箱地址，邮箱中的“@”符号用“点”替代。

| 名称（示例） | 类型 | 值（示例）             | TTL  | 优先级 |
| ------------ | ---- | ---------------------- | ---- | ------ |
| rp.          | RP   | info.zap-hosting.com.  | 1440 | 0      |

### SRV记录

SRV记录用于将“**子域**”或“**主域名**”转发到“**A记录**”。  
SRV记录适用于不使用标准端口的应用，因此需要端口转发。  
优先级定义了当多个SRV记录用于同一服务时，优先使用哪个服务。  
优先级必须是整数，数值越小优先级越高。

| 名称（示例）       | 类型 | 值（示例）                  | TTL  | 优先级 |
| ------------------ | ---- | --------------------------- | ---- | ------ |
| zap-hosting.com.   | SRV  | 0 2006 zap-hosting.com.     | 1440 | 0      |

### TXT记录

TXT记录用于添加任意文本内容。在某些系统中，内容用于编码管理数据。TXT记录常用于创建SPF、DMARC和DKIM条目。

| 名称（示例）       | 类型 | 值（示例）      | TTL  | 优先级 |
| ------------------ | ---- | --------------- | ---- | ------ |
| zap-hosting.com.   | TXT  | v=spf1 -all     | 1440 | 0      |
