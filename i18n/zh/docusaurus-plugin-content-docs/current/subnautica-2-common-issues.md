## 介绍

Subnautica 2 服务器问题可能包括启动失败、崩溃、延迟、连接问题以及游戏内性能不佳。通过本指南，你将学会识别 Subnautica 2 游戏服务器的常见问题，并应用实用解决方案，让你的游戏重新在线并顺畅运行。

:::info 早期访问版本变更
如果 Subnautica 2 服务器软件仍在积极更新，不同版本之间的行为、配置文件或可用功能可能有所不同。如果你发现服务器表现异常，请务必与 ZAP-Hosting 面板中当前安装的版本进行对比。
:::

## 问题与解决方案

服务器问题通常不是单一原因导致。很多情况下，问题与过时文件、错误设置、资源不足或临时网络状况有关。以下章节将帮助你逐步排查最常见的问题。

### 常见问题

常见问题通常影响服务器的基本可用性、可见性或启动行为。这些是你在深入排查高级原因前应首先检查的内容。

<details>
  <summary>游戏中无法看到服务器</summary>

如果你的 Subnautica 2 服务器在游戏中不可见，可能是服务器未正确启动或初始化未完成。原因可能是配置值无效、安装文件不完整或启动错误。

请先检查以下内容：

- 确认服务器在 ZAP-Hosting 面板中显示为运行状态
- 查看服务器控制台是否有启动错误
- 检查日志文件是否有缺失文件或初始化失败的记录
- 确认游戏内服务器浏览器使用了正确的搜索过滤条件
- 启动后等待几分钟，有些服务器需要额外时间才能公开显示

如果服务器仍未出现，尝试重启一次，并从启动过程开始监控控制台输出。

</details>

<details>
  <summary>服务器无法启动</summary>

服务器完全无法启动，通常是文件损坏、更新不完整或配置无效导致。

请检查以下内容：

| 检查项 | 重要原因 |
| --- | --- |
| 服务器控制台输出 | 显示具体启动错误（如果有） |
| 最近的配置更改 | 单个无效值可能阻止启动 |
| 最近上传或修改的文件 | 手动更改的文件可能不完整或不兼容 |
| 当前游戏版本 | 服务器文件版本不匹配会导致启动失败 |

如果你最近修改了配置文件，请回滚最近的更改后重试。如果问题出现在更新或手动修改文件后，恢复备份通常能节省时间。

:::tip 重大更改前请备份
在编辑重要文件或更新服务器前，请先创建备份，以便出现新问题时快速回滚。
:::

</details>

<details>
  <summary>服务器密码或访问设置无效</summary>

如果服务器显示在线但玩家无法加入，请仔细检查访问相关设置。错误密码、客户端保存的过时信息或公私服设置不匹配都可能导致访问受阻。

请确认：

- 玩家输入的密码与服务器当前密码完全一致
- 修改访问设置后服务器已重启
- 客户端未使用旧的保存连接数据
- 如果服务器版本支持，私有或白名单设置配置正确

如果不确定是否配置导致问题，暂时简化设置并测试最小限制。

</details>

### 崩溃问题

意外崩溃会中断游戏，且频繁发生时可能导致数据丢失。最常见原因是服务器文件过时、不兼容的模组或资源过载。

<details>
  <summary>保持服务器更新</summary>

运行最新的 Subnautica 2 服务器版本对稳定性、兼容性和安全性至关重要。旧版本可能包含已修复的漏洞，或无法正常支持当前游戏客户端。

过时服务器可能导致：

- 启动失败
- 随机崩溃
- 版本不匹配错误
- 游戏行为不稳定

![](https://screensaver01.zap-hosting.com/index.php/s/SCREENSHOT_PLACEHOLDER/preview)

如果服务器在游戏更新后开始崩溃，先检查面板中是否有新版本可用。更新后重启服务器并再次测试。

</details>

<details>
  <summary>错误或不兼容的模组及附加内容</summary>

如果当前版本的 Subnautica 2 支持模组或其他服务器端附加内容，这些内容可能是崩溃的常见原因。不兼容或过时的内容可能部分加载，但在游戏过程中触发错误。

典型表现包括：

- 服务器启动后崩溃
- 玩家加入时服务器崩溃
- 世界加载时服务器卡死
- 添加新内容后出现错误

排查步骤如下：

1. 停止服务器
2. 移除或禁用最近添加的内容
3. 以最小配置启动服务器
4. 测试稳定性
5. 逐个重新启用内容

此方法帮助你准确定位导致问题的文件或包。

:::caution 版本兼容性
仅使用确认与当前 Subnautica 2 服务器版本兼容的模组、框架或扩展。即使一个过时组件也可能导致频繁崩溃。
:::

</details>

<details>
  <summary>世界或存档数据损坏</summary>

如果服务器在加载世界时或玩家连接后不久崩溃，存档数据可能已损坏。通常因强制关机、中断写入或反复崩溃导致。

可能的迹象包括：

- 服务器每次都在同一加载阶段崩溃
- 新世界能正常启动，旧世界无法启动
- 控制台报告存档或世界加载错误

怀疑存档损坏时，尝试使用备份或新世界测试。如果新世界能正常启动，问题很可能出在现有存档数据。

:::danger 数据丢失风险
请勿立即删除现有世界文件。先备份一份，以便后续尝试恢复。
:::

</details>

### 性能问题

性能问题通常表现为延迟、操作延迟、世界加载缓慢或游戏不稳定。通常由配置不当、硬件资源不足或附加内容过重引起。

<details>
  <summary>服务器配置不当</summary>

错误的配置值会增加 CPU、内存或存储负载，降低整体性能。如果你的 Subnautica 2 服务器运行缓慢或不稳定，请仔细检查设置。

特别要确认：

- 玩家相关限制是否符合你的套餐规格
- 世界相关设置是否合理，不超出需求
- 最近是否更改了实验性或非默认设置
- 应用更改后服务器是否已重启

通常你可以在 ZAP-Hosting 面板的 **Settings** 或 **Configs** 文件中管理配置。

</details>

<details>
  <summary>服务器套餐资源不足</summary>

资源不足的服务器会导致卡顿、加载时间长和不稳定。随着玩家数量增加、附加内容增多或世界规模扩大，这种情况更明显。

ZAP-Hosting 在订购时会推荐适合的配置，帮助你选择合适资源。

![](https://screensaver01.zap-hosting.com/index.php/s/SCREENSHOT_PLACEHOLDER/preview)

以下表格为一般排查参考：

| 现象 | 可能原因 | 推荐操作 |
| --- | --- | --- |
| 频繁卡顿 | CPU 资源不足 | 减少负载或升级套餐 |
| 加载缓慢 | 存储或内存瓶颈 | 检查使用情况，考虑增加资源 |
| 玩家多时崩溃 | 总体容量不足 | 降低负载或扩容服务器 |
| 空服时性能好，满员时差 | 套餐资源不足以支撑活跃度 | 根据预期玩家数升级套餐 |

:::note 最低推荐
推荐套餐仅为起点。如果你的 Subnautica 2 服务器使用更大世界、更多玩家或附加内容，建议从一开始就选择更高配置。
:::

</details>

<details>
  <summary>模组或附加内容导致资源占用高</summary>

即使附加内容不导致崩溃，也可能严重影响性能。优化不佳的内容会增加 CPU 使用率、占用过多内存或触发频繁后台处理。

排查方法是对比有无附加内容时的性能表现。如果最小配置下服务器表现正常，逐步恢复内容直到问题复现。

此方法尤其适用于：

- 安装新内容后开始卡顿
- 服务器运行一段时间后变得不稳定
- 性能下降仅在特定区域或场景出现
- 资源使用增加但配置未变

</details>

<details>
  <summary>大型世界或长期运行存档导致性能下降</summary>

随着服务器世界规模扩大和玩家活动积累，性能可能下降。这在长期运行且存档数据庞大的服务器中很常见。

如果你发现性能起初良好，后来变差，请考虑：

- 测试较新备份是否表现更好
- 检查是否同时在线玩家过多
- 移除不必要的附加内容
- 确认问题是否只出现在特定世界

如果新测试世界性能明显更好，性能下降可能与世界大小或存档复杂度有关。

</details>

### 网络问题

网络问题影响玩家连接和游戏体验，表现为高延迟、回弹、断线或完全无法连接。

<details>
  <summary>延迟波动、卡顿和网络延迟</summary>

延迟波动和网络延迟通常由服务器负载和路由状况共同引起。即使游戏在线，硬件过载或地理距离远也会导致体验不佳。

常见原因包括：

- 服务器位置远离玩家群体
- 服务器 CPU 或内存负载过高
- 同时连接玩家过多
- 后台任务或其他进程占用资源
- 玩家与服务器间临时路由或丢包问题

提升网络质量请检查：

| 检查项 | 结果 |
| --- | --- |
| 选择靠近玩家的服务器位置 | 降低平均延迟 |
| 减少不必要的服务器负载 | 游戏中减少卡顿 |
| 保持服务器更新 | 提升稳定性和兼容性 |
| 不同时间段测试 | 识别临时路由问题 |

如果只有单个玩家受影响，问题可能出在该玩家的网络。如果所有玩家都受影响，原因更可能是服务器端或路由问题。

</details>

<details>
  <summary>玩家随机断线</summary>

随机断线可能由不稳定路由、服务器资源过载或临时软件不稳定引起。如果玩家无明显游戏内原因被踢出，比较断线时间与控制台消息及性能峰值。

请检查：

- 断线是否发生在高峰负载时
- 服务器是否短暂重启或卡死
- 是否所有玩家同时断线或仅个别玩家
- 问题是否出现在最近更新或配置更改后

如果所有玩家同时断线，通常是服务器或网络路径问题。若仅单个玩家断线，原因多为该玩家本地网络问题。

</details>

<details>
  <summary>DDoS 攻击及其他网络威胁</summary>

游戏服务器偶尔会遭受恶意流量攻击，包括分布式拒绝服务（DDoS）攻击。这些攻击可能导致卡顿、连接失败或临时宕机。

ZAP-Hosting 为托管服务提供 DDoS 保护，帮助减轻常见及高级攻击影响。但如果怀疑 Subnautica 2 服务器遭受攻击，请联系支持并提供尽可能详细的信息，例如：

- 问题开始时间
- 是否所有玩家均受影响
- 服务器是否完全无法访问
- 问题是否在特定时间重复出现

:::info 内置防护
DDoS 缓解由基础设施层面处理。怀疑恶意流量时，通常无需先更改游戏服务器设置。请收集症状并联系支持。
:::

</details>

### 游戏相关服务器问题

有些问题不是直接崩溃或网络故障，但仍影响玩家的游戏体验。

<details>
  <summary>客户端与服务器版本不匹配</summary>

游戏更新后玩家无法加入，可能是服务器和客户端版本不一致，尤其是补丁发布后常见。

解决步骤：

1. 确认服务器已更新
2. 更新后重启服务器
3. 确保玩家更新了游戏客户端
4. 再次测试连接

如果只有部分玩家能加入，先比较他们的游戏版本与服务器版本。

</details>

<details>
  <summary>修改设置后无效</summary>

如果修改配置后游戏内无变化，服务器可能仍使用缓存或未更新的运行时数据，或者编辑的文件不是当前生效文件。

请确认：

- 编辑了正确的文件
- 配置值已成功保存
- 修改后服务器已完全重启
- 你的服务器版本支持所更改的设置

如果当前 Subnautica 2 版本的配置行为不明确，避免猜测，逐项测试更改以明确生效内容。

</details>

## 预防措施

预防问题通常比事后修复更简单。定期维护能显著提升稳定性。

### 推荐维护清单

使用以下清单降低未来问题风险：

| 任务 | 理由 |
| --- | --- |
| 定期创建备份 | 保护世界和配置，防止数据丢失 |
| 谨慎更新服务器 | 减少漏洞和版本不匹配问题 |
| 一次只测试一项更改 | 出问题时更易排查 |
| 变更后监控性能 | 及早发现 CPU 或内存使用上升 |
| 仅保留必要内容 | 降低兼容性和性能风险 |

### 定期使用备份

定期备份能节省大量时间，防止永久数据丢失。配置更改、更新或世界问题导致故障时，可以恢复到已知正常状态，无需重建。

你可以使用现有的[备份方案](gameserver-backups.md)为游戏服务器创建手动或定时备份。

:::tip 风险更改前备份
更新服务器、编辑重要文件或添加新模组和内容前，请先备份。这样出现新问题时有安全回滚点。
:::
