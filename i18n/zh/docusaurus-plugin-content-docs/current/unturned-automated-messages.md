## 介绍

自动服务器消息是一种简单但超有效的方式，直接在游戏内向玩家传达重要信息。你可以用它来展示服务器规则、发布活动公告、分享实用小贴士，或者定期发送提醒，完全不用管理员手动操作。

通过**计划任务**，服务器消息可以自动发送到**Unturned**游戏内聊天，执行预设命令并按固定时间间隔循环播放。

## 自动消息是怎么运作的

自动消息是通过安排一个任务，在指定时间或间隔执行服务器命令来创建的。这个命令可以用来发送聊天消息、广播或者服务器通知。

配置好后，任务会在后台自动运行，向所有在线玩家显示消息。这样就能保证持续稳定的沟通，无需工作人员一直在线。



## 创建自动消息

要创建自动服务器消息，打开你的游戏服务器控制面板，进入**计划任务**栏目。新建一个计划任务，选择执行**命令**的选项。在命令栏输入你的游戏服务器支持的消息命令。

```
<消息命令> <你的消息内容>
```

把命令替换成你想用来触发消息的游戏命令或扩展命令，`<你的消息内容>`替换成你想让玩家看到的文本。

然后你可以定义消息发送的频率，比如每几分钟一次、每小时一次，或者在一天中的特定时间发送。详细的计划任务创建教程请参考[计划任务](gameserver-scheduled-tasks.md)指南。

## 自定义消息间隔

计划任务让你完全掌控消息出现的时间和频率。常见用例包括：

- 每隔几分钟显示服务器规则
- 在固定时间宣布即将到来的活动
- 给新玩家或回归玩家发送游戏小贴士
- 提醒玩家加入Discord或访问官网链接

你可以随时调整间隔，无需重启服务器，方便灵活地随着服务器发展做出改变。



## 重要提示

自动服务器消息的功能取决于游戏本身及其支持的命令。虽然基础的定时消息可以用标准服务器命令实现，但在格式、条件或高级自定义方面可能有限制。

如果游戏支持额外的插件、Mod或扩展，提供更多灵活性和定制选项，使用这些方案可能更合适。在这种情况下，专门的消息系统通常比默认游戏命令功能更丰富，更适合复杂或高度定制的服务器环境。



## 总结

自动服务器消息是保持玩家信息通畅和活跃的可靠方式，且无需人工干预。通过计划任务执行消息命令，服务器管理员能确保全服持续稳定的沟通。

设置完成后，自动消息会独立运行，只需调整游戏专用的消息命令和文本即可轻松复用。
