## 介绍

你在你的 Webspace 上运行邮件服务，却发现有些邮件投递失败？那你来对地方了。要保证邮件流量的可靠性，有很多方面需要注意。在本指南中，你将了解影响邮件投递的关键因素，帮助你提升邮件的可靠性。

## 黑名单

如果出现滥用行为，某些服务商可能会将邮件服务器的 IP 地址拉入黑名单，从而阻止邮件成功投递。结果就是邮件无法送达，或者被标记为危险/垃圾邮件。作为服务商，我们始终致力于主动防范此类滥用，并尽快阻断针对客户 Webspace 的滥用报告。

为了降低被拉黑的风险，我们采取了多项安全措施，包括监控邮件流量中的可疑行为和强制执行垃圾邮件过滤。

如果我们的 Webspace 实例仍被某些服务商拉入黑名单，我们会立即联系对方，尽快解决问题。

:::warning
如果你对邮件的顺畅和可靠投递有较高要求，选择带有专用 IP 的服务会是更好的选择。根据项目的规模和复杂度，我们推荐使用 **[VPS](https://zap-hosting.com/en/vps-hosting)** 或 **[独立服务器](https://zap-hosting.com/en/dedicated-server-hosting)** 来实现。
:::

## 遵守内容与安全标准

遵守内容和安全标准是运行邮件服务器的关键，能确保传输数据的完整性、可用性和机密性。这些标准涵盖了多种实践和协议，旨在保护邮件服务器免受未授权访问、滥用及其他安全威胁。你可以在下面的 **身份验证标准** 和 **内容规范** 部分了解更多。

### 身份验证标准

身份验证标准帮助邮件服务商和服务器确认发件人的可信度。为了证明你有权代表某个域发送邮件，业界制定了三大身份验证标准：**SPF**、**DKIM** 和 **DMARC**。

**SPF（发件人策略框架）：** SPF 通过指定哪些 IP 地址被授权代表某个域发送邮件，帮助防止邮件伪造。举例来说，你可以在域名 "example.com" 的 DNS 设置中添加 SPF 记录，定义允许哪些邮件服务器发送 "@example.com" 的邮件。如果邮件来自未授权服务器，收件方的邮件服务器可以将其标记为可疑或拒收。

**DKIM（域密钥识别邮件）：** DKIM 在邮件头部添加数字签名，帮助验证邮件在传输过程中未被篡改，且确实来自指定域。比如你从 "yourdomain.com" 发送邮件，DKIM 签名确保邮件内容未被修改，并确认域的真实性。收件方邮件服务器会用此签名检查邮件完整性。

**DMARC（基于域的消息认证、报告和一致性）：** DMARC 在 SPF 和 DKIM 基础上增加保护层，允许域名所有者指定未通过 SPF 或 DKIM 检查的邮件如何处理（如隔离或拒收）。例如，如果你的域设置了 DMARC 策略为“拒收”，任何未通过 SPF 或 DKIM 的邮件都会被收件方服务器拒绝。此外，DMARC 还会向域名所有者提供报告，显示域名的邮件使用情况，帮助监控潜在滥用。

### 内容规范

邮件内容也非常重要。要保持文本与图片的合理比例，避免使用视频、JavaScript 或表单等敏感内容，因为这些可能触发垃圾邮件过滤或导致投递问题。此外，每封邮件末尾应包含退订选项和公司信息。

其他最佳实践包括使用清晰简洁的语言，避免过于激进的促销信息，并确保邮件对移动设备友好。

## 测试措施与可靠性

市面上有多种服务可以测试你的邮件投递可靠性。这些服务不仅能帮你发现问题，还会解释原因并给出改进建议。推荐使用的服务有 [mail-tester.com](https://www.mail-tester.com/) 和 [mailgenius.com](https://www.mailgenius.com/)。

测试方法很简单：你只需将邮件发送到这些服务提供的邮箱地址，它们会分析所有相关信息，随后你可以在它们的网站上查看测试结果。这些工具对确保邮件配置正确、提升投递率、避免邮件被标记为垃圾邮件非常有帮助。

## 总结

如果你按照本指南的建议操作并成功实施，现在你的邮件投递安全性和可靠性应该大幅提升。你的邮件不仅更有可能成功送达，还能有效防止未授权访问和滥用。
