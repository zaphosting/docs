## 介绍

自动服务器消息是一种简单但超有效的方式，能直接在游戏内向玩家传达重要信息。你可以用它来展示服务器规则、发布活动公告、分享实用小贴士，或者定时提醒玩家，无需管理员手动操作。

借助**计划任务**，服务器消息可以自动发送到**Wreckfest 2**的游戏内聊天，通过预设命令按固定时间表执行。

## 自动消息是怎么运作的

自动消息是通过安排一个任务，在指定时间或间隔执行服务器命令来创建的。这个命令可以用来发送聊天消息、广播或服务器通知。

配置好后，任务会在后台自动运行，向所有在线玩家显示消息。这样就能保证持续稳定的沟通，无需工作人员一直在线。



## 创建自动消息

要创建自动服务器消息，打开你的游戏服务器控制面板，进入**计划任务**栏目。新建一个计划任务，选择执行**命令**的选项。在命令栏输入你的游戏服务器支持的消息命令。

```
<message command> <你的消息内容>
```

把 `<message command>` 替换成你想用来触发消息的游戏命令或扩展命令，把 `<你的消息内容>` 换成你想让玩家看到的文本。

然后你可以设置消息发送的频率，比如每几分钟一次、每小时一次，或者在一天中的特定时间发送。关于创建计划任务的详细步骤，请参考[计划任务](gameserver-scheduled-tasks.md)指南。

## 自定义消息间隔

计划任务让你完全掌控消息出现的时间和频率。常见用法包括：

- 每隔几分钟显示服务器规则
- 在固定时间宣布即将到来的活动
- 给新玩家或回归玩家发送游戏小贴士
- 提醒玩家加入Discord或访问官网链接

你可以随时调整间隔，无需重启服务器，灵活应对服务器的发展变化。



## 重要提示

自动服务器消息的功能取决于游戏本身及其支持的命令。虽然用标准服务器命令可以实现基础的定时消息，但在格式、条件判断或高级定制方面可能有限制。

如果游戏支持额外的插件、Mod或扩展，能提供更多灵活性和定制选项，使用这些方案会更有价值。这类专业消息系统通常比默认游戏命令功能更丰富，更适合复杂或高度定制的服务器环境。



## 总结

自动服务器消息是保持玩家信息通畅和活跃的可靠方式，无需人工干预。通过计划任务执行消息命令，服务器管理员能确保全服持续稳定的沟通。

设置完成后，自动消息会独立运行，只需调整游戏专用的消息命令和文本即可轻松复用。
