## 介绍

自定义地图让 Rust 服务器管理员能够打造超越默认程序生成地图的独特游戏体验。许多社区通过自定义地图引入新的地标、改变地形布局、举办特别活动，甚至完全重新设计环境。

这些地图通常使用 **RustEdit** 等工具制作，并通过 **Lone Design**、**Codefling** 或社区地图库等平台分发。一旦你拿到地图文件或下载链接，就可以直接在服务器设置中配置。

## 准备工作

在配置自定义地图之前，你需要一个有效的 **地图下载 URL**。自定义地图一般托管在地图分发平台，或者由地图作者提供。

这个链接通常指向一个 `.map` 文件或可下载的地图包。确保链接是公开可访问的，这样 Rust 服务器才能在启动时下载。示例地图 URL：

```
https://example-map-host.com/maps/custommap.map
```

## 配置

要在你的 Rust 服务器上配置自定义地图，打开你的 **游戏服务器管理面板**，进入 **设置**。

找到名为 **Level URL** 的配置项。这个设置允许服务器在启动时下载并加载自定义 Rust 地图。将你的自定义地图 URL 粘贴到 **Level URL** 字段中。例如：

```
https://example-map-host.com/maps/custommap.map
```

输入 URL 后，保存面板中的设置。服务器启动时，Rust 会自动从提供的 URL 下载地图并加载，而不是生成程序地图。

如果你的自定义地图需要额外的配置参数（比如特定的种子或世界大小），这些通常由地图作者提供，可能也需要添加到服务器配置中。

保存配置后，**重启服务器**，以便下载并加载自定义地图。

## 重要提示

自定义地图必须与你运行的 Rust 服务器版本兼容。如果地图文件过时或损坏，服务器可能无法启动，或者会回退到程序生成地图。

大型自定义地图可能会增加服务器启动时间，因为服务器需要先下载并准备地图，玩家才能进入。

如果切换地图后玩家无法连接，确认地图 URL 是否正确且能从互联网访问。
