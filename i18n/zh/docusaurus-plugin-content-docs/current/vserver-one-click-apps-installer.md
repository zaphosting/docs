## 介绍

一键应用安装器让你可以浏览丰富的应用目录，并直接从 VPS 网页界面部署应用。在本指南中，你将学习如何准备服务器、选择正确的安装类型、安装和管理应用，以及在需要时使用高级设置和故障排除工具。

## 准备工作

在使用一键应用安装器之前，你的 VPS 必须满足最低要求，并使用受支持的操作系统。

:::info 最低要求
应用功能要求你的 VPS 在线，并配置有足够资源以稳定运行平台服务和已安装的应用。
:::

| 要求 | 最低配置 | 推荐配置 |
| --- | --- | --- |
| 服务器状态 | 在线 | 在线 |
| 存储空间 | 30 GB | 50 GB |
| 内存 | 2 GB | 4 GB |
| CPU | 2 核 | 4 核 |
| 操作系统 | Debian 或 Ubuntu LTS | Debian 13 - EN - 64位 |

:::caution 建议使用全新操作系统
强烈建议在全新操作系统安装环境下设置应用功能。这有助于避免与现有服务、占用端口、Docker 配置或服务器上的自定义设置发生冲突。
:::

如果需要重新安装 VPS，请从 VPS 管理界面选择全新支持的 Debian 或 Ubuntu LTS 镜像。

## 访问一键应用安装器

打开 ZAP-Hosting 网页界面中的 VPS，找到“一键应用”部分。从这里开始初始设置，之后可以返回管理服务器上安装的所有应用。

首次打开此功能时，需要选择平台在 VPS 上的安装方式。

## 选择安装类型

你可以选择完整安装或仅节点安装。正确选项取决于该 VPS 是否既托管管理面板和应用，还是仅作为另一个面板的关联节点。

### 完整安装

如果你想将服务器设置为面板和节点服务器，选择**完整安装**。这是首次使用者和大多数标准部署的推荐选项。

通过此方式，你可以：

- 从应用目录部署应用
- 管理项目和环境
- 将所有应用集中管理
- 使用集成的网页界面管理工具
- 需要时访问高级面板

![img](https://screensaver01.zap-hosting.com/index.php/s/nqGL3PFPicTArY9/preview)

:::tip 大多数用户推荐
如果你是首次使用应用功能，选择完整安装选项。它提供最简单的设置，并让你直接通过 ZAP-Hosting 网页界面全面管理。
:::

### 仅节点安装

如果你希望该 VPS 仅作为节点使用，选择**仅节点安装**。之后可以将其链接到另一台 VPS 服务器或外部面板实例。

此选项适合：

- 跨多个 VPS 系统扩展
- 在一个面板集中管理
- 将服务器链接到现有部署
- 扩展已有的多节点架构

:::note 高级部署选项
仅节点安装适用于已有其他面板实例或想将服务器连接到外部环境的用户。
:::

## 安装应用功能

选择安装类型后，从网页界面启动设置。初始安装通常只需几分钟。

安装过程中，支持一键应用功能的平台服务会部署到你的 VPS。完成后，网页界面将开放应用目录和管理部分。

:::info 安装行为
具体安装进度可能因 VPS 性能、操作系统状态和网络状况而异。如果安装未成功完成，请稍后查看本指南的故障排除部分。
:::

## 浏览应用目录

安装完成后，你可以打开应用目录浏览可用应用。目录包含超过 300 个即装即用的应用，并支持筛选，帮助你快速找到合适的应用。

![](https://screensaver01.zap-hosting.com/index.php/s/fBwcCiNJTZooqnB/preview)

### 搜索与筛选应用

在目录中，你可以按名称搜索应用，按类别筛选。这有助于缩小应用范围，快速找到适合你项目的部署目标。

常用操作包括：

- 按应用名称搜索
- 按类别筛选
- 查看应用条目详情
- 选择应用进行部署

如果你管理多个应用下载目标或在安装前比较多个服务，这个流程特别有用。

### 开始安装应用

从目录选择应用后，通过网页界面开始安装。根据应用不同，你可能需要填写如下值：

- `[your_project_name]`
- `[your_environment_name]`
- `[your_domain]`
- `[your_app_name]`

这些占位符代表你部署的具体值。例如，`[your_domain]` 应替换为你想用于该应用的域名。

:::tip 项目与环境
使用项目和环境来保持部署有序。尤其当你托管多个应用、测试版本或区分生产与预发布服务时，这非常有帮助。
:::

## 管理已安装应用

部署完成后，已安装的应用会在网页界面的管理部分列出。你可以在这里监控和控制服务，无需手动操作底层容器。

![](https://screensaver01.zap-hosting.com/index.php/s/WTMSqkJ5zPKWjmn/preview)

### 可用管理操作

已安装应用部分允许你直接通过网页界面执行常用生命周期操作。

| 操作 | 作用 |
| --- | --- |
| 启动 | 启动已停止的应用 |
| 停止 | 停止正在运行的应用 |
| 查看日志 | 打开应用日志以便排查问题 |
| 重新安装 | 重新部署应用 |

这些工具让你更轻松地集中管理所有应用，尤其适合想用简单网页控制而非命令行操作的用户。

### 组织项目与环境

项目和环境帮助你清晰结构化部署。你可以用它们区分：

- 个人与生产应用
- 测试与线上环境
- 多客户或团队部署
- 同一 VPS 上的不同技术栈

随着应用商店应用数量增长，管理更多服务时，这种组织方式尤为重要。

## 域名、云访问与 SSL

一键应用功能支持自定义域名和 ZAP Cloud 集成，允许你安全地暴露应用并为部署申请 SSL 证书。

:::info 域名与 SSL 支持
你可以为应用使用自定义域名，并在平台支持的情况下享受 SSL 证书自动申请。
:::

分配域名时，确保你的 DNS 记录指向 `[your_server_ip]`。如果域名配置不正确，应用可能无法从互联网访问。

| 值 | 说明 |
| --- | --- |
| `[your_domain]` | 分配给应用的域名或子域名 |
| `[your_server_ip]` | 你的 VPS 公网 IP 地址 |

:::caution DNS 必须指向你的 VPS
如果 `[your_domain]` 的 DNS 未指向 `[your_server_ip]`，域名验证和应用访问可能失败。
:::

## 使用高级设置

高级用户可以通过高级设置部分访问外部面板，获得更多功能。

![](https://screensaver01.zap-hosting.com/index.php/s/BHnZWBGAzgo5myp/preview)

当你需要更深度定制、扩展部署控制或访问标准网页界面之外的功能时，这非常有用。

### 重要支持限制

:::caution 高级面板使用支持范围
欢迎高级用户使用外部 Coolify 面板进行更多定制和功能扩展。但支持仅限于 ZAP-Hosting 网页界面。如果你在网页界面外做出更改，可能无法获得针对这些自定义修改的帮助。
:::

### 何时使用高级面板

如果你需要：

- 更详细的部署定制
- 直接节点链接流程
- 高级应用配置
- 超出简化界面的面板级管理

可以使用高级面板。如果失去访问权限或集成停止响应，先使用下文的故障排除工具，再考虑手动修改。

## 链接节点

如果选择了仅节点安装类型，你可以将此 VPS 链接到另一台 VPS 或外部面板实例，实现基础设施扩展并从单一位置管理多个节点。

节点链接适合：

- 在多个 VPS 系统间分布应用
- 集中管理
- 扩容而不创建孤立部署
- 连接到外部面板环境

:::note 外部面板链接
你可以将节点链接到另一台 VPS 或外部面板实例。具体链接细节取决于目标环境。
:::

## 故障排除降级状态

如果网页界面显示降级状态，说明平台无法联系或获取一键应用功能实例的数据。请按顺序执行以下故障排除步骤。

### 状态指示

故障排除弹窗可能显示如下指示：

| 状态 | 含义 |
| --- | --- |
| Coolify 状态：活动 | 面板服务似乎正在运行 |
| API 状态：可访问 | 当前可访问 API |
| 服务状态：在线 | 你的 VPS 本身在线 |

如果有部分功能异常，继续执行以下步骤。

### 步骤 1：确认 VPS 在线

首先确认你的 VPS 服务在 ZAP-Hosting 网页界面显示为在线。服务器离线时，应用平台无法运行。

### 步骤 2：刷新状态

使用**刷新状态**操作，重新运行健康检查，获取最新数据。这能解决临时的间歇性问题。

:::tip 临时连接问题
降级状态有时由短暂通信故障引起。刷新状态是最快的初步检查。
:::

### 步骤 3：重启 Docker

如果状态仍降级，使用**重启 Docker**操作。此操作会强制重启平台实例。成功后等待 `1-2 分钟`，再刷新状态。

:::caution 服务重启
重启 Docker 可能会短暂中断正在运行的应用。
:::

### 步骤 4：刷新凭据

最后，使用**刷新凭据**。当状态显示在线但网页界面功能（如应用安装）异常时，这一步尤其重要。

此操作：

- **不会**删除你的数据
- 刷新 root 用户凭据
- 刷新网页界面使用的 API 令牌
- 成功后自动更新面板凭据

:::info 安全的凭据刷新
刷新凭据不会移除已安装应用或部署数据，仅更新访问和通信所需的凭据。
:::

### 何时联系支持

完成所有故障排除步骤后问题依旧，请通过 ZAP-Hosting 网站提交支持工单。

## 最佳实践

为保持一键应用安装器稳定且易于管理，请遵循以下建议：

| 最佳实践 | 原因 |
| --- | --- |
| 使用全新操作系统安装 | 减少与现有服务和端口冲突 |
| 满足推荐资源配置 | 提升性能和稳定性 |
| 将应用组织到项目和环境 | 保持部署可管理 |
| 分配域名前配置 DNS | 避免访问和 SSL 问题 |
| 谨慎使用高级设置 | 避免不受支持的自定义更改 |

:::tip 保持部署有序
如果计划托管大量应用，部署首批服务前，先制定清晰的 `[your_project_name]`、`[your_environment_name]` 和 `[your_app_name]` 命名方案。
:::
