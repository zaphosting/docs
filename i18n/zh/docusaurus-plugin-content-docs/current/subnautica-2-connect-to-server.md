不知道如何连接到您的 **Subnautica 2** 服务器，或者需要哪些信息才能开始？本指南将教您如何获取所需的服务器信息，建立连接，并排查常见的连接问题。

## 获取服务器信息

您需要所有相关信息来连接到您的服务器。您可以在游戏服务器的网页控制面板的仪表盘中找到服务器详情。重要信息包括：

- IP 地址和端口
- 服务器名称
- 服务器密码

使用上述信息，您应该能够识别您的服务器，并确保连接到正确的服务器。

| 详情 | 说明 |
| --- | --- |
| `IP 地址` | 您的 Subnautica 2 服务器的公网地址 |
| `端口` | 用于连接的游戏端口 |
| `服务器名称` | 如果游戏有服务器列表或会话浏览器，游戏内显示的名称 |
| `服务器密码` | 仅当服务器设置了密码保护时需要 |

## 建立服务器连接

连接 **Subnautica 2** 服务器的具体方式取决于您使用的游戏版本当前支持的多人游戏选项。通常，您需要服务器的 **IP 地址** 和 **端口**，有时还需要 **服务器密码**。

### 通过游戏内多人菜单加入

启动 **Subnautica 2**，从主菜单打开多人游戏或服务器加入部分。如果游戏提供直接连接选项，请准确输入您在 ZAP-Hosting 网页界面中看到的服务器 `IP 地址` 和 `端口`。如果设置了密码，系统会提示时输入 `服务器密码`。

确认信息后，游戏应尝试建立与服务器的连接。

:::note 直接连接选项说明
菜单和按钮的具体文字可能因游戏版本不同而有所变化。如果没有看到直接连接选项，请确认游戏是否要求通过服务器列表、邀请系统或会话浏览器加入。
:::

### 通过服务器列表或会话浏览器加入

如果 **Subnautica 2** 使用内置服务器浏览器而非手动直接连接字段，请使用配置的 `服务器名称` 搜索您的服务器。这就是为什么在尝试连接前检查网页界面中的服务器名称很重要。

找到正确服务器后：

1. 选择服务器条目
2. 确认显示的名称与您的服务器匹配
3. 加入服务器
4. 如有提示，输入密码

:::tip 使用唯一的服务器名称
如果您的服务器在列表中不易识别，请在服务器配置中设置一个清晰且唯一的服务器名称。这样更容易找到正确条目，避免连接到错误服务器。
:::

### 如果服务器未立即显示

如果您的服务器未出现在游戏内浏览器中，这并不总意味着服务器离线。有时服务器列表刷新需要时间，或者游戏只有在服务器成功初始化后才显示。

您应确认以下事项：

- 服务器在 ZAP-Hosting 网页界面显示为在线
- 使用了正确的游戏端口
- 服务器已完全启动
- 服务器名称配置正确
- 如果需要密码，已正确输入

## 可能的问题及解决方案

连接游戏服务器时常会受到多种因素影响。以下是最常见的问题及解决方案。

### 服务器不可见

如果服务器未成功完成初始化，可能导致不可见。这可能是由于配置错误或文件损坏。通常可以在服务器控制台或日志文件中找到更多信息。

:::caution 服务器必须启动完成
请勿在服务器完成启动流程前尝试连接。如果服务器仍在加载或重启中，玩家可能无法连接，或者服务器尚未出现在任何列表中。
:::

### 连接信息错误

如果无法建立连接，请确认您使用的 `IP 地址`、`端口` 和 `服务器密码` 是否正确。即使是小的输入错误也会导致连接失败。

| 检查项 | 需要确认的内容 |
| --- | --- |
| `IP 地址` | 与服务器仪表盘中显示的地址一致 |
| `端口` | 与分配的游戏端口一致 |
| `密码` | 完全按照配置输入，包括大小写 |
| `服务器名称` | 如果通过浏览器搜索，名称需与配置匹配 |

### 服务器在线但连接失败

如果服务器显示在线但仍无法连接，请查看网页界面中的服务器控制台和日志。错误信息可以帮助判断服务器是否加载正确，或是否有配置问题阻止玩家连接。

:::tip 优先查看日志
服务器控制台和日志文件通常是最快找出连接失败原因的途径。它们能帮助确认服务器是否正常监听，以及连接请求是否到达服务器。
:::
