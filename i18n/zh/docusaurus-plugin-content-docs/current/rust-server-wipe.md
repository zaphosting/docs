## 介绍

在 **Rust** 中，服务器清档会重置游戏世界的部分内容，为所有玩家创造一个全新的开始。随着时间推移，Rust 地图会被大量基地、积累的资源和稳定的玩家团队占据，这会影响服务器的平衡和性能。

服务器清档会清除现有进度，恢复一个干净的环境，让玩家可以从零开始。许多 Rust 社区会定期进行清档，以保持游戏的公平性和刺激感，尤其是在新更新或新地图发布时。

根据清档类型，可能会重置**地图**、**玩家蓝图**或**两者**。

## 服务器清档的作用

Rust 服务器清档会删除存储的世界或玩家进度数据，包括：

- 玩家建造的建筑和基地  
- 玩家背包和存储物品  
- 地图探索数据  
- 已学会的蓝图（视清档类型而定）

清档后，服务器就像一个全新的世界，玩家需要重新收集资源、重建基地并重新学习进度。



## Rust 清档类型

Rust 支持多种清档类型，取决于你想重置的内容。以下是可用清档类型的说明。

### 地图清档

**地图清档**会删除整个世界存档文件。所有基地、建筑和地图进度都会被清除。服务器重启时，会生成或下载一个新地图。蓝图进度保持不变，除非同时执行蓝图清档。

### 蓝图清档

**蓝图清档**会重置所有已学会的制作蓝图。玩家必须重新研究物品才能制作。此类清档只影响进度，不会重置地图本身。

### 全部清档

**全部清档**会同时重置地图和所有玩家蓝图。这会创建一个完全全新的服务器环境，通常用于赛季重置或重大更新。



## 为什么服务器清档很重要

服务器清档有助于维护健康的 Rust 服务器环境。随着时间推移，服务器会积累大量建筑、储物箱和世界实体，这会对服务器性能和游戏平衡产生负面影响。定期清档带来多重好处：

- 恢复服务器性能  
- 为所有玩家创造全新开始  
- 防止强势战队长期控制地图  
- 允许服务器运行新地图或新地图种子

许多 Rust 社区会根据服务器风格，每周、每两周或每月定期进行清档。



## 如何执行服务器清档

要对你的 Rust 服务器执行清档，请通过 [FTP 访问](gameserver-ftpaccess.md) 进入服务器文件。接着，在 Rust 服务器目录中找到存档文件。通常包括：

```
proceduralmap.*
player.blueprints.*
player.deaths.*
player.identities.*
player.states.*
```
删除特定文件决定了执行的清档类型。删除 `proceduralmap.*` 文件会执行**地图清档**，清除现有世界。删除 `player.blueprints.*` 文件会执行**蓝图清档**。

![img](https://screensaver01.zap-hosting.com/index.php/s/QdeoFWbaaQz7Yaf/preview)

同时删除两者则会执行**全部清档**。删除所需文件后，重新启动服务器。服务器会自动生成新世界，玩家可以重新开始。
