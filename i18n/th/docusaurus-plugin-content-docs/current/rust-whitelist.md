## แนะนำ

Whitelist คือกลไกควบคุมการเข้าถึงที่จำกัดว่าใครสามารถเข้าร่วมเซิร์ฟเวอร์ของคุณได้ เซิร์ฟเวอร์เฉพาะ Rust ไม่มีฟีเจอร์ whitelist ในตัว ดังนั้นวิธีที่ผู้ให้บริการเช่าเซิร์ฟเวอร์ส่วนใหญ่ใช้คือการติดตั้ง uMod Oxide พร้อมปลั๊กอิน whitelist ที่บล็อกการเชื่อมต่อเว้นแต่ผู้เล่นจะได้รับอนุญาตอย่างชัดเจน

## การเปิดใช้งาน Whitelist

เชื่อมต่อกับเซิร์ฟเวอร์ของคุณผ่าน FTP และติดตั้ง uMod Oxide หากยังไม่มี หลังจากติดตั้ง uMod แล้ว ให้อัปโหลดไฟล์ปลั๊กอิน whitelist ชื่อ `Whitelist.cs` ไปยังโฟลเดอร์ปลั๊กอิน ซึ่งโดยทั่วไปจะอยู่ที่ `oxide/plugins/Whitelist.cs`

เมื่อปลั๊กอินถูกติดตั้งแล้ว ให้รีสตาร์ทเซิร์ฟเวอร์หรือโหลดปลั๊กอินใหม่เพื่อให้ปลั๊กอินทำงาน ปลั๊กอิน whitelist โดยทั่วไปจะทำงานโดยการกำหนดสิทธิ์เฉพาะก่อนที่ผู้เล่นจะสามารถเชื่อมต่อได้

## การจัดการผู้เล่นที่อยู่ใน Whitelist

เปิด Live Console ในแดชบอร์ดการจัดการเซิร์ฟเวอร์เกมของ ZAP-Hosting และมอบสิทธิ์ whitelist ให้กับผู้เล่นโดยใช้ SteamID64 ของพวกเขา

```text
oxide.grant user 76561198000000000 whitelist.allow
```

ถ้าต้องการยกเลิกการเข้าถึง ให้ลบสิทธิ์นั้นออก

```text
oxide.revoke user 76561198000000000 whitelist.allow
```

ถ้าต้องการตรวจสอบว่าสิทธิ์ของผู้ใช้มีอะไรบ้าง ให้เรียกดูสิทธิ์ของพวกเขา

```text
oxide.show user 76561198000000000
```

## การตรวจสอบการทำงานของ Whitelist

ลองเชื่อมต่อด้วยบัญชีที่ไม่มีสิทธิ์ whitelist ปลั๊กอินควรปฏิเสธการเชื่อมต่อ จากนั้นลองเชื่อมต่อด้วยบัญชีที่คุณมอบ `whitelist.allow` เพื่อยืนยันการเข้าถึง

ถ้าทุกคนยังสามารถเข้าร่วมได้ ให้ตรวจสอบว่าปลั๊กอินถูกโหลดและ uMod กำลังทำงานอยู่ การรีสตาร์ทเซิร์ฟเวอร์หลังจากอัปโหลดปลั๊กอินเป็นวิธีที่น่าเชื่อถือที่สุดเพื่อให้แน่ใจว่าปลั๊กอินทำงาน

## สรุป

ถ้าทำตามขั้นตอนทั้งหมดข้างต้นถูกต้องแล้ว whitelist ของคุณจะเปิดใช้งานและคุณสามารถควบคุมได้อย่างแม่นยำว่าใครสามารถเข้าร่วมเซิร์ฟเวอร์ได้ หากการเข้าถึงยังไม่ทำงานตามที่คาดไว้ ให้รีสตาร์ทเซิร์ฟเวอร์อีกครั้งและตรวจสอบไฟล์หรือผลลัพธ์คำสั่งเพื่อยืนยันว่าการเปลี่ยนแปลงถูกนำไปใช้แล้ว

ถ้ามีคำถามเพิ่มเติมหรือต้องการความช่วยเหลือ อย่าลังเลที่จะติดต่อทีมซัพพอร์ตของเรา ซึ่งพร้อมให้บริการทุกวันเพื่อช่วยเหลือคุณ! 🙂
