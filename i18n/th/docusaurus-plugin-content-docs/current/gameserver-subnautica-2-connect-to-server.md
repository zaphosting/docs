## Introduction
ไม่แน่ใจว่าจะเชื่อมต่อกับเซิร์ฟเวอร์ **Subnautica 2** ของคุณอย่างไรหรือจำเป็นต้องใช้ข้อมูลอะไรบ้าง? คู่มือนี้จะแสดงวิธีรับรายละเอียดเซิร์ฟเวอร์ที่จำเป็น สร้างการเชื่อมต่อ และแก้ไขปัญหาการเชื่อมต่อที่พบบ่อย

:::info แจ้งเตือน Early Access
Subnautica 2 อาจยังอยู่ในช่วงพัฒนาตามเวอร์ชันที่ปล่อยออกมา ดังนั้นฟีเจอร์มัลติเพลเยอร์ พฤติกรรมของตัวเลือกเซิร์ฟเวอร์ และขั้นตอนการเชื่อมต่ออาจมีการเปลี่ยนแปลงได้ตลอดเวลา
:::

## รับรายละเอียดเซิร์ฟเวอร์

คุณจะต้องมีข้อมูลทั้งหมดที่จำเป็นสำหรับการเชื่อมต่อกับเซิร์ฟเวอร์ของคุณ คุณสามารถดูรายละเอียดเซิร์ฟเวอร์ได้จากแดชบอร์ดในแผงควบคุมเว็บของเซิร์ฟเวอร์เกม ข้อมูลสำคัญได้แก่:

- ที่อยู่ IP และพอร์ต
- ชื่อเซิร์ฟเวอร์
- รหัสผ่านเซิร์ฟเวอร์

โดยใช้ข้อมูลข้างต้น คุณจะสามารถระบุเซิร์ฟเวอร์ของคุณและมั่นใจได้ว่าคุณเชื่อมต่อกับเซิร์ฟเวอร์ที่ถูกต้อง

| รายละเอียด | คำอธิบาย |
| --- | --- |
| `ที่อยู่ IP` | ที่อยู่สาธารณะของเซิร์ฟเวอร์ Subnautica 2 ของคุณ |
| `พอร์ต` | พอร์ตเกมที่ใช้สำหรับการเชื่อมต่อ |
| `ชื่อเซิร์ฟเวอร์` | ชื่อที่แสดงในเกมหากมีรายการเซิร์ฟเวอร์หรือเบราว์เซอร์เซสชัน |
| `รหัสผ่านเซิร์ฟเวอร์` | จำเป็นเฉพาะเมื่อเซิร์ฟเวอร์ของคุณตั้งรหัสผ่านไว้ |

## สร้างการเชื่อมต่อกับเซิร์ฟเวอร์

วิธีเชื่อมต่อกับเซิร์ฟเวอร์ **Subnautica 2** จะแตกต่างกันไปตามตัวเลือกมัลติเพลเยอร์ที่มีในเวอร์ชันเกมที่คุณใช้ โดยทั่วไปคุณจะต้องใช้ **ที่อยู่ IP** และ **พอร์ต** ของเซิร์ฟเวอร์ และในบางกรณีอาจต้องใช้ **รหัสผ่านเซิร์ฟเวอร์** ด้วย

### เข้าร่วมผ่านเมนูมัลติเพลเยอร์ในเกม

เริ่มเกม **Subnautica 2** และเปิดส่วนมัลติเพลเยอร์หรือเข้าร่วมเซิร์ฟเวอร์จากเมนูหลัก หากเกมมีตัวเลือกเชื่อมต่อโดยตรง ให้กรอก `ที่อยู่ IP` และ `พอร์ต` ของเซิร์ฟเวอร์ตามที่แสดงในแผงควบคุมเว็บของ ZAP-Hosting หากตั้งรหัสผ่านไว้ ให้กรอกรหัสผ่านเมื่อระบบถาม

หลังจากยืนยันข้อมูล เกมจะพยายามเชื่อมต่อกับเซิร์ฟเวอร์ของคุณ

:::note ตัวเลือกเชื่อมต่อโดยตรง
คำศัพท์ในเมนูและปุ่มอาจแตกต่างกันตามเวอร์ชันเกม หากคุณไม่เห็นตัวเลือกเชื่อมต่อโดยตรง ให้ตรวจสอบว่าเกมต้องเข้าร่วมผ่านรายการเซิร์ฟเวอร์ ระบบเชิญ หรือเบราว์เซอร์เซสชันแทนหรือไม่
:::

### เข้าร่วมผ่านรายการเซิร์ฟเวอร์หรือเบราว์เซอร์เซสชัน

ถ้า **Subnautica 2** ใช้เบราว์เซอร์เซิร์ฟเวอร์ในเกมแทนช่องกรอกเชื่อมต่อโดยตรง ให้ค้นหาเซิร์ฟเวอร์ของคุณโดยใช้ `ชื่อเซิร์ฟเวอร์` ที่ตั้งไว้ นี่คือเหตุผลที่ควรตรวจสอบชื่อเซิร์ฟเวอร์ในแผงควบคุมเว็บก่อนเชื่อมต่อ

เมื่อเจอเซิร์ฟเวอร์ที่ถูกต้องในรายการ:

1. เลือกเซิร์ฟเวอร์นั้น
2. ตรวจสอบว่าชื่อที่แสดงตรงกับเซิร์ฟเวอร์ของคุณ
3. เข้าร่วมเซิร์ฟเวอร์
4. กรอกรหัสผ่านหากระบบร้องขอ

:::tip ใช้ชื่อเซิร์ฟเวอร์ที่ไม่ซ้ำกัน
ถ้าเซิร์ฟเวอร์ของคุณหายากหรือไม่ชัดเจนในรายการ ให้ตั้งชื่อเซิร์ฟเวอร์ที่ชัดเจนและไม่ซ้ำกันในคอนฟิกเซิร์ฟเวอร์ เพื่อให้ง่ายต่อการค้นหาและหลีกเลี่ยงการเชื่อมต่อผิดเซิร์ฟเวอร์
:::

### ถ้าเซิร์ฟเวอร์ไม่แสดงทันที

ถ้าเซิร์ฟเวอร์ของคุณไม่แสดงในเบราว์เซอร์ในเกม ไม่ได้หมายความว่าเซิร์ฟเวอร์ออฟไลน์เสมอไป บางครั้งรายการเซิร์ฟเวอร์อาจใช้เวลาปรับปรุง หรือเกมอาจแสดงเซิร์ฟเวอร์หลังจากเริ่มต้นสำเร็จแล้วเท่านั้น

คุณควรตรวจสอบดังนี้:

- เซิร์ฟเวอร์ออนไลน์ในแผงควบคุมเว็บของ ZAP-Hosting
- ใช้พอร์ตเกมที่ถูกต้อง
- เซิร์ฟเวอร์เริ่มทำงานเต็มที่แล้ว
- ชื่อเซิร์ฟเวอร์ตั้งค่าอย่างถูกต้อง
- กรอกรหัสผ่านถ้าจำเป็น

## ปัญหาที่อาจเกิดขึ้นและวิธีแก้ไข

การเชื่อมต่อกับเซิร์ฟเวอร์เกมอาจได้รับผลกระทบจากหลายปัจจัย ปัญหาที่พบบ่อยและวิธีแก้ไขมีดังนี้

### ไม่เห็นเซิร์ฟเวอร์

เซิร์ฟเวอร์ไม่แสดงอาจเกิดจากการเริ่มต้นไม่สมบูรณ์ เช่น การตั้งค่าผิดพลาดหรือไฟล์เสียหาย ข้อมูลเพิ่มเติมมักจะพบได้ในคอนโซลเซิร์ฟเวอร์หรือไฟล์ล็อก

:::caution ต้องเริ่มเซิร์ฟเวอร์ก่อน
อย่าพยายามเชื่อมต่อก่อนที่เซิร์ฟเวอร์จะเริ่มทำงานเสร็จสมบูรณ์ หากเซิร์ฟเวอร์กำลังโหลดหรือรีสตาร์ท ผู้เล่นอาจเชื่อมต่อไม่ได้หรือเซิร์ฟเวอร์อาจยังไม่แสดงในรายการใดๆ
:::

### รายละเอียดเชื่อมต่อผิด

ถ้าเชื่อมต่อไม่ได้ ให้ตรวจสอบว่าคุณใช้ `ที่อยู่ IP` `พอร์ต` และ `รหัสผ่านเซิร์ฟเวอร์` ถูกต้อง แม้แต่การพิมพ์ผิดเล็กน้อยก็อาจทำให้เชื่อมต่อไม่สำเร็จ

| ตรวจสอบ | สิ่งที่ต้องยืนยัน |
| --- | --- |
| `ที่อยู่ IP` | ตรงกับที่แสดงในแดชบอร์ดเซิร์ฟเวอร์ของคุณ |
| `พอร์ต` | ตรงกับพอร์ตเกมที่กำหนดไว้ |
| `รหัสผ่าน` | กรอกตามที่ตั้งไว้เป๊ะ รวมตัวพิมพ์ใหญ่และเล็ก |
| `ชื่อเซิร์ฟเวอร์` | ตรงกับชื่อเซิร์ฟเวอร์ที่ตั้งไว้ถ้าค้นหาในเบราว์เซอร์ |

### เซิร์ฟเวอร์ออนไลน์แต่เชื่อมต่อไม่สำเร็จ

ถ้าเซิร์ฟเวอร์ดูเหมือนออนไลน์แต่ยังเชื่อมต่อไม่ได้ ให้ตรวจสอบคอนโซลและล็อกในแผงควบคุมเว็บ ข้อผิดพลาดในนั้นอาจบอกว่าเซิร์ฟเวอร์โหลดไม่สมบูรณ์หรือมีปัญหาการตั้งค่าที่บล็อกการเชื่อมต่อผู้เล่น

:::tip ตรวจสอบล็อกก่อน
คอนโซลเซิร์ฟเวอร์และไฟล์ล็อกมักเป็นวิธีที่เร็วที่สุดในการหาสาเหตุที่เชื่อมต่อไม่สำเร็จ ช่วยยืนยันว่าเซิร์ฟเวอร์รับฟังการเชื่อมต่อและการพยายามเข้าร่วมถึงเซิร์ฟเวอร์หรือไม่
:::

### ไม่มีวิธีแก้ไขหรือไม่ช่วย

ยังมีปัญหาอยู่หลังจากลองแก้ไขแล้ว? ทีมซัพพอร์ตของเราพร้อมช่วยเหลือทุกวัน เพียงสร้าง **[ตั๋ว](https://zap-hosting.com/en/customer/support/)** บนเว็บไซต์ของเราและอธิบายปัญหาของคุณให้ละเอียดที่สุด รวมที่อยู่ IP เซิร์ฟเวอร์ พอร์ต และคำอธิบายสั้นๆ ว่าเกิดอะไรขึ้นตอนพยายามเชื่อมต่อ
