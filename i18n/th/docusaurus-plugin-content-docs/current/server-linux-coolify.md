## แนะนำ

[Coolify](https://coolify.io/) คือทางเลือกโอเพนซอร์สสำหรับ Heroku และ Vercel ที่เซลฟ์โฮสต์ได้เอง ช่วยให้คุณจัดการเซิร์ฟเวอร์, แอปพลิเคชัน และฐานข้อมูลได้ง่ายมาก ๆ

![img](https://screensaver01.zap-hosting.com/index.php/s/nrXeZsgjXn43sfw/preview)

แดชบอร์ดของ Coolify ใช้งานง่าย ทำให้การเซลฟ์โฮสต์เป็นเรื่องง่าย คุณจะได้ควบคุมโครงสร้างพื้นฐานเต็มที่, ดีพลอยอัตโนมัติผ่าน GitHub และช่วยลดค่าใช้จ่ายคลาวด์ของคุณ

ในคู่มือนี้ เราจะพาคุณผ่านทุกขั้นตอน ตั้งค่าบัญชีแอดมินล่วงหน้า และคอนฟิกทุกอย่างที่คุณต้องรู้

## กรณีการใช้งานของ Coolify

Coolify ใช้ได้หลากหลายสถานการณ์ในชีวิตประจำวัน เหมาะกับทุกคนที่อยากอัตโนมัติดีพลอยแอป, โฮสต์ฐานข้อมูล หรือรัน background workers Coolify เหมาะทั้งมือใหม่และมือโปร

Coolify รองรับการดีพลอย Static Sites, Node.js, Python, PHP, Rust และ Docker containers ในตัว พร้อมเทมเพลตคลิกเดียวยอดนิยมอย่าง WordPress, Ghost, Grafana, Appwrite, Nextcloud และฐานข้อมูล PostgreSQL มันจัดการใบรับรอง SSL จาก Let's Encrypt และการตั้งค่า reverse proxy ผ่าน Traefik ให้อัตโนมัติ

## การเตรียมตัว

แม้ Coolify จะเบาและเรียบง่าย แต่การใช้ทรัพยากรอาจเพิ่มขึ้นมากตามจำนวนแอปที่ดีพลอย, การสร้าง Docker และฐานข้อมูลที่รันพร้อมกัน เราแนะนำสเปคฮาร์ดแวร์ดังนี้สำหรับโฮสต์ Coolify บน VPS/เซิร์ฟเวอร์เฉพาะของคุณ

| ฮาร์ดแวร์   | ขั้นต่ำ       | แนะนำ                      |
| ---------- | ------------ | -------------------------- |
| CPU        | 2 คอร์ vCPU  | 4 คอร์ vCPU                |
| RAM        | 2 GB         | 4 GB+                      |
| พื้นที่ดิสก์ | 30 GB        | 50 GB+                     |

<InlineServiceLink />

ก่อนติดตั้ง Coolify คุณต้องเชื่อมต่อกับเซิร์ฟเวอร์และอัปเดตระบบปฏิบัติการให้เป็นเวอร์ชันล่าสุด ต้องเชื่อมต่อผ่าน SSH ด้วยสิทธิ์ root เรามีคู่มือ [SSH Initial access](vserver-linux-ssh.md) ถ้าคุณต้องการความช่วยเหลือในขั้นตอนนี้

:::tip[การยืนยันตัวตนด้วย SSH Key]
เรา **แนะนำอย่างยิ่ง** ให้ตั้งค่า SSH Key แทนการใช้รหัสผ่านสำหรับการยืนยันตัวตน ดูวิธีทำได้ในคู่มือ [SSH Key Setup](vserver-linux-sshkey.md)
:::

## การติดตั้ง

หลังจากเชื่อมต่อกับเซิร์ฟเวอร์แล้ว ให้อัปเดตแพ็กเกจระบบและติดตั้ง `curl` ซึ่งจำเป็นสำหรับดึงสคริปต์ติดตั้ง Coolify:

```bash
sudo apt update && sudo apt upgrade -y && sudo apt install curl -y
```

### เตรียมข้อมูลบัญชีแอดมินของคุณ

Coolify มีนโยบายความปลอดภัยเข้มงวดสำหรับบัญชีแอดมิน ก่อนรันคำสั่งติดตั้ง ให้เตรียมข้อมูลบัญชี root ของ Coolify ตามข้อกำหนดตรวจสอบดังนี้:

* **ชื่อผู้ใช้**
  * ความยาวขั้นต่ำ: 3 ตัวอักษร
  * ความยาวสูงสุด: 255 ตัวอักษร
  * ใช้ได้เฉพาะตัวอักษร, ตัวเลข, เว้นวรรค, ขีดล่าง และขีดกลาง
* **อีเมล**
  * ต้องเป็นอีเมลที่ถูกต้อง
  * ต้องมีระเบียน DNS ที่ถูกต้อง
  * ความยาวสูงสุด: 255 ตัวอักษร
* **รหัสผ่าน**
  * ความยาวขั้นต่ำ: 8 ตัวอักษร
  * ต้องมีตัวอักษรพิมพ์ใหญ่และพิมพ์เล็ก
  * ต้องมีตัวเลขอย่างน้อยหนึ่งตัว
  * ต้องมีสัญลักษณ์พิเศษอย่างน้อยหนึ่งตัว
  * ห้ามใช้รหัสผ่านที่ใช้บ่อยหรือเคยถูกแฮก

### รันสคริปต์ติดตั้ง

Coolify มีสคริปต์ติดตั้งอัตโนมัติที่รวดเร็ว เพื่อความปลอดภัย เราจะใช้ environment variables กำหนดบัญชี root แอดมินล่วงหน้า ป้องกันการเข้าถึงโดยไม่ได้รับอนุญาตในระหว่างการตั้งค่า

รันคำสั่งนี้โดยเปลี่ยนชื่อผู้ใช้, อีเมล และรหัสผ่านเป็นข้อมูลของคุณเอง:

```bash
env ROOT_USERNAME=zaphosting ROOT_USER_EMAIL=info@zap-hosting.com ROOT_USER_PASSWORD=zaphosting bash -c 'curl -fsSL https://cdn.coollabs.io/coolify/install.sh | bash'
```
:::note[โอเพนซอร์ส]
คุณสามารถดูซอร์สโค้ดสคริปต์ติดตั้งได้ที่ [Official Coolify Repository](https://github.com/coollabsio/coolify/blob/v4.x/scripts/install.sh)
:::

สคริปต์จะติดตั้ง Docker, ตั้งค่าเครือข่ายที่จำเป็น และคอนฟิกคอนเทนเนอร์ Coolify อัตโนมัติ ใช้เวลาสักครู่ขึ้นอยู่กับฮาร์ดแวร์เซิร์ฟเวอร์ของคุณ

![img](https://screensaver01.zap-hosting.com/index.php/s/Zx5DXzEH2MmrkrX/preview)

### การเข้าถึง Coolify

หลังติดตั้งเสร็จ ขั้นตอนถัดไปคือเข้าถึงแดชบอร์ดผ่านเว็บเบราว์เซอร์ พิมพ์ที่อยู่ IP เซิร์ฟเวอร์ของคุณตามด้วยพอร์ต 8000: `http://[your_server_ip]:8000`

เพราะคุณใส่ข้อมูลบัญชีไว้ตอนติดตั้งแล้ว คุณจะเจอหน้าล็อกอินเลย ไม่ใช่หน้าตั้งค่าเริ่มต้น

ใส่อีเมลและรหัสผ่านที่ใช้ในคำสั่งติดตั้งเพื่อเข้าสู่แดชบอร์ด

### ตั้งค่าระเบียน DNS

หลังล็อกอินเข้าดashboard Coolify สิ่งแรกที่ควรทำคือเชื่อมต่อโดเมน การเข้าถึง Coolify ผ่าน IP (`http://[your_server_ip]:8000`) ไม่ปลอดภัย

การเชื่อมต่อโดเมนช่วยให้ Coolify สร้างใบรับรอง SSL ฟรีให้อัตโนมัติสำหรับแดชบอร์ดและแอปในอนาคตทั้งหมด

ไปที่หน้าจัดการ DNS ของผู้ให้บริการโดเมนคุณ แล้วเพิ่มระเบียน `A` สองรายการชี้ไปที่เซิร์ฟเวอร์ของคุณ ถ้าโดเมนจดทะเบียนกับ ZAP-Hosting คุณสามารถดูคู่มือ [Domain Records](domain-records.md) เพื่อเรียนรู้วิธีเพิ่มระเบียนเหล่านี้ง่าย ๆ

| ชื่อ / โฮสต์ | ประเภท | ค่า                 | TTL           |
| ------------ | ------ | ------------------- | ------------- |
| `@`          | A      | `[your_server_ip]`  | อัตโนมัติ / ต่ำสุด |
| `*`          | A      | `[your_server_ip]`  | อัตโนมัติ / ต่ำสุด |

:::info[การตั้งค่า DNS Routing]
* ระเบียน `@` ช่วยให้ Coolify ใช้โดเมนหลักของคุณได้
* ระเบียน `*` (wildcard) ช่วยให้ Coolify จัดการเส้นทางซับโดเมนใหม่ ๆ อัตโนมัติ (เช่น `api.[your_domain]`) โดยไม่ต้องเพิ่มระเบียน DNS ทีละโปรเจกต์
:::

### ตั้งค่าโดเมนของอินสแตนซ์

เพื่อความปลอดภัยของอินสแตนซ์ Coolify ด้วยใบรับรอง SSL ให้กำหนดโดเมนสำหรับมัน ไปที่ **Settings** ในแถบเมนูซ้าย แล้วหา **Instance Domain** ใส่โดเมนของคุณพร้อม prefix `https://` (เช่น `https://coolify.[your_domain]`) แล้วกด **Save** เพื่อบันทึก

## ดีพลอยแอปแรกของคุณ

หลังจากนั้น คุณจะเข้าไปที่แท็บ **Projects** และเริ่มดีพลอยแอปแรกได้เลย!

คลิก **+ Add Resource** เพื่อเพิ่ม public repository หรือไฟล์ Docker Compose หรือเลือกจากบริการที่ตั้งค่าล่วงหน้ามากมายในไลบรารีเทมเพลตคลิกเดียว Coolify ของคุณพร้อมใช้งานแล้ว!
