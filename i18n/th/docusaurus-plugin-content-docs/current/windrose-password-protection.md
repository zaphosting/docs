## Introduction

Windrose รองรับการป้องกันด้วยรหัสผ่าน ซึ่งช่วยให้คุณจำกัดการเข้าถึงเซิร์ฟเวอร์เฉพาะของคุณด้วยรหัสผ่านสำหรับเข้าร่วม ในคู่มือนี้ คุณจะได้เรียนรู้วิธีเปิดใช้งาน เปลี่ยน หรือเอารหัสผ่านออกบนเซิร์ฟเวอร์ Windrose ของคุณผ่านการจัดการเซิร์ฟเวอร์เกมของ ZAP-Hosting

## Preparation

ก่อนเริ่มต้น ให้แน่ใจว่าคุณสามารถเข้าถึงเซิร์ฟเวอร์ Windrose ของคุณในเว็บอินเทอร์เฟซของ ZAP-Hosting และมีสิทธิ์แก้ไขไฟล์เซิร์ฟเวอร์

:::info สิทธิ์ที่ต้องมี
คุณต้องเข้าถึงแดชบอร์ดเซิร์ฟเวอร์และพื้นที่จัดการไฟล์เพื่อแก้ไขไฟล์ `ServerDescription.json`
:::

## ทำความเข้าใจกับการป้องกันด้วยรหัสผ่านของ Windrose

การตั้งรหัสผ่านช่วยป้องกันไม่ให้ผู้เล่นที่ไม่ได้รับอนุญาตเข้าร่วมเซิร์ฟเวอร์เกม Windrose ของคุณ ผู้เล่นยังสามารถเข้าถึงเซิร์ฟเวอร์ผ่านโค้ดเชิญได้ แต่ต้องใส่รหัสผ่านที่ถูกต้องก่อนจึงจะเชื่อมต่อได้

สิ่งนี้มีประโยชน์ถ้าคุณต้องการ:
- จำกัดการเข้าถึงเฉพาะเพื่อน พนักงาน หรือกลุ่มส่วนตัว
- ปกป้องเซิร์ฟเวอร์ทดสอบจากการเข้าถึงสาธารณะ
- ควบคุมผู้ที่สามารถเข้าร่วมโดยไม่ต้องพึ่งพาระบบจัดการรหัสผ่านแยกต่างหากหรือรหัสผ่านบัญชี

:::note พฤติกรรมของรหัสผ่าน
Windrose ใช้รหัสผ่านที่ตั้งค่าในไฟล์เซิร์ฟเวอร์เอง ซึ่งแยกจากรหัสผ่านบัญชี ZAP-Hosting ของคุณและไม่เกี่ยวข้องกับเครื่องมืออย่างตัวจัดการรหัสผ่าน การผสานรวม Google Password Manager หรือรายการรหัสผ่านใดๆ
:::

## เปิดใช้งานการป้องกันด้วยรหัสผ่าน

เพื่อใช้รหัสผ่าน คุณต้องหยุดเซิร์ฟเวอร์ก่อน แก้ไขไฟล์การตั้งค่า แล้วเริ่มเซิร์ฟเวอร์ใหม่อีกครั้ง

### หยุดเซิร์ฟเวอร์

เปิดแดชบอร์ดเซิร์ฟเวอร์ของคุณในเว็บอินเทอร์เฟซ ZAP-Hosting แล้วหยุดเซิร์ฟเวอร์อย่างสมบูรณ์ก่อนทำการเปลี่ยนแปลง

:::caution หยุดเซิร์ฟเวอร์ก่อนแก้ไข
คุณควรหยุดเซิร์ฟเวอร์ก่อนแก้ไขไฟล์ `ServerDescription.json` เพื่อหลีกเลี่ยงความขัดแย้งของการตั้งค่าหรือไฟล์ถูกเขียนทับขณะเซิร์ฟเวอร์กำลังทำงาน
:::

### เปิดไฟล์การตั้งค่า

ในแถบด้านข้างของแผงควบคุม ให้เปิดเมนู `Files`

จากนั้นค้นหาและเปิดไฟล์ `ServerDescription.json` ในไดเรกทอรีหลักของรายการไฟล์เซิร์ฟเวอร์ของคุณ

### แก้ไขการตั้งค่ารหัสผ่าน

ในไฟล์ `ServerDescription.json` ตั้งค่า `IsPasswordProtected` เป็น `true` และกำหนดรหัสผ่านที่คุณต้องการในช่อง `Password`

```json
"IsPasswordProtected": true,
"Password": "YourPasswordHere"
```

แทนที่ `YourPasswordHere` ด้วยรหัสผ่านที่ปลอดภัยของคุณเอง

:::tip เลือกรหัสผ่านที่แข็งแรง
ใช้รหัสผ่านที่ไม่ซ้ำกับรหัสผ่านบัญชีหรือบริการอื่นๆ เพื่อช่วยปกป้องเซิร์ฟเวอร์ Windrose ของคุณจากการเข้าถึงที่ไม่ต้องการ
:::

### บันทึกและเริ่มเซิร์ฟเวอร์ใหม่

หลังจากแก้ไขไฟล์แล้ว ให้คลิก `Save Content`

เมื่อไฟล์ถูกบันทึกแล้ว ให้เริ่มเซิร์ฟเวอร์ของคุณอีกครั้งจากแดชบอร์ด เซิร์ฟเวอร์ Windrose ของคุณจะต้องใช้รหัสผ่านที่ตั้งค่าไว้เมื่อผู้เล่นพยายามเข้าร่วม

## อ้างอิงการตั้งค่า

ตารางด้านล่างอธิบายการตั้งค่าที่เกี่ยวข้องในไฟล์ `ServerDescription.json`:

| คีย์ | ประเภท | ตัวอย่าง | คำอธิบาย |
| --- | --- | --- | --- |
| `IsPasswordProtected` | Boolean | `true` | เปิดหรือปิดการป้องกันด้วยรหัสผ่าน |
| `Password` | String | `"YourPasswordHere"` | กำหนดรหัสผ่านสำหรับเข้าร่วมเมื่อเปิดใช้งานการป้องกันด้วยรหัสผ่าน |

## เปลี่ยนรหัสผ่านที่มีอยู่

ถ้าคุณต้องการเปลี่ยนรหัสผ่านในภายหลัง ขั้นตอนจะเหมือนกับการตั้งค่าเริ่มต้น

### อัปเดตรหัสผ่าน

หยุดเซิร์ฟเวอร์ เปิดไฟล์ `ServerDescription.json` แล้วเปลี่ยนค่าที่ช่อง `Password` เป็นรหัสผ่านใหม่ของคุณ

ตัวอย่าง:

```json
"IsPasswordProtected": true,
"Password": "[your_new_password]"
```

ตัวแทน `[your_new_password]` คือรหัสผ่านใหม่ที่คุณต้องการให้ผู้เล่นใช้เมื่อเข้าร่วมเซิร์ฟเวอร์

### บันทึกและเริ่มเซิร์ฟเวอร์ใหม่อีกครั้ง

บันทึกไฟล์และเริ่มเซิร์ฟเวอร์ใหม่เพื่อให้การเปลี่ยนแปลงรหัสผ่านมีผล

:::note ตรวจสอบรหัสผ่าน
ถ้าผู้เล่นไม่สามารถเข้าร่วมหลังจากเปลี่ยนรหัสผ่าน ให้ขอให้พวกเขาตรวจสอบการใส่รหัสผ่านอย่างละเอียด รวมถึงตัวพิมพ์ใหญ่และพิมพ์เล็กถ้ามี
:::

## เอาการป้องกันด้วยรหัสผ่านออก

ถ้าคุณไม่ต้องการป้องกันเซิร์ฟเวอร์ Windrose ด้วยรหัสผ่านอีกต่อไป คุณสามารถปิดฟีเจอร์นี้ในไฟล์การตั้งค่าเดียวกัน

### ปิดการตั้งค่า

หยุดเซิร์ฟเวอร์และเปิดไฟล์ `ServerDescription.json` อีกครั้ง

ตั้งค่า `IsPasswordProtected` เป็น `false` ค่าของ `Password` จะถูกละเว้นเมื่อปิดใช้งานตัวเลือกนี้

```json
"IsPasswordProtected": false,
"Password": ""
```

### บันทึกและเริ่มเซิร์ฟเวอร์ใหม่

คลิก `Save Content` แล้วเริ่มเซิร์ฟเวอร์อีกครั้ง ผู้เล่นจะสามารถเข้าร่วมได้โดยไม่ต้องใส่รหัสผ่าน

## การแก้ไขปัญหา

ถ้าการป้องกันด้วยรหัสผ่านไม่ทำงานตามที่คาดไว้ ให้ตรวจสอบประเด็นต่อไปนี้

| ปัญหา | สาเหตุที่เป็นไปได้ | วิธีแก้ไข |
| --- | --- | --- |
| เซิร์ฟเวอร์ไม่ถามรหัสผ่าน | `IsPasswordProtected` ยังตั้งเป็น `false` | ตั้งเป็น `true` บันทึกไฟล์ และเริ่มเซิร์ฟเวอร์ใหม่ |
| ผู้เล่นไม่สามารถเข้าร่วมด้วยรหัสผ่านที่ถูกต้อง | รหัสผ่านถูกใส่ผิด | ตรวจสอบค่าที่ตั้งไว้และขอให้ผู้เล่นตรวจสอบการใส่รหัสผ่านอย่างละเอียด |
| การเปลี่ยนแปลงไม่ถูกนำไปใช้ | เซิร์ฟเวอร์ไม่ได้ถูกรีสตาร์ทหลังแก้ไข | รีสตาร์ทเซิร์ฟเวอร์หลังบันทึกไฟล์ `ServerDescription.json` |
| การเปลี่ยนแปลงไฟล์หายไป | แก้ไขไฟล์ขณะเซิร์ฟเวอร์กำลังทำงาน | หยุดเซิร์ฟเวอร์ก่อน แล้วทำการเปลี่ยนแปลงใหม่ บันทึกและรีสตาร์ท |

:::danger หลีกเลี่ยงการเก็บรหัสผ่านสำคัญในที่สาธารณะ
อย่าแชร์รหัสผ่านเซิร์ฟเวอร์ของคุณในโพสต์ชุมชน สกรีนช็อต หรือการส่งออกไฟล์สาธารณะ และหลีกเลี่ยงการใช้รหัสผ่านจากรายการรหัสผ่านเก่าหรือใช้ซ้ำกับข้อมูลรับรองจากบริการอื่น
:::

## Conclusion

ยินดีด้วย คุณได้ตั้งค่าการป้องกันด้วยรหัสผ่านบนเซิร์ฟเวอร์ Windrose ของคุณเรียบร้อยแล้ว สำหรับคำถามหรือความช่วยเหลือเพิ่มเติม กรุณาติดต่อทีมซัพพอร์ตของเรา ซึ่งพร้อมให้บริการทุกวันเพื่อช่วยเหลือคุณ! 🙂
