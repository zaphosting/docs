## บทนำ

คุณสามารถส่งต่อโดเมนทั้งหมดหรือแค่ซับโดเมนไปยังเซิร์ฟเวอร์ TeamSpeak 3 ของคุณได้
ด้วยวิธีนี้ใครๆ ก็ไม่ต้องจำที่อยู่ IP ที่ซับซ้อนซึ่งประกอบด้วยตัวเลข และสามารถเชื่อมต่อกับเซิร์ฟเวอร์ TeamSpeak 3 ของคุณได้ง่ายๆ ผ่านโดเมน

:::note
โดยค่าเริ่มต้น เซิร์ฟเวอร์ TeamSpeak 3 **ใหม่** จะได้รับนามแฝง `.zap.cloud` พร้อมพรีฟิกซ์สุ่มที่สามารถใช้เชื่อมต่อได้ทันที
:::

## การใช้การตั้งค่า TSDNS แบบด่วน

เราได้พัฒนาระบบ TSDNS ใหม่สำหรับเซิร์ฟเวอร์ TeamSpeak 3 ของเรา ซึ่งช่วยให้คุณตั้งค่าซับโดเมนสำหรับเซิร์ฟเวอร์ TeamSpeak 3 ของคุณได้ในไม่กี่คลิก! หากคุณยังไม่มีโดเมนกับเรา ให้ข้ามไปยังส่วนถัดไปเพื่อทำการตั้งค่า DNS ด้วยตนเอง

เริ่มต้นโดยไปที่ส่วน **Settings->TSDNS** บนเว็บอินเทอร์เฟซของ TeamSpeak บนเว็บไซต์ของเรา ที่นี่คุณจะสามารถเลือกซับโดเมน (พรีฟิกซ์ก่อนโดเมน) รวมถึงเลือกใช้ `zap-ts3.com` ของเรา หรือโดเมนของคุณเองที่มีอยู่ในบัญชี ZAP-Hosting ของคุณ

:::info
สำหรับฟีเจอร์ตั้งค่าแบบด่วนนี้ คุณจำเป็นต้อง [ซื้อโดเมนคุณภาพตรงกับเราที่นี่](https://zap-hosting.com/en/shop/product/domain/) หรือย้ายโดเมนของคุณมายังบัญชี ZAP-Hosting โดเมนเหล่านี้จะปรากฏในส่วน TSDNS เมื่อถูกตั้งค่าแล้ว หรือคุณสามารถใช้ `zap-ts3.com` แบบฟรีได้เลย
:::

ในตัวอย่างนี้ คุณจะเห็นเราสร้างซับโดเมนใหม่โดยใช้โดเมน `.zap.cloud` ซึ่งจะนำคุณไปยังเซิร์ฟเวอร์ TeamSpeak 3 ของเราเมื่อใช้งาน

![](https://screensaver01.zap-hosting.com/index.php/s/Mi3wK7cY639Rp3x/preview)

:::note
โปรดทราบว่าอาจใช้เวลาถึง 3 ชั่วโมงสำหรับโดเมนในการแพร่กระจายและเริ่มทำงาน
:::

## การตั้งค่า DNS แบบปกติ

เพื่อส่งต่อที่อยู่ IP ให้สร้างซับโดเมนที่ส่งต่อไปยังที่อยู่ IP ของเซิร์ฟเวอร์ TeamSpeak 3 หรือส่งต่อโดเมนทั้งหมดโดยไม่ต้องใช้ซับโดเมน
วิธีนี้เพียงพอถ้าเซิร์ฟเวอร์ TeamSpeak 3 ใช้พอร์ตเริ่มต้น **9987**
แต่ถ้าเซิร์ฟเวอร์ TeamSpeak 3 ใช้พอร์ตอื่นที่ไม่ใช่พอร์ตเริ่มต้น จะต้องเพิ่ม **ระเบียน SRV** เพื่อส่งต่อโดเมนหรือซับโดเมนไปยังพอร์ตที่ถูกต้อง

### บริการ SRV

ชื่อบริการจะเริ่มต้นด้วย **ขีดล่าง** เสมอ
สำหรับเซิร์ฟเวอร์ TeamSpeak 3 ชื่อบริการจะเป็น **_ts3** เสมอ ไม่มีข้อยกเว้น

### โปรโตคอล SRV

โปรโตคอลคือโปรโตคอลอินเทอร์เน็ตที่ใช้สำหรับการเชื่อมต่อ โดยมีให้เลือก **UDP** และ **TCP**
การระบุโปรโตคอลจะเริ่มต้นด้วย **ขีดล่าง** เช่นกัน และจะเป็น **_udp** หรือ **_tcp**
สำหรับเซิร์ฟเวอร์ TeamSpeak 3 จะใช้ **_udp** เสมอ ไม่มีข้อยกเว้น

## การส่งต่อโดเมนโดยไม่ใช้ซับโดเมน

หากต้องการส่งต่อโดเมนไปยังเซิร์ฟเวอร์เกมโดยไม่สร้างซับโดเมน ให้เปิดโดเมนของคุณในแดชบอร์ด จากนั้นเปิดการจัดการ DNS จากเมนูด้านซ้าย

![](https://puu.sh/Fuzfa/0927cbb177.png)

![](https://puu.sh/FuzhO/6f4694ab62.png)

ที่นี่คุณจะเห็นระเบียน DNS ทั้งหมดที่มีอยู่สำหรับโดเมนของคุณ
ถ้าคุณยังไม่ได้สร้างระเบียนของตัวเอง คุณสามารถดูระเบียนทั้งหมดที่มีอยู่โดยคลิกที่ถังขยะสีแดงทางขวา

![](https://puu.sh/Fuzm8/39f3c72fa6.png)

จากนั้นคลิก **New entry** เพื่อสร้างระเบียน DNS ใหม่
ในช่อง **Name** ให้ใส่ชื่อโดเมน เช่น **teamspeak-server.de** เลือก **type เป็น A** และในช่อง **value** ให้ใส่ที่อยู่ IP ของเซิร์ฟเวอร์ TeamSpeak 3 ของคุณ เช่น **88.214.57.116**
ช่อง **TTL** สามารถปล่อยไว้ตามค่าเริ่มต้นได้

![](https://screensaver01.zap-hosting.com/index.php/s/cTGTdBFrigs7HDG/preview)

เมื่อกรอกข้อมูลครบแล้ว คลิก **Save** ระเบียนจะถูกบันทึกในตั้งค่า DNS และจะแสดงผลภายใน 24 ชั่วโมง

:::info
การเปลี่ยนแปลงระเบียน DNS อาจใช้เวลานานถึง 24 ชั่วโมงกว่าจะมีผล ไม่มีใครควบคุมเวลานี้ได้
:::

## การส่งต่อโดเมนโดยใช้ซับโดเมน

ถ้าคุณต้องการสร้างซับโดเมน เช่น ts.teamspeak-server.de ให้ทำเหมือนตัวอย่างก่อนหน้า
แต่ในช่อง **Name** ให้ใส่ชื่อซับโดเมนแทนชื่อโดเมนหลัก
ตัวอย่างจะเป็นแบบนี้:

![](https://screensaver01.zap-hosting.com/index.php/s/ocaqgX2DSdspGQ8/preview)

## การส่งต่อโดเมนพร้อมพอร์ต (SRV)

ถ้าไม่ได้ใช้พอร์ตมาตรฐาน **9987** จะต้องสร้าง **ระเบียน SRV** เพิ่มเติม เพื่อส่งต่อโดเมนไม่เพียงแค่ไปยัง IP ของเซิร์ฟเวอร์เกม แต่ยังไปยังพอร์ตที่ถูกต้องด้วย

ก่อนอื่นคุณต้องสร้าง **ซับโดเมน** ตามที่อธิบายไว้ข้างต้น หรือส่งต่อโดเมนโดยตรงไปยังเซิร์ฟเวอร์ TeamSpeak 3 ก็ได้ ทั้งสองวิธีใช้ได้

### การส่งต่อโดยไม่ใช้ซับโดเมน

หลังจากส่งต่อโดเมนไปยังที่อยู่ IP ของเซิร์ฟเวอร์ TeamSpeak 3 ตามที่อธิบายไว้ข้างต้นแล้ว ให้คลิก **New Entry** และสร้างระเบียนที่มีลักษณะดังนี้:

![](https://screensaver01.zap-hosting.com/index.php/s/H3NGmYZHyJsCYnf/preview)

ช่อง **Name** จะใส่ชื่อบริการ ซึ่งในกรณีนี้คือเซิร์ฟเวอร์ TeamSpeak 3 ของเรา คือ **_ts3** รวมถึงโปรโตคอลที่ใช้คือ **_udp** และตามด้วยชื่อโดเมน
ช่อง **Type** ระบุชนิดของระเบียน ในที่นี้คือ **SRV**
ช่อง **value** จะใส่พอร์ตเกมและโดเมน เช่น **0 1234 teamspeak-server.de** โดยที่ **1234** คือ **พอร์ตของเกม** และ **teamspeak-server.de** คือโดเมนที่ส่งต่อไป
ค่า **0** ไม่มีผลและคงที่เสมอ ช่อง **TTL** และ **Prio** ไม่ต้องแก้ไข

จากนั้นคลิก **Save**

:::info
สำคัญมาก! ต้องใส่ **จุด (dot)** ที่ท้ายโดเมนในช่อง **value** ด้วยนะ!
:::

### การส่งต่อโดยใช้ซับโดเมน

สำหรับซับโดเมนก็เหมือนกันเลย เริ่มจากสร้างซับโดเมนและส่งต่อไปยังที่อยู่ IP ของเซิร์ฟเวอร์ TeamSpeak 3 จากนั้นคลิก **New Entry** และสร้างระเบียนด้วยข้อมูลดังนี้:

![](https://screensaver01.zap-hosting.com/index.php/s/aEkro3z9WBgWf8g/preview)

ช่อง **Name** ใส่ชื่อบริการ คือ **_ts3** รวมถึงโปรโตคอล **_udp** และชื่อโดเมนพร้อมซับโดเมน เช่น **ts.teamspeak-server.de**
ช่อง **Type** คือ **SRV**
ช่อง **value** ใส่พอร์ตและโดเมนพร้อมซับโดเมน เช่น **0 1234 ts.teamspeak-server.de** โดยที่ **1234** คือ **พอร์ตของเซิร์ฟเวอร์ TeamSpeak 3** และ **ts.teamspeak-server.de** คือโดเมนพร้อมซับโดเมนที่ส่งต่อไป
ค่า **0** ไม่มีผลและคงที่ ช่อง **TTL** และ **Prio** ไม่ต้องแก้ไข
