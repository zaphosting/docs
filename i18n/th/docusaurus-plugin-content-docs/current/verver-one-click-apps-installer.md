## Introduction

ตัวติดตั้งแอปคลิกเดียวช่วยให้คุณเรียกดูแคตตาล็อกแอปขนาดใหญ่และติดตั้งแอปโดยตรงจากเว็บอินเทอร์เฟซของ VPS ในคู่มือนี้ คุณจะได้เรียนรู้วิธีเตรียมเซิร์ฟเวอร์ เลือกประเภทการติดตั้งที่ถูกต้อง ติดตั้งและจัดการแอป รวมถึงใช้การตั้งค่าขั้นสูงและเครื่องมือแก้ไขปัญหาเมื่อจำเป็น

<InlineVoucher />

## Preparation

ก่อนใช้ตัวติดตั้งแอปคลิกเดียว VPS ของคุณต้องตรงตามข้อกำหนดขั้นต่ำและใช้ระบบปฏิบัติการที่รองรับ

:::info ข้อกำหนดขั้นต่ำ
ฟีเจอร์แอปต้องการให้ VPS ของคุณออนไลน์และมีทรัพยากรเพียงพอสำหรับรันบริการแพลตฟอร์มและแอปที่ติดตั้งอย่างเสถียร
:::

| ข้อกำหนด | ขั้นต่ำ | แนะนำ |
| --- | --- | --- |
| สถานะเซิร์ฟเวอร์ | ออนไลน์ | ออนไลน์ |
| พื้นที่จัดเก็บ | 30 GB | 50 GB |
| RAM | 2 GB | 4 GB |
| CPU | 2 คอร์ | 4 คอร์ |
| ระบบปฏิบัติการ | Debian หรือ Ubuntu LTS | Debian 13 - EN - 64bit |

:::caution แนะนำติดตั้งระบบปฏิบัติการใหม่
แนะนำอย่างยิ่งให้ตั้งค่าฟีเจอร์แอปบนระบบปฏิบัติการที่ติดตั้งใหม่ เพื่อหลีกเลี่ยงความขัดแย้งกับบริการที่มีอยู่ พอร์ตที่ถูกใช้งาน การตั้งค่า Docker หรือการตั้งค่าที่ปรับแต่งไว้บนเซิร์ฟเวอร์
:::

หากต้องการติดตั้ง VPS ใหม่ก่อนดำเนินการต่อ ให้ใช้ภาพระบบ Debian หรือ Ubuntu LTS ที่รองรับและติดตั้งใหม่จากแผงจัดการ VPS ของคุณ

## Accessing the One Click Apps Installer

เปิด VPS ของคุณในเว็บอินเทอร์เฟซ ZAP-Hosting แล้วค้นหาส่วน One Click Apps จากตรงนั้นคุณสามารถเริ่มการตั้งค่าเริ่มต้นและกลับมาจัดการแอปทั้งหมดที่ติดตั้งบนเซิร์ฟเวอร์ได้ภายหลัง

ครั้งแรกที่เปิดฟีเจอร์นี้ คุณต้องเลือกวิธีติดตั้งแพลตฟอร์มบน VPS ของคุณ

## Choosing an Installation Type

คุณสามารถเลือกได้ระหว่างการติดตั้งแบบเต็มและการติดตั้งเฉพาะโหนด ตัวเลือกที่ถูกต้องขึ้นอยู่กับว่า VPS นี้จะโฮสต์ทั้งแผงควบคุมและแอป หรือทำหน้าที่เป็นโหนดที่เชื่อมต่อกับแผงควบคุมอื่นเท่านั้น

### Full Installation

เลือก **Full Installation** หากคุณต้องการตั้งค่าเซิร์ฟเวอร์พร้อมแผงควบคุมและเซิร์ฟเวอร์เองในฐานะโหนด นี่คือทางเลือกที่แนะนำสำหรับผู้ใช้ครั้งแรกและการติดตั้งมาตรฐานส่วนใหญ่

ด้วยวิธีนี้ คุณจะสามารถ:

- ติดตั้งแอปจากแคตตาล็อกแอป
- จัดการโปรเจกต์และสภาพแวดล้อม
- จัดระเบียบแอปทั้งหมดในที่เดียว
- ใช้เครื่องมือจัดการผ่านเว็บอินเทอร์เฟซที่รวมมาให้
- เข้าถึงแผงควบคุมขั้นสูงได้ในภายหลังหากต้องการ

![img](https://screensaver01.zap-hosting.com/index.php/s/nqGL3PFPicTArY9/preview)

:::tip แนะนำสำหรับผู้ใช้ส่วนใหญ่
ถ้านี่เป็นครั้งแรกที่คุณใช้ฟีเจอร์แอป ให้เลือกตัวเลือกติดตั้งแบบเต็ม เพราะเป็นการตั้งค่าที่ง่ายที่สุดและให้การจัดการครบถ้วนโดยตรงจากเว็บอินเทอร์เฟซของ ZAP-Hosting
:::

### Node-only Installation

เลือก **Node-only Installation** หากคุณต้องการให้ VPS นี้ทำหน้าที่เป็นโหนดเท่านั้น จากนั้นคุณสามารถเชื่อมต่อกับ VPS อื่นหรือแผงควบคุมภายนอกได้

ตัวเลือกนี้เหมาะสำหรับกรณีที่คุณต้องการ:

- ขยายระบบผ่าน VPS หลายเครื่อง
- รวมศูนย์การจัดการบนแผงควบคุมเดียว
- เชื่อมต่อเซิร์ฟเวอร์นี้กับการติดตั้งที่มีอยู่แล้ว
- ขยายระบบหลายโหนดที่มีอยู่

:::note ตัวเลือกการติดตั้งขั้นสูง
การติดตั้งแบบโหนดเท่านั้นเหมาะสำหรับผู้ใช้ที่มีแผงควบคุมอื่นอยู่แล้วหรือผู้ที่ต้องการเชื่อมต่อเซิร์ฟเวอร์นี้กับสภาพแวดล้อมภายนอก
:::

## Installing the Apps Feature

หลังเลือกประเภทการติดตั้ง ให้เริ่มการตั้งค่าจากเว็บอินเทอร์เฟซ การติดตั้งเริ่มต้นมักใช้เวลาไม่กี่นาที

ในขั้นตอนนี้ บริการแพลตฟอร์มที่จำเป็นสำหรับฟีเจอร์ One Click Apps จะถูกติดตั้งบน VPS ของคุณ เมื่อเสร็จสิ้น แคตตาล็อกแอปและส่วนจัดการจะพร้อมใช้งานในเว็บอินเทอร์เฟซ

:::info พฤติกรรมการติดตั้ง
รายละเอียดความคืบหน้าการติดตั้งอาจแตกต่างกันตามประสิทธิภาพ VPS สถานะระบบปฏิบัติการ และเงื่อนไขเครือข่าย หากการตั้งค่าไม่สำเร็จ ให้ตรวจสอบส่วนแก้ไขปัญหาภายหลังในคู่มือนี้
:::

## Browsing the App Catalogue

เมื่อการติดตั้งเสร็จสิ้น คุณสามารถเปิดแคตตาล็อกแอปและเรียกดูแอปที่มีให้เลือกได้ แคตตาล็อกมีแอปพร้อมติดตั้งมากกว่า 300 รายการและรองรับการกรองเพื่อช่วยให้คุณค้นหาแอปที่ต้องการได้อย่างรวดเร็ว

![](https://screensaver01.zap-hosting.com/index.php/s/fBwcCiNJTZooqnB/preview)

### Searching and Filtering Apps

ในแคตตาล็อก คุณสามารถค้นหาแอปตามชื่อและกรองตามหมวดหมู่ ช่วยให้จำกัดแอปทั้งหมดในอินเทอร์เฟซสไตล์สโตร์และหาจุดติดตั้งที่เหมาะสมสำหรับโปรเจกต์ของคุณ

การทำงานทั่วไปได้แก่:

- ค้นหาตามชื่อแอป
- กรองตามหมวดหมู่
- ตรวจสอบรายการแอปที่มี
- เลือกแอปเพื่อติดตั้ง

เวิร์กโฟลว์นี้มีประโยชน์มากหากคุณจัดการเป้าหมายดาวน์โหลดแอปจำนวนมากหรือเปรียบเทียบบริการหลายตัวก่อนติดตั้ง

### Starting an App Installation

หลังเลือกแอปจากแคตตาล็อก ให้เริ่มติดตั้งผ่านเว็บอินเทอร์เฟซ ขึ้นอยู่กับแอป คุณอาจต้องกำหนดค่าต่างๆ เช่น:

- `[your_project_name]`
- `[your_environment_name]`
- `[your_domain]`
- `[your_app_name]`

ตัวแปรเหล่านี้แทนค่าที่เฉพาะเจาะจงสำหรับการติดตั้งของคุณ เช่น `[your_domain]` ควรถูกแทนที่ด้วยโดเมนที่คุณต้องการใช้กับแอป

:::tip โปรเจกต์และสภาพแวดล้อม
ใช้โปรเจกต์และสภาพแวดล้อมเพื่อจัดระเบียบการติดตั้งของคุณ โดยเฉพาะอย่างยิ่งเมื่อคุณโฮสต์แอปหลายตัว ทดสอบเวอร์ชัน หรือแยกบริการระหว่างโปรดักชันและสเตจจิ้ง
:::

## Managing Installed Apps

หลังติดตั้ง แอปที่ติดตั้งจะแสดงในส่วนจัดการของเว็บอินเทอร์เฟซ จากตรงนี้คุณสามารถตรวจสอบและควบคุมบริการโดยไม่ต้องจัดการคอนเทนเนอร์ด้วยตนเอง

![](https://screensaver01.zap-hosting.com/index.php/s/WTMSqkJ5zPKWjmn/preview)

### Available Management Actions

ส่วนแอปที่ติดตั้งช่วยให้คุณทำงานวงจรชีวิตทั่วไปได้โดยตรงจากเว็บอินเทอร์เฟซ

| การกระทำ | จุดประสงค์ |
| --- | --- |
| Start | เริ่มแอปที่หยุดทำงาน |
| Stop | หยุดแอปที่กำลังทำงาน |
| Check logs | เปิดบันทึกแอปเพื่อแก้ไขปัญหา |
| Reinstall | ติดตั้งแอปใหม่ |

เครื่องมือเหล่านี้ช่วยให้จัดการแอปทั้งหมดได้ง่ายขึ้นจากที่เดียว โดยเฉพาะถ้าคุณต้องการควบคุมผ่านเว็บแทนการใช้คำสั่งด้วยตนเอง

### Organizing Projects and Environments

โปรเจกต์และสภาพแวดล้อมช่วยให้คุณจัดโครงสร้างการติดตั้งได้ชัดเจน คุณสามารถใช้แยก:

- แอปส่วนตัวและแอปโปรดักชัน
- สภาพแวดล้อมทดสอบและใช้งานจริง
- การติดตั้งลูกค้าหรือทีมหลายราย
- สแตกต่างๆ บน VPS เดียวกัน

การจัดระเบียบนี้สำคัญมากขึ้นเมื่อคอลเลกชันแอปในสโตร์ของคุณเพิ่มขึ้นและคุณจัดการบริการมากขึ้นบนโหนดหนึ่งหรือหลายโหนด

## Domains, Cloud Access, and SSL

ฟีเจอร์ One Click Apps รองรับโดเมนที่กำหนดเองและการรวมกับ ZAP Cloud ช่วยให้คุณเปิดเผยแอปอย่างปลอดภัยและจัดเตรียมใบรับรอง SSL สำหรับการติดตั้งของคุณ

:::info รองรับโดเมนและ SSL
คุณสามารถใช้โดเมนที่กำหนดเองสำหรับแอปและได้รับประโยชน์จากการจัดเตรียมใบรับรอง SSL เมื่อแพลตฟอร์มรองรับ
:::

เมื่อกำหนดโดเมน ให้แน่ใจว่าระเบียน DNS ชี้ไปที่ `[your_server_ip]` หากโดเมนของคุณตั้งค่าไม่ถูกต้อง แอปอาจไม่สามารถเข้าถึงได้จากอินเทอร์เน็ต

| ค่า | คำอธิบาย |
| --- | --- |
| `[your_domain]` | โดเมนหรือซับโดเมนที่กำหนดให้แอปของคุณ |
| `[your_server_ip]` | ที่อยู่ IP สาธารณะของ VPS คุณ |

:::caution DNS ต้องชี้ไปยัง VPS ของคุณ
หาก DNS สำหรับ `[your_domain]` ไม่ชี้ไปที่ `[your_server_ip]` การตรวจสอบโดเมนและการเข้าถึงแอปอาจล้มเหลว
:::

## Using Advanced Settings

ผู้ใช้ขั้นสูงสามารถเข้าถึงฟังก์ชันเพิ่มเติมผ่านแผงควบคุมภายนอกที่มีในส่วนการตั้งค่าขั้นสูง

![](https://screensaver01.zap-hosting.com/index.php/s/BHnZWBGAzgo5myp/preview)

สิ่งนี้มีประโยชน์หากคุณต้องการปรับแต่งลึกขึ้น ควบคุมการติดตั้งขยาย หรือเข้าถึงฟีเจอร์ที่เกินกว่าเว็บอินเทอร์เฟซมาตรฐาน

### Important Support Limitation

:::caution ขอบเขตการสนับสนุนสำหรับการใช้แผงควบคุมขั้นสูง
ผู้ใช้ขั้นสูงสามารถใช้แผง Coolify ภายนอกเพื่อปรับแต่งและฟังก์ชันเพิ่มเติมได้ แต่การสนับสนุนจำกัดเฉพาะเว็บอินเทอร์เฟซของ ZAP-Hosting หากคุณเปลี่ยนแปลงนอกเว็บอินเทอร์เฟซ อาจไม่ได้รับความช่วยเหลือสำหรับการปรับแต่งเหล่านั้น
:::

### When to Use the Advanced Panel

คุณอาจต้องใช้แผงขั้นสูงหากต้องการ:

- ปรับแต่งการติดตั้งอย่างละเอียด
- เวิร์กโฟลว์การเชื่อมต่อโหนดโดยตรง
- การตั้งค่าแอประดับสูง
- การจัดการแผงควบคุมที่เกินกว่าหน้าเว็บแบบง่าย

หากคุณสูญเสียการเข้าถึงหรือการเชื่อมต่อหยุดตอบสนอง ให้ใช้เครื่องมือแก้ไขปัญหาที่อธิบายไว้ด้านล่างก่อนทำการเปลี่ยนแปลงด้วยตนเอง

## Linking Nodes

ถ้าคุณเลือกการติดตั้งแบบโหนดเท่านั้น คุณสามารถเชื่อมต่อ VPS นี้กับ VPS อื่นหรือแผงควบคุมภายนอกได้ ช่วยให้ขยายโครงสร้างพื้นฐานและจัดการโหนดหลายตัวจากที่เดียว

การเชื่อมต่อโหนดเหมาะสำหรับกรณีที่คุณต้องการ:

- กระจายแอปไปยัง VPS หลายเครื่อง
- รวมศูนย์การจัดการ
- ขยายความจุโดยไม่สร้างการติดตั้งแยก
- เชื่อมต่อกับสภาพแวดล้อมแผงควบคุมภายนอก

:::note การเชื่อมต่อแผงควบคุมภายนอก
คุณสามารถเชื่อมต่อโหนดกับ VPS อื่นหรือแผงควบคุมภายนอก รายละเอียดการเชื่อมต่อขึ้นอยู่กับสภาพแวดล้อมเป้าหมายที่ใช้งานอยู่แล้ว
:::

## Troubleshooting Degraded Status

ถ้าเว็บอินเทอร์เฟซแสดงสถานะเสื่อม หมายความว่าแพลตฟอร์มไม่สามารถติดต่อหรือดึงข้อมูลจากอินสแตนซ์ที่รันฟีเจอร์ One Click Apps ได้ ให้ทำตามขั้นตอนแก้ไขปัญหาเรียงตามลำดับเวลา

### Status Indicators

ป๊อปอัปแก้ไขปัญหาอาจแสดงตัวชี้วัดดังนี้:

| สถานะ | ความหมาย |
| --- | --- |
| Coolify Status: Active | บริการแผงควบคุมดูเหมือนกำลังทำงาน |
| API Status: Accessible | API สามารถเข้าถึงได้ในขณะนี้ |
| Service Status: Online | VPS ของคุณออนไลน์ |

ถ้าส่วนใดส่วนหนึ่งไม่ทำงาน ให้ดำเนินการตามขั้นตอนด้านล่างต่อไป

### Step 1: Confirm the VPS Is Online

ตรวจสอบให้แน่ใจว่า VPS ของคุณออนไลน์ในเว็บอินเทอร์เฟซ ZAP-Hosting หากเซิร์ฟเวอร์ออฟไลน์ แพลตฟอร์มแอปจะไม่ทำงาน

### Step 2: Refresh the Status

ใช้คำสั่ง **Refresh Status** เพื่อตรวจสอบสุขภาพระบบอีกครั้งด้วยข้อมูลล่าสุด วิธีนี้ช่วยแก้ไขปัญหาชั่วคราวที่เกิดขึ้นเป็นบางครั้งได้

:::tip ปัญหาการเชื่อมต่อชั่วคราว
สถานะเสื่อมอาจเกิดจากปัญหาการสื่อสารชั่วคราว การรีเฟรชสถานะเป็นการตรวจสอบเบื้องต้นที่เร็วที่สุด
:::

### Step 3: Restart Docker

ถ้าสถานะยังเสื่อมอยู่ ให้ใช้คำสั่ง **Restart Docker** เพื่อบังคับให้อินสแตนซ์แพลตฟอร์มรีสตาร์ท หลังจากตอบสนองสำเร็จ รอ `1-2 นาที` แล้วรีเฟรชสถานะอีกครั้ง

:::caution การรีสตาร์ทบริการ
การรีสตาร์ท Docker อาจทำให้แอปที่กำลังรันหยุดชั่วคราวในระหว่างที่บริการกำลังรีสตาร์ท
:::

### Step 4: Refresh Credentials

ขั้นตอนสุดท้าย ใช้คำสั่ง **Refresh Credentials** ซึ่งสำคัญมากถ้าสถานะดูออนไลน์แต่ฟังก์ชันในเว็บอินเทอร์เฟซ เช่น การติดตั้งแอป ไม่ทำงาน

คำสั่งนี้:

- **ไม่** ลบข้อมูลของคุณ
- รีเฟรชข้อมูลรับรองผู้ใช้ root
- รีเฟรชโทเค็น API ที่เว็บอินเทอร์เฟซใช้
- อัปเดตข้อมูลรับรองแผงควบคุมโดยอัตโนมัติเมื่อสำเร็จ

:::info การรีเฟรชข้อมูลรับรองอย่างปลอดภัย
การรีเฟรชข้อมูลรับรองไม่ลบแอปที่ติดตั้งหรือข้อมูลการติดตั้งใดๆ เพียงแค่ต่ออายุข้อมูลรับรองที่จำเป็นสำหรับการเข้าถึงและการสื่อสาร
:::

### When to Contact Support

ถ้าปัญหายังไม่หายหลังทำตามขั้นตอนแก้ไขปัญหาทั้งหมด ให้สร้างตั๋วซัพพอร์ตผ่านเว็บไซต์ ZAP-Hosting

## Best Practices

เพื่อให้ตัวติดตั้งแอปคลิกเดียวเสถียรและจัดการง่าย ให้ทำตามคำแนะนำเหล่านี้:

| แนวทางปฏิบัติที่ดีที่สุด | เหตุผล |
| --- | --- |
| ใช้การติดตั้งระบบปฏิบัติการใหม่ | ลดความขัดแย้งกับบริการและพอร์ตที่มีอยู่ |
| ตรงตามทรัพยากรที่แนะนำ | เพิ่มประสิทธิภาพและความเสถียร |
| จัดระเบียบแอปเป็นโปรเจกต์และสภาพแวดล้อม | ทำให้การติดตั้งจัดการง่ายขึ้น |
| ตั้งค่า DNS ก่อนกำหนดโดเมน | ป้องกันปัญหาการเข้าถึงและ SSL |
| ใช้การตั้งค่าขั้นสูงอย่างระมัดระวัง | หลีกเลี่ยงการเปลี่ยนแปลงที่ไม่รองรับ |

:::tip รักษาการติดตั้งให้เป็นระเบียบ
ถ้าคุณวางแผนโฮสต์แอปจำนวนมาก ให้สร้างระบบตั้งชื่อที่ชัดเจนสำหรับ `[your_project_name]`, `[your_environment_name]`
