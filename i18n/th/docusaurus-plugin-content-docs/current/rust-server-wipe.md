## บทนำ

ใน **Rust** การ wipe เซิร์ฟเวอร์จะรีเซ็ตส่วนต่างๆ ของโลกในเกมเพื่อสร้างจุดเริ่มต้นใหม่สำหรับผู้เล่นทุกคน เมื่อเวลาผ่านไป แผนที่ Rust จะเต็มไปด้วยฐานขนาดใหญ่ ทรัพยากรที่สะสม และกลุ่มผู้เล่นที่ตั้งตัวแล้ว ซึ่งสิ่งนี้อาจส่งผลต่อสมดุลและประสิทธิภาพของเซิร์ฟเวอร์

การ wipe เซิร์ฟเวอร์จะลบความก้าวหน้าที่มีอยู่และคืนสภาพแวดล้อมให้สะอาด เพื่อให้ผู้เล่นเริ่มต้นใหม่ได้อีกครั้ง ชุมชน Rust หลายแห่งทำการ wipe เป็นประจำเพื่อรักษาความยุติธรรมและความสนุกของเกม โดยเฉพาะอย่างยิ่งเมื่อมีการอัปเดตหรือแผนที่ใหม่ๆ ออกมา

ขึ้นอยู่กับประเภทของการ wipe อาจจะรีเซ็ตแค่ **แผนที่** , **blueprints ของผู้เล่น** หรือ **ทั้งสองอย่าง**

## การ Wipe เซิร์ฟเวอร์ทำอะไรบ้าง

การ wipe เซิร์ฟเวอร์ Rust จะลบข้อมูลความก้าวหน้าของโลกหรือผู้เล่นที่เก็บไว้ ซึ่งอาจรวมถึง:

- โครงสร้างและฐานที่ผู้เล่นสร้างขึ้น  
- อินเวนทอรีและไอเท็มที่เก็บไว้ของผู้เล่น  
- ข้อมูลการสำรวจแผนที่  
- blueprints ที่เรียนรู้แล้ว (ขึ้นอยู่กับประเภทของการ wipe)

หลังจาก wipe เซิร์ฟเวอร์จะทำงานเหมือนโลกใหม่ที่ผู้เล่นต้องเก็บทรัพยากร สร้างฐานใหม่ และเรียนรู้ความก้าวหน้าซ้ำอีกครั้ง



## ประเภทของการ Wipe ใน Rust

Rust รองรับหลายประเภทของการ wipe ขึ้นอยู่กับสิ่งที่คุณต้องการรีเซ็ต ประเภทของการ wipe ที่มีจะอธิบายดังนี้

### การ Wipe แผนที่

**การ wipe แผนที่** จะลบไฟล์เซฟโลกทั้งหมด ฐานทั้งหมด โครงสร้าง และความก้าวหน้าบนแผนที่จะถูกลบ เมื่อเซิร์ฟเวอร์รีสตาร์ท แผนที่ใหม่จะถูกสร้างขึ้นหรือดาวน์โหลดมา ความก้าวหน้าของ blueprints จะไม่เปลี่ยนแปลงเว้นแต่จะทำการ wipe blueprints ด้วย



### การ Wipe Blueprint

**การ wipe blueprint** จะรีเซ็ต blueprints การคราฟต์ที่ผู้เล่นเรียนรู้ทั้งหมด ผู้เล่นต้องวิจัยไอเท็มใหม่ก่อนจะคราฟต์ได้ การ wipe ประเภทนี้จะส่งผลแค่ความก้าวหน้าและไม่รีเซ็ตแผนที่



### การ Wipe แบบเต็ม

**การ wipe แบบเต็ม** จะรีเซ็ตทั้งแผนที่และ blueprints ของผู้เล่นทั้งหมด สร้างสภาพแวดล้อมเซิร์ฟเวอร์ใหม่ทั้งหมด มักใช้สำหรับการรีเซ็ตตามฤดูกาลหรืออัปเดตใหญ่ๆ



## ทำไมการ Wipe เซิร์ฟเวอร์ถึงสำคัญ

การ wipe เซิร์ฟเวอร์ช่วยรักษาสภาพแวดล้อมเซิร์ฟเวอร์ Rust ให้ดีอยู่เสมอ เมื่อเวลาผ่านไป เซิร์ฟเวอร์จะสะสมโครงสร้างจำนวนมาก กล่องเก็บของ และเอนทิตีในโลก ซึ่งอาจส่งผลเสียต่อประสิทธิภาพเซิร์ฟเวอร์และสมดุลของเกม การ wipe เป็นประจำมีข้อดีหลายอย่าง:

- คืนประสิทธิภาพเซิร์ฟเวอร์
- สร้างจุดเริ่มต้นใหม่สำหรับผู้เล่นทุกคน
- ป้องกันกลุ่มผู้เล่นที่ครองแผนที่ได้ตลอดไป
- เปิดโอกาสให้เซิร์ฟเวอร์ใช้แผนที่หรือ seed ใหม่ๆ

ชุมชน Rust หลายแห่งจะทำการ wipe ตามตารางรายสัปดาห์ รายสองสัปดาห์ หรือรายเดือน ขึ้นอยู่กับสไตล์เซิร์ฟเวอร์



## วิธีการ Wipe เซิร์ฟเวอร์

เพื่อทำการ wipe เซิร์ฟเวอร์ Rust ของคุณ ให้เข้าถึงไฟล์เซิร์ฟเวอร์ผ่าน [การเข้าถึง FTP](gameserver-ftpaccess.md) จากนั้นค้นหาไฟล์เซฟในไดเรกทอรีเซิร์ฟเวอร์ Rust ไฟล์เหล่านี้มักจะมี:

```
proceduralmap.*
player.blueprints.*
player.deaths.*
player.identities.*
player.states.*
```
การลบไฟล์เฉพาะจะกำหนดประเภทของการ wipe ที่ทำ การลบไฟล์ `proceduralmap.*` จะเป็นการ **wipe แผนที่** ลบโลกที่มีอยู่ การลบไฟล์ `player.blueprints.*` จะเป็นการ **wipe blueprint**

![img](https://screensaver01.zap-hosting.com/index.php/s/QdeoFWbaaQz7Yaf/preview)

ถ้าลบทั้งสองอย่างจะเป็นการ **wipe แบบเต็ม** หลังจากลบไฟล์ที่ต้องการแล้ว ให้เริ่มเซิร์ฟเวอร์ใหม่ เซิร์ฟเวอร์จะสร้างโลกใหม่โดยอัตโนมัติและให้ผู้เล่นเริ่มต้นใหม่ได้เลย
