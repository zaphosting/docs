## แนะนำ

การจัดการสิทธิ์บนเซิร์ฟเวอร์ Hytale ช่วยให้คุณควบคุมสิ่งที่ผู้เล่นสามารถทำและไม่สามารถทำได้ ซึ่งรวมถึงการกำหนดระดับการเข้าถึงที่แตกต่างกันให้กับผู้เล่นที่ไว้ใจได้, ผู้ดูแล และแอดมิน การตั้งค่าสิทธิ์อย่างถูกต้องเป็นสิ่งสำคัญเพื่อรักษาสภาพแวดล้อมเซิร์ฟเวอร์ที่สมดุลและปลอดภัย ที่ผู้เล่นจะได้สนุกกับเกมโดยไม่มีการแสวงหาประโยชน์หรือการละเมิดที่ไม่ตั้งใจ

ซอฟต์แวร์เซิร์ฟเวอร์ของ Hytale รองรับระดับสิทธิ์แบบลำดับชั้นที่กำหนดคำสั่งและการกระทำที่แต่ละผู้เล่นสามารถทำได้ ซึ่งสามารถจัดการผ่านคอนโซลสดหรือผ่านการตั้งค่าคอนฟิกขึ้นอยู่กับการตั้งค่าเซิร์ฟเวอร์ของคุณ

:::info ประกาศ Early Access

Hytale เปิดตัวเมื่อวันที่ 13 มกราคม 2026 และปัจจุบันอยู่ในช่วง Early Access เนื่องจากเกมยังอยู่ในช่วงพัฒนาต่อเนื่อง ซอฟต์แวร์เซิร์ฟเวอร์, ไฟล์คอนฟิก, การรองรับม็อด และขั้นตอนการติดตั้งอาจมีการเปลี่ยนแปลงไปตามเวลา

:::

## ภาพรวมระดับสิทธิ์

สิทธิ์กำหนดการกระทำที่ผู้เล่นได้รับอนุญาตให้ทำบนเซิร์ฟเวอร์ ในระดับพื้นฐานที่สุด ผู้เล่นทั่วไปจะมีสิทธิ์เล่นเกมมาตรฐาน เช่น การเคลื่อนที่, การโต้ตอบกับโลก และการแชท ระดับสิทธิ์ที่สูงขึ้น เช่น สถานะผู้ดูแลหรือแอดมิน จะให้สิทธิ์เข้าถึงคำสั่งเซิร์ฟเวอร์ที่ส่งผลต่อการเล่นเกม, ผู้เล่นคนอื่น, การตั้งค่าเซิร์ฟเวอร์ และเครื่องมือการดูแล

ระบบสิทธิ์ประกอบด้วยสองส่วนหลัก:

- **สิทธิ์ผู้ใช้** ที่ใช้กับผู้เล่นเฉพาะเจาะจงโดยตรง
- **สิทธิ์กลุ่ม** ที่อนุญาตให้รวมสิทธิ์และมอบหมายให้กับผู้เล่นหลายคนพร้อมกัน

ผู้เล่นแต่ละคนจะถูกระบุภายในด้วย UUID ซึ่งจำเป็นเมื่อจัดการสิทธิ์ผ่านคำสั่ง



## การจัดการสิทธิ์ผู้ใช้

สิทธิ์ผู้ใช้ช่วยให้คุณมอบหรือเพิกถอนสิทธิ์เฉพาะสำหรับผู้เล่นคนเดียว

### ดูสิทธิ์ผู้ใช้

เพื่อแสดงสิทธิ์ทั้งหมดที่มอบให้กับผู้ใช้โดยตรง คำสั่งนี้จะแสดงสิทธิ์ทั้งหมดที่ใช้กับผู้ใช้ที่ระบุ

```
/perm user list <uuid>
```



### เพิ่มสิทธิ์ให้ผู้ใช้

เพื่อมอบสิทธิ์หนึ่งหรือหลายสิทธิ์ให้กับผู้ใช้โดยตรง สิทธิ์ที่ระบุจะถูกเพิ่มทันทีและมีผลโดยไม่ต้องรีสตาร์ทเซิร์ฟเวอร์

```
/perm user add <uuid> <permissions>
```



### ลบสิทธิ์จากผู้ใช้

เพื่อเพิกถอนสิทธิ์จากผู้ใช้ จะลบเฉพาะสิทธิ์ที่ระบุโดยไม่กระทบสิทธิ์อื่น ๆ

```
/perm user remove <uuid> <permissions>
```



## การจัดการการมอบหมายกลุ่มผู้ใช้

นอกจากสิทธิ์โดยตรงแล้ว ผู้ใช้ยังสามารถรับสิทธิ์ผ่านกลุ่มได้

### ดูสิทธิ์กลุ่มของผู้ใช้

คำสั่งนี้จะแสดงว่าผู้ใช้เป็นสมาชิกของกลุ่มสิทธิ์ใดบ้างในปัจจุบัน

```
/perm user group list <uuid>
```



### เพิ่มผู้ใช้ในกลุ่ม

เพื่อมอบหมายผู้ใช้ให้กับกลุ่มสิทธิ์ เมื่อเพิ่มแล้ว ผู้ใช้จะได้รับสิทธิ์ทั้งหมดที่กำหนดไว้สำหรับกลุ่มนั้น

```
/perm user group add <uuid> <group>
```



### ลบผู้ใช้ออกจากกลุ่ม

เพื่อลบผู้ใช้ออกจากกลุ่มสิทธิ์ หลังจากลบแล้ว ผู้ใช้จะไม่ได้รับสิทธิ์จากกลุ่มนั้นอีกต่อไป

```
/perm user group remove <uuid> <group>
```



## การจัดการสิทธิ์กลุ่ม

กลุ่มช่วยให้การจัดการสิทธิ์เป็นศูนย์กลางและนำกลับมาใช้ซ้ำกับผู้ใช้หลายคนได้

### ดูสิทธิ์กลุ่ม

เพื่อแสดงสิทธิ์ทั้งหมดที่มอบให้กับกลุ่ม คำสั่งนี้ให้ภาพรวมของสิทธิ์ทั้งหมดที่เกี่ยวข้องกับกลุ่มที่ระบุ

```
/perm group list <group>
```



### เพิ่มสิทธิ์ให้กลุ่ม

เพื่อเพิ่มสิทธิ์หนึ่งหรือหลายสิทธิ์ให้กับกลุ่ม ผู้ใช้ทั้งหมดที่อยู่ในกลุ่มนี้จะได้รับสิทธิ์ใหม่ทันที

```
/perm group add <group> <permissions>
```



### ลบสิทธิ์จากกลุ่ม

คำสั่งนี้จะลบเฉพาะสิทธิ์ที่ระบุออกจากกลุ่มโดยไม่กระทบสิทธิ์อื่น ๆ

```
/perm group remove <group> <permissions>
```
