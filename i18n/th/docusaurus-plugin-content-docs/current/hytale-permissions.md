## บทนำ

การจัดการสิทธิ์บนเซิร์ฟเวอร์ Hytale ช่วยให้คุณควบคุมได้ว่าใครสามารถทำอะไรได้บ้าง ซึ่งรวมถึงการมอบระดับการเข้าถึงที่แตกต่างกันให้กับผู้เล่นที่ไว้ใจได้, ผู้ดูแล และแอดมิน การตั้งค่าสิทธิ์อย่างถูกต้องเป็นสิ่งสำคัญเพื่อรักษาสภาพแวดล้อมเซิร์ฟเวอร์ที่สมดุลและปลอดภัย ที่ผู้เล่นจะได้สนุกกับเกมโดยไม่มีการแฮ็กหรือการใช้งานในทางที่ผิดโดยไม่ได้ตั้งใจ

ซอฟต์แวร์เซิร์ฟเวอร์ของ Hytale รองรับระดับสิทธิ์แบบลำดับชั้นที่กำหนดคำสั่งและการกระทำที่ผู้เล่นแต่ละคนสามารถทำได้ ซึ่งสามารถจัดการผ่านคอนโซลสดหรือผ่านการตั้งค่าคอนฟิก ขึ้นอยู่กับการตั้งค่าเซิร์ฟเวอร์ของคุณ

:::info แจ้งเตือน Early Access

Hytale เปิดตัวเมื่อวันที่ 13 มกราคม 2026 และปัจจุบันอยู่ในช่วง Early Access เนื่องจากเกมยังอยู่ในช่วงพัฒนาต่อเนื่อง ซอฟต์แวร์เซิร์ฟเวอร์, ไฟล์คอนฟิก, การรองรับม็อด และขั้นตอนการติดตั้ง อาจมีการเปลี่ยนแปลงไปตามเวลา

:::

## ภาพรวมระดับสิทธิ์

สิทธิ์กำหนดว่าผู้เล่นสามารถทำอะไรได้บ้างบนเซิร์ฟเวอร์ ในระดับพื้นฐานที่สุด ผู้เล่นทั่วไปจะมีสิทธิ์เล่นเกมมาตรฐาน เช่น การเคลื่อนที่, การโต้ตอบกับโลก และการแชท ระดับสิทธิ์ที่สูงขึ้น เช่น ผู้ดูแลหรือแอดมิน จะได้รับสิทธิ์เข้าถึงคำสั่งเซิร์ฟเวอร์ที่ส่งผลต่อการเล่นเกม, ผู้เล่นอื่น, การตั้งค่าเซิร์ฟเวอร์ และเครื่องมือดูแล

ระบบสิทธิ์ประกอบด้วยสองส่วนหลัก:

- **สิทธิ์ผู้ใช้** ที่ใช้กับผู้เล่นเฉพาะเจาะจงโดยตรง
- **สิทธิ์กลุ่ม** ที่ช่วยให้รวมสิทธิ์และมอบให้กับผู้เล่นหลายคนพร้อมกันได้

ผู้เล่นแต่ละคนจะถูกระบุด้วย UUID ภายใน ซึ่งจำเป็นสำหรับการจัดการสิทธิ์ผ่านคำสั่ง



## การจัดการสิทธิ์ผู้ใช้

สิทธิ์ผู้ใช้ช่วยให้คุณมอบหรือยกเลิกสิทธิ์เฉพาะสำหรับผู้เล่นคนเดียวได้

### ดูสิทธิ์ผู้ใช้

แสดงสิทธิ์ทั้งหมดที่มอบให้กับผู้ใช้โดยตรง คำสั่งนี้จะแสดงสิทธิ์ทุกอย่างที่ใช้กับผู้ใช้ที่ระบุ

```
/perm user list <uuid>
```



### เพิ่มสิทธิ์ให้ผู้ใช้

มอบสิทธิ์หนึ่งหรือหลายสิทธิ์ให้กับผู้ใช้โดยตรง สิทธิ์ที่ระบุจะถูกเพิ่มทันทีและมีผลโดยไม่ต้องรีสตาร์ทเซิร์ฟเวอร์

```
/perm user add <uuid> <permissions>
```



### ลบสิทธิ์จากผู้ใช้

ยกเลิกสิทธิ์จากผู้ใช้ โดยจะลบเฉพาะสิทธิ์ที่ระบุไว้เท่านั้น ส่วนสิทธิ์อื่นๆ จะยังคงอยู่เหมือนเดิม

```
/perm user remove <uuid> <permissions>
```



## การจัดการการมอบหมายกลุ่มผู้ใช้

นอกจากสิทธิ์โดยตรงแล้ว ผู้ใช้ยังสามารถได้รับสิทธิ์ผ่านกลุ่มได้

### ดูสิทธิ์กลุ่มของผู้ใช้

คำสั่งนี้แสดงว่าผู้ใช้เป็นสมาชิกของกลุ่มสิทธิ์ใดบ้างในขณะนี้

```
/perm user group list <uuid>
```



### เพิ่มผู้ใช้เข้าสู่กลุ่ม

มอบหมายผู้ใช้เข้าสู่กลุ่มสิทธิ์ เมื่อเพิ่มแล้ว ผู้ใช้จะได้รับสิทธิ์ทั้งหมดที่กำหนดไว้ในกลุ่มนั้น

```
/perm user group add <uuid> <group>
```



### ลบผู้ใช้ออกจากกลุ่ม

ลบผู้ใช้ออกจากกลุ่มสิทธิ์ หลังจากลบแล้ว ผู้ใช้จะไม่ได้รับสิทธิ์จากกลุ่มนั้นอีกต่อไป

```
/perm user group remove <uuid> <group>
```



## การจัดการสิทธิ์กลุ่ม

กลุ่มช่วยให้จัดการสิทธิ์ได้แบบรวมศูนย์และนำกลับมาใช้ซ้ำกับผู้ใช้หลายคนได้

### ดูสิทธิ์กลุ่ม

แสดงสิทธิ์ทั้งหมดที่มอบให้กับกลุ่ม คำสั่งนี้ให้ภาพรวมของสิทธิ์ทั้งหมดที่เกี่ยวข้องกับกลุ่มที่ระบุ

```
/perm group list <group>
```



### เพิ่มสิทธิ์ให้กลุ่ม

เพิ่มสิทธิ์หนึ่งหรือหลายสิทธิ์ให้กับกลุ่ม ผู้ใช้ทุกคนที่อยู่ในกลุ่มนี้จะได้รับสิทธิ์ใหม่ทันที

```
/perm group add <group> <permissions>
```



### ลบสิทธิ์จากกลุ่ม

ลบเฉพาะสิทธิ์ที่ระบุออกจากกลุ่มโดยไม่กระทบสิทธิ์อื่นๆ

```
/perm group remove <group> <permissions>
```
