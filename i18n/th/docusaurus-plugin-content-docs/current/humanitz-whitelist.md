## บทนำ

Whitelist ช่วยให้คุณควบคุมได้อย่างแม่นยำว่าใครสามารถเข้าร่วมเซิร์ฟเวอร์เกม HumanitZ ของคุณได้ เมื่อเปิดใช้งาน จะมีเพียงผู้เล่นที่มี Steam ID อยู่ในไฟล์ผู้เล่นที่อนุญาตเท่านั้นที่สามารถเชื่อมต่อได้

## การตั้งค่า

ฟังก์ชัน whitelist ถูกควบคุมภายในไฟล์ `GameServerSettings.ini` คุณสามารถเข้าถึงไฟล์นี้ได้ในแผงควบคุมเซิร์ฟเวอร์ของคุณภายใต้ **Configs** เพื่อเปิดใช้งานโหมด whitelist ให้ค้นหาและตั้งค่าพารามิเตอร์ดังนี้:

```
OnlyAllowedPlayers=1
```

- `OnlyAllowedPlayers=1` – เปิดใช้งานโหมด whitelist จะอนุญาตให้เฉพาะผู้เล่นที่ระบุในรายชื่อเท่านั้นที่สามารถเข้าร่วมได้
- `OnlyAllowedPlayers=0` – ปิดโหมด whitelist ผู้เล่นทุกคนสามารถเข้าร่วมได้

หลังจากเปิดใช้งานโหมด whitelist แล้ว คุณต้องสร้างไฟล์ whitelist ด้วยตัวเอง

เชื่อมต่อกับเซิร์ฟเวอร์ของคุณผ่าน [FTP](gameserver-ftpaccess.md) โดยใช้โปรแกรม FTP ที่คุณชอบ เมื่อเชื่อมต่อแล้ว ให้ไปที่ไดเรกทอรีหลักของเซิร์ฟเวอร์และสร้างไฟล์ใหม่ชื่อ `F_MVPAccess.txt` ในไฟล์นี้ ให้ใส่ Steam ID ที่ไม่ซ้ำกัน **หนึ่งบรรทัดต่อหนึ่ง Steam ID** สำหรับผู้เล่นแต่ละคนที่คุณต้องการอนุญาต

```
76561198000000001
76561198000000002
```

บันทึกไฟล์หลังจากเพิ่ม Steam ID แล้ว

## การตรวจสอบกิจกรรมผู้เล่น

HumanitZ มีไฟล์เสริมสำหรับติดตามการเชื่อมต่อของผู้เล่น `F_ConnectedPlayers.txt` จะแสดงผู้เล่นที่กำลังเชื่อมต่ออยู่ และ `PlayerConnectedLog.txt` จะบันทึกการเชื่อมต่อและการตัดการเชื่อมต่อของผู้เล่นทั้งหมด รวมถึงชื่อ Steam และ Steam ID ที่ไม่ซ้ำ พร้อมเวลาที่เกิดเหตุการณ์

ไฟล์เหล่านี้มีประโยชน์สำหรับการระบุ Steam ID และตรวจสอบกิจกรรมบนเซิร์ฟเวอร์ หลังจากแก้ไขไฟล์ whitelist หรือไฟล์แบนแล้ว ให้บันทึกไฟล์และรีสตาร์ทเซิร์ฟเวอร์เพื่อให้การเปลี่ยนแปลงมีผลอย่างถูกต้อง

## สรุป

ยินดีด้วย! ด้วยการเปิดใช้งานโหมด whitelist และจัดการไฟล์ `F_MVPAccess.txt` กับ `F_BannedPlayers.txt` คุณได้ตั้งค่าการควบคุมการเข้าถึงบนเซิร์ฟเวอร์เกม HumanitZ ของคุณเรียบร้อยแล้ว

หากมีคำถามเพิ่มเติมหรือต้องการความช่วยเหลือ อย่าลังเลที่จะติดต่อทีมซัพพอร์ตของเรา ซึ่งพร้อมให้บริการคุณทุกวัน! 🙂
